#шаблоны_проектирования  #порожающие

Когда заранее ивестен интерфейс, но не определено, какая реализация будет использоваться.
Получаем ссылку на объект в зависимости от входящих условий.

Вся суть в том, что для создания объекта реализующего некий интерфейс используется метод, который сам подберёт необходимую реализацию от условия.

``` JAVA
//некоторый интерфейс
public interface Car {   
    void drive();  
    void start();  
}

//реализации
public class Geep implements Car{  
    String name = "geep";  
    @Override  
    public void drive() {  System.out.println("hi i'm " + name + " drive!!");  }  
    @Override  
    public void start() {  System.out.println("hi i'm " + name + " start!!");  }  
}

public class Moskvich implements Car{  
    String name = "moskvich";  
    @Override  
    public void drive() {  System.out.println("hi i'm " + name + " drive!!");  }  
    @Override  
    public void start() {  System.out.println("hi i'm " + name + " start!!");  }  
  
}

//некоторое условие 
public enum Country {  RUSSIA, USA  }

//класс отдающий фабричный метод, который сам создаст необходимую реализацию 
public Car getCar(Country country){  
    Car car = null;  
    switch (country) {  
        case USA:  car = new Geep(); break;  
        case RUSSIA: car = new Moskvich();  
    }  
    return car;  
}

// использование фабричного метода
public static void main( String[] args ) {  
    CarSelector carSelector = CarSelector.getInstance();  
    Car car = carSelector.getCar(Country.RUSSIA);  
    car.drive();  
}
```
