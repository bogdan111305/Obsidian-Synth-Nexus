# Создание служб REST


consumes - формат входящих данных 
produces - формат данных в ответе 


### Автоматическое создание доступа к ресурсам

``` Groovy 
implementation 'org.springframework.boot:spring-boot-starter-data-rest'
```

стартер **сам создаёт ссылки на ресурсы** (контроллер) исходя из репозиториев: 

например при репозитории employeeRepository автоматически создадутся стандартные endpoints типа employee/{id} и тд  

мы можем настроить базовый url сами: 
``` Java
@Entity
@RestResource(rel="employees", path="employees")
public class Employee{...}
```

на самом деле это реалезация механизма HATEOAS (Hypermedia as Engine of Application State) 


### Деление на страницы и упорядочивание 

Все ссылки будут поддерживать параметры *page*, *size* и *sort*. 
По умолчанию на странице 20 элементов. Однако  с помощью выше перечисленных параметров можно настроить так как нужно.

Примеры: 
`localhost:8080/data-api/employees?size=5 `  : первая страница 5 работников 
`localhost:8080/data-api/employees?sort=burthday,desc&page=0&size=12` : отсортированные по  дню рождения в порядке убывания

### Использование служб REST

взаимодействие spring с rest api в качестве клиента 

инструмены: 
- RestTemplate - простой клиент rest
- Traverson - обёртка предоставляемая HAMOS
- WebClient - реактивный, асинхронный клиент spring


#### Get 

*getForObject* - сам сконвретирует приходящий json в наш объект 

``` Java
String utl = "https://jsonplaceholder.typicode.com/posts"
Post post = restTemplate.getForObject(url+"/{id}", Post.class, id);  
post.setTitle("Соси бибу");  
```

этот метод перегружен, и позволяет различными способами вставлять параметры и URI

Например через Map: 
``` Java
Map<String, String> parameters = new HashMap<>();  
parameters.put("id", String.valueOf(id));  
Post post = restTemplate.getForObject(url+"/{id}", Post.class, parameters);
```

Или использовать URI параметр: 

``` Java
Map<String, String> parameters = new HashMap<>();  
parameters.put("id", String.valueOf(id));  
  
URI uri = UriComponentsBuilder.fromHttpUrl(url + "/{id}")  
        .build(parameters);  
  
Post post = restTemplate.getForObject(uri, Post.class);
```

*getForEntity* - позволяет получить не только объект но и весь ResponseEntity

``` Java
ResponseEntity<Post> response = restTemplate.getForEntity(uri, Post.class);  
HttpHeaders headers = response.getHeaders();  
Post post = response.getBody();
```

#### PUT
передача uri в нём точно такая же как и в get
``` Java
public void updateIngredient(Ingredient ingredient) { 
	 rest.put("http://localhost:8080/ingredients/{id}", ingredient, ingredient.getId()); 
}
```

#### DELETE
``` Java
public void deleteIngredient(Ingredient ingredient) { 
	rest.delete("http://localhost:8080/ingredients/{id}", ingredient.getId()); 
}
```

#### POST
``` Java
public Ingredient createIngredient(Ingredient ingredient) { 
	return rest.postForObject("http://localhost:8080/ingredients", ingredient, Ingredient.class); 
}
```

*rest.postForLocation* - после запроса получить местоположение нового ресурса - но в таком случае не вернётся объект вернётся только строка.

*rest.postForEntity *- после запроса вернёт всю необходимую информацию
