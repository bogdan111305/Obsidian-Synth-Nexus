# Введение

предоставляет автоконфигурации spring модулей

``` Groovy
org.springframework.boot:spring-boot-autoconfigure
```


### Динамическое подключение модулей

Иногда может быть важно, чтобы некоторые модули подключались только в зависимости от некоего условия (condition) : 
- Classpath - если в classpath лежит например PostgresDriver, должна автоматически включиться JPA конфигурация 
- Properties - если есть какая-то property
- Bean - если некоторый бин создаю я сам и не хочу его импортить 
- SpEL - задавать другие сложные условия
- ...

такой механизм реализован через обычную аннотацию **@Conditional**

### Spring Boot Starters

это обычные зависимости (jars), которые добавляют нужный функционал в проект и следуют определённому шаблону наименования: **spring-boot-starter-\***

все они относятся к группе **org.springframework.boot**

\*неофициальные стартеры не обязательно следуют шаблону, например:
*org.springframework.kafka:spring-kafka*
