# Работа со строками в Java

> [!QUOTE] Суть
> `String` — иммутабельный, хранится в String Pool (heap). `"a" == "a"` — true (пул), `new String("a") == new String("a")` — false! `StringBuilder` — мутабельный, не thread-safe, быстрый. `StringBuffer` — thread-safe, медленнее. Java 9+: compact strings (`byte[]` вместо `char[]`).

> [!WARNING] Ловушка: сравнение строк через ==
> `==` сравнивает **ссылки**, а не содержимое. `"hello" == new String("hello")` → `false`. Всегда используй `.equals()` для сравнения содержимого строк. Исключение: интернированные строки (`intern()`), но это оптимизация, а не рабочая практика.

## 1. Класс `String`

Класс `String` в пакете `java.lang` представляет неизменяемые (immutable) строки символов в кодировке UTF-16. Неизменяемость гарантирует потокобезопасность и возможность использования пула строк для оптимизации памяти.
### Особенности `String`

- **Неизменяемость**: После создания объект `String` нельзя изменить; операции создают новый объект.
- **Пул строк**: Строковые литералы и строки, возвращенные через `intern()`, хранятся в пуле строк в куче (heap), что позволяет повторно использовать одинаковые строки.
- **Хранение**: В Java 8 данные хранятся в `char[]` (2 байта на символ), в Java 9+ — в `byte[]` с компактным представлением (Latin-1 или UTF-16) для экономии памяти.
- **Потокобезопасность**: Неизменяемость делает `String` безопасным для многопоточных приложений.
### Создание объектов `String`

```java
String literal = "Привет"; // В пуле строк
String newString = new String("Привет"); // Новый объект в куче
String interned = newString.intern(); // Ссылка на пул строк
System.out.println(literal == interned); // true
```

**Заметка**: Использование `new String("text")` создает лишний объект в куче. Предпочитайте строковые литералы или `intern()`.
### Основные методы `String`

|Метод|Описание|Пример|
|---|---|---|
|`length()`|Возвращает длину строки.|`"Привет".length()` → 6|
|`charAt(int index)`|Возвращает символ по индексу.|`"Привет".charAt(0)` → 'П'|
|`substring(int begin, int end)`|Извлекает подстроку.|`"Привет".substring(1, 3)` → "ри"|
|`toLowerCase()`, `toUpperCase()`|Преобразует регистр.|`"Привет".toUpperCase()` → "ПРИВЕТ"|
|`trim()`|Удаляет пробелы в начале и конце.|`" Привет ".trim()` → "Привет"|
|`replace(char old, char new)`|Заменяет символы.|`"Привет".replace('П', 'Б')` → "Бривет"|
|`replaceAll(String regex, String replacement)`|Заменяет по регулярному выражению.|`"a1b2".replaceAll("\\d", "_")` → "a_b_"|
|`split(String regex)`|Разделяет строку по регулярному выражению.|`"a,b,c".split(",")` → `["a", "b", "c"]`|
|`startsWith(String prefix)`, `endsWith(String suffix)`|Проверяет начало/конец строки.|`"Привет".startsWith("Пр")` → `true`|
|`contains(CharSequence s)`|Проверяет наличие подстроки.|`"Привет".contains("ри")` → `true`|
|`indexOf(String str)`, `lastIndexOf(String str)`|Возвращает индекс первого/последнего вхождения.|`"Привет".indexOf("р")` → 1|
|`isEmpty()`, `isBlank()`|Проверяет, пуста ли строка или содержит только пробелы (Java 11+).|`" ".isBlank()` → `true`|
|`concat(String str)`|Конкатенирует строки.|`"При".concat("вет")` → "Привет"|
|`equals(Object obj)`, `equalsIgnoreCase(String str)`|Сравнивает строки.|`"Привет".equals("привет")` → `false`|
|`compareTo(String str)`, `compareToIgnoreCase(String str)`|Сравнивает лексикографически.|`"а".compareTo("б")` → -1|
|`intern()`|Возвращает строку из пула строк.|`new String("Привет").intern() == "Привет"` → `true`|
|`join(CharSequence delimiter, CharSequence... elements)`|Объединяет строки с разделителем (Java 8+).|`String.join(",", "a", "b")` → "a,b"|
|`strip()`, `stripLeading()`, `stripTrailing()`|Удаляет пробелы (Unicode-aware, Java 11+).|`" Привет ".strip()` → "Привет"|
|`repeat(int count)`|Повторяет строку (Java 11+).|`"А".repeat(3)` → "ААА"|
|`lines()`|Возвращает Stream строк, разделенных переносами (Java 11+).|`"a\nb".lines().count()` → 2|
### Внутренняя реализация

- **Java 8**: Данные хранятся в `char[]` (UTF-16, 2 байта на символ).
- **Java 9+**: Используется `byte[]` с флагом кодировки (`LATIN1` или `UTF16`). Если строка содержит только символы Latin-1, она занимает 1 байт на символ.
- **Неизменяемость**: Внутренний массив помечен как `final`, а методы, изменяющие строку, возвращают новый объект.
### Пул строк

- **Литерал** (`"text"`) → помещается в String Pool; одинаковые литералы ссылаются на один объект.
- **`new String("text")`** → всегда создаёт новый объект в heap, минуя пул.
- **`intern()`** → возвращает ссылку из пула (добавляет туда строку, если её нет); позволяет сравнивать через `==`.

```java
String s1 = "Привет";
String s2 = new String("Привет").intern();
System.out.println(s1 == s2); // true
```
### Производительность

- **Плюсы**: Неизменяемость обеспечивает потокобезопасность и оптимизацию памяти через пул строк.
- **Минусы**: Конкатенация с помощью `+` или `concat()` создает множество объектов, снижая производительность.

```java
String result = "";
for (int i = 0; i < 1000; i++) {
    result += i; // Плохо: создает 1000 объектов String
}
```

**Решение**: Используйте `StringBuilder` или другие альтернативы для конкатенации.
### Быстрое сравнение

| | `String` | `StringBuilder` | `StringBuffer` |
|--|--|--|--|
| Мутабельность | ❌ | ✅ | ✅ |
| Thread-safe | ✅ | ❌ | ✅ |
| Производительность | базовая | быстро | медленнее |
| Использование | константы | построение строк | многопоточность |

## 2. Класс `StringBuilder`

`StringBuilder` — это изменяемый класс для построения строк, оптимизированный для однопоточных приложений. Он предназначен для эффективной конкатенации и модификации строк.
### Особенности `StringBuilder`

- **Изменяемость**: Внутренний массив (`byte[]` в Java 9+, `char[]` в Java 8) можно изменять без создания новых объектов.
- **Непотокобезопасность**: Не синхронизирован, что делает его быстрее, но небезопасным для многопоточных приложений.
- **Производительность**: Значительно быстрее `String` при многократных модификациях.
### Основные методы `StringBuilder`

|Метод|Описание|Пример|
|---|---|---|
|`append(T value)`|Добавляет значение (поддерживает примитивы, строки, объекты).|`new StringBuilder().append("А").append(1)` → "А1"|
|`insert(int offset, T value)`|Вставляет значение по индексу.|`new StringBuilder("АБ").insert(1, "В")` → "АВБ"|
|`delete(int start, int end)`|Удаляет подстроку.|`new StringBuilder("АБВ").delete(1, 2)` → "АВ"|
|`replace(int start, int end, String str)`|Заменяет подстроку.|`new StringBuilder("АБВ").replace(1, 2, "Г")` → "АГВ"|
|`reverse()`|Переворачивает строку.|`new StringBuilder("АБВ").reverse()` → "ВБА"|
|`toString()`|Преобразует в `String`.|`new StringBuilder("Привет").toString()` → "Привет"|
|`capacity()`|Возвращает текущую емкость внутреннего массива.|`new StringBuilder("А").capacity()` → 16 (по умолчанию)|
|`ensureCapacity(int min)`|Увеличивает емкость, если нужно.|`sb.ensureCapacity(50)`|
|`setLength(int newLength)`|Устанавливает новую длину строки.|`sb.setLength(2)`|
### Пример

```java
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i); // Эффективно: один объект
}
System.out.println(sb.toString()); // 0123...999
```
## 3. Класс `StringBuffer`

`StringBuffer` — это потокобезопасный аналог `StringBuilder`, предназначенный для многопоточных приложений.
### Особенности `StringBuffer`

- **Изменяемость**: Аналогична `StringBuilder`.
- **Потокобезопасность**: Все методы синхронизированы, что снижает производительность, но обеспечивает безопасность в многопоточных сценариях.
- **Использование**: Рекомендуется только при необходимости встроенной синхронизации.
### Основные методы `StringBuffer`

Идентичны `StringBuilder`, но синхронизированы: `append`, `insert`, `delete`, `replace`, `reverse`, `toString`, `capacity`, `ensureCapacity`, `setLength`.
### Пример

```java
StringBuffer sb = new StringBuffer();
ExecutorService executor = Executors.newFixedThreadPool(2);
for (int i = 0; i < 1000; i++) {
    executor.submit(() -> sb.append("A"));
}
executor.shutdown();
executor.awaitTermination(10, TimeUnit.SECONDS);
System.out.println(sb.length()); // 1000 (потокобезопасно)
```
## 4. Класс `StringJoiner` (Java 8+)

`StringJoiner` — это утилитный класс, введенный в Java 8, предназначенный для объединения строк с разделителем, с возможностью добавления префикса и суффикса. Он упрощает создание форматированных строк, таких как CSV или списки.
### Особенности `StringJoiner`

- **Цель**: Объединение строк с разделителями.
- **Внутренняя реализация**: Использует `StringBuilder` для построения строки, что обеспечивает высокую производительность.
- **Потокобезопасность**: Не потокобезопасен, так как основан на `StringBuilder`.
- **Гибкость**: Поддерживает префиксы, суффиксы и значение для пустого состояния.
### Основные методы `StringJoiner`

|Метод|Описание|Пример|
|---|---|---|
|`add(CharSequence)`|Добавляет строку.|`joiner.add("А")` → добавляет "А"|
|`toString()`|Возвращает итоговую строку.|`joiner.toString()` → "[А,Б]"|
|`setEmptyValue(CharSequence)`|Устанавливает значение для пустого `StringJoiner`.|`joiner.setEmptyValue("пусто")`|
|`merge(StringJoiner)`|Объединяет другой `StringJoiner`.|`joiner.merge(anotherJoiner)`|
|`length()`|Возвращает длину текущей строки.|`joiner.length()` → длина результата|
### Пример

```java
StringJoiner joiner = new StringJoiner(",", "[", "]");
joiner.add("А").add("Б").add("В");
System.out.println(joiner.toString()); // [А,Б,В]

StringJoiner emptyJoiner = new StringJoiner(",");
emptyJoiner.setEmptyValue("пусто");
System.out.println(emptyJoiner.toString()); // пусто

StringJoiner other = new StringJoiner(",");
other.add("Г").add("Д");
joiner.merge(other);
System.out.println(joiner.toString()); // [А,Б,В,Г,Д]
```
### Внутренняя реализация

- Использует `StringBuilder` для хранения данных.
- Автоматически добавляет разделитель между элементами, исключая первый.
- Префикс и суффикс добавляются при вызове `toString()`.
### Производительность

- Сопоставима с `StringBuilder`, так как использует его внутренне.
- Дополнительные накладные расходы минимальны и связаны с управлением префиксом/суффиксом.
### Когда использовать

- Для создания строк с разделителями (например, CSV, списки).
- Когда требуется автоматическое добавление префикса/суффикса.
- Для упрощения кода при работе с последовательностями строк.
### Преимущества

- Читаемый и лаконичный API.
- Упрощает форматирование списков.
- Эффективен для задач объединения строк.
### Недостатки

- Ограничен задачами объединения строк.
- Не подходит для сложных манипуляций (например, удаление или замена).

## 5. `String.format` и `String.formatted` (Java 15+)

`String.format` и `String.formatted` — это методы для форматирования строк с использованием шаблонов и спецификаторов формата (например, `%s`, `%d`).
### Особенности

- **Цель**: Создание форматированных строк с переменными.
- **Внутренняя реализация**: Использует `StringBuilder` для построения результата.
- **Потокобезопасность**: Потокобезопасны, так как создают новый объект для каждого вызова.
- **Синтаксис**: Поддерживает спецификаторы формата, аналогичные C-style `printf`.
### Основные спецификаторы

|Спецификатор|Описание|Пример|
|---|---|---|
|`%s`|Строка|`String.format("Имя: %s", "Алексей")` → "Имя: Алексей"|
|`%d`|Целое число|`String.format("Возраст: %d", 30)` → "Возраст: 30"|
|`%f`|Число с плавающей точкой|`String.format("Цена: %.2f", 12.345)` → "Цена: 12.35"|
|`%n`|Перенос строки|`String.format("А%nБ")` → "А\nБ"|
|`%x`|Шестнадцатеричное число|`String.format("Число: %x", 255)` → "Число: ff"|
### `String.format`

- Форматирует строку на основе шаблона и аргументов.

```java
String name = "Алексей";
int age = 30;
String result = String.format("Имя: %s, Возраст: %d", name, age);
System.out.println(result); // Имя: Алексей, Возраст: 30
```
### `String.formatted` (Java 15+)

- Упрощает форматирование, позволяя вызывать метод непосредственно на строковом литерале.

```java
String result = "Имя: %s, Возраст: %d".formatted(name, age);
System.out.println(result); // Имя: Алексей, Возраст: 30
```
### Пример с форматированием

```java
double price = 123.45678;
String formatted = String.format("Цена: %10.2f рублей", price);
System.out.println(formatted); // Цена:     123.46 рублей

String formatted2 = "Цена: %10.2f рублей".formatted(price);
System.out.println(formatted2); // Цена:     123.46 рублей
```
### Производительность

- Накладные расходы связаны с парсингом шаблона и преобразованием аргументов.
- Менее производителен, чем `StringBuilder`, но более читаем для форматированных строк.
### Когда использовать

- Для создания читаемых форматированных строк (например, логов, отчетов).
- Когда требуется точное форматирование чисел, дат или строк.
### Преимущества

- Читаемый и декларативный синтаксис.
- Поддержка сложного форматирования (например, выравнивание, точность).
- `String.formatted` упрощает код в Java 15+.
### Недостатки

- Меньшая производительность при большом количестве операций.
- Ограничен задачами форматирования.
## 6. Текстовые блоки (Java 15+)

Текстовые блоки (text blocks) — это синтаксическая особенность Java, введенная в Java 15 (стабилизирована после предварительных версий в Java 13 и 14), для упрощения работы с многострочными строками.
### Особенности

- **Синтаксис**: Используется тройная кавычка `"""` для определения многострочных строк.
- **Неизменяемость**: Текстовые блоки — это объекты `String`, поэтому они неизменяемы.
- **Обработка пробелов**: Автоматически удаляет начальные пробелы (на основе минимального отступа) и поддерживает экранирование.
- **Потокобезопасность**: Как и `String`, безопасны для многопоточных приложений.
### Пример

```java
String html = """
    <html>
        <body>
            <p>Привет, мир!</p>
        </body>
    </html>
""";
System.out.println(html);
// Вывод:
// <html>
//     <body>
//         <p>Привет, мир!</p>
//     </body>
// </html>

String csv = """
    Имя,Возраст
    Алексей,30
    Мария,25
""";
System.out.println(csv);
// Вывод:
// Имя,Возраст
// Алексей,30
// Мария,25
```
### Экранирование и форматирование

- Поддерживает экранирование (например, `\n`, `\t`).
- Может использоваться с `String.formatted` для интерполяции.

```java
String name = "Алексей";
String block = """
    Пользователь: %s
    Статус: активен
""".formatted(name);
System.out.println(block);
// Вывод:
// Пользователь: Алексей
// Статус: активен
```
### Производительность

- Текстовые блоки — это `String`, поэтому они не подходят для динамической модификации.
- Эффективны для статических многострочных строк, но менее гибки, чем `StringBuilder`.
### Когда использовать

- Для статических многострочных строк (например, HTML, JSON, SQL-запросов).
- Когда важна читаемость и структура текста.
### Преимущества

- Упрощает написание многострочных строк.
- Автоматическая обработка отступов.
- Интеграция с `String.formatted` для динамического форматирования.
### Недостатки

- Неизменяемость ограничивает использование для динамических модификаций.
- Требует Java 15+.
## 7. Stream API и `Collectors.joining` (Java 8+)

`Stream API`, введенная в Java 8, в сочетании с коллектором `Collectors.joining` предоставляет функциональный способ объединения строк из потоков данных.
### Особенности

- **Цель**: Объединение строк из коллекций или потоков с разделителем, префиксом и суффиксом.
- **Внутренняя реализация**: Использует `StringBuilder` для построения результата.
- **Потокобезопасность**: Зависит от конфигурации потока (параллельные потоки требуют осторожности).
- **Интеграция**: Легко комбинируется с другими операциями Stream API (фильтрация, маппинг).
### Основные варианты `Collectors.joining`

|Метод|Описание|Пример|
|---|---|---|
|`joining()`|Объединяет без разделителя.|`stream.collect(Collectors.joining())` → "АБВ"|
|`joining(CharSequence delimiter)`|Объединяет с разделителем.|`stream.collect(Collectors.joining(","))` → "А,Б,В"|
|`joining(CharSequence delimiter, CharSequence prefix, CharSequence suffix)`|Объединяет с разделителем, префиксом и суффиксом.|`stream.collect(Collectors.joining(",", "[", "]"))` → "[А,Б,В]"|
### Пример

```java
List<String> words = Arrays.asList("Привет", "мир", "!");
String result = words.stream()
    .collect(Collectors.joining(" ", "<", ">"));
System.out.println(result); // <Привет мир !>

String filtered = words.stream()
    .filter(s -> s.length() > 3)
    .collect(Collectors.joining(","));
System.out.println(filtered); // Привет
```
### Производительность

- Сопоставима с `StringJoiner`, так как использует `StringBuilder` внутренне.
- Накладные расходы на создание потока и его обработку могут быть заметны при малом количестве данных.
### Когда использовать

- Для объединения строк из коллекций или результатов обработки Stream API.
- Когда требуется комбинировать объединение с фильтрацией, маппингом или другими операциями.
### Преимущества

- Декларативный и функциональный стиль.
- Легкая интеграция с Stream API.
- Гибкость в обработке данных.
### Недостатки

- Накладные расходы на создание потока.
- Меньшая гибкость для сложных манипуляций со строками.
## 8. Регулярные выражения

Регулярные выражения (regex) используются для поиска, анализа и модификации строк на основе шаблонов. Они интегрированы в `String` через методы `replaceAll`, `replaceFirst`, `split`, `matches`, а также поддерживаются классами `Pattern` и `Matcher` из пакета `java.util.regex`.
### Основные методы `String` для регулярных выражений

- `replaceAll(String regex, String replacement)`: Заменяет все вхождения.
- `replaceFirst(String regex, String replacement)`: Заменяет первое вхождение.
- `split(String regex)`: Разделяет строку по шаблону.
- `matches(String regex)`: Проверяет, соответствует ли вся строка шаблону.
### Классы `Pattern` и `Matcher`

- **`Pattern`**: Компилирует регулярное выражение в объект для многократного использования.
- **`Matcher`**: Выполняет операции поиска и сопоставления.
### Пример

```java
String text = "Контакт: +7-123-456-7890, другой: +7-987-654-3210";
Pattern pattern = Pattern.compile("\\+\\d-\\d{3}-\\d{3}-\\d{4}");
Matcher matcher = pattern.matcher(text);
while (matcher.find()) {
    System.out.println(matcher.group()); // +7-123-456-7890, +7-987-654-3210
}

String replaced = text.replaceAll("\\d{3}-\\d{4}", "XXX-XXXX");
System.out.println(replaced); // Контакт: +7-XXX-XXXX, другой: +7-XXX-XXXX
```
### Производительность

- **Кэширование `Pattern`**: Компиляция регулярных выражений дорогая, поэтому кэшируйте `Pattern` в статическом поле.
- **Простые операции**: Для тривиальных замен используйте методы `String` без regex.

```java
static final Pattern NUMBER_PATTERN = Pattern.compile("\\d+");
// Хорошо: кэшированный Pattern
for (String s : strings) {
    NUMBER_PATTERN.matcher(s).replaceAll("_");
}
```
## 9. Сравнение инструментов

|Инструмент|Изменяемость|Потокобезопасность|Производительность|Основное использование|
|---|---|---|---|---|
|`String`|Нет|Да|Низкая при конкатенации|Постоянные строки, пул строк|
|`StringBuilder`|Да|Нет|Высокая|Однопоточная конкатенация|
|`StringBuffer`|Да|Да|Средняя|Многопоточная конкатенация|
|`StringJoiner`|Да|Нет|Высокая|Объединение с разделителями|
|`String.format`/`formatted`|Нет|Да|Средняя|Форматированные строки|
|Text Blocks|Нет|Да|Низкая при модификациях|Многострочные шаблоны|
|`Collectors.joining`|Да|Зависит от потока|Средняя|Объединение строк в Stream API|
## 10. Подводные камни и лучшие практики
### Подводные камни

1. **Конкатенация с `String`**:
    - Частое использование `+` создает множество объектов. Используйте `StringBuilder` или `StringJoiner`.
2. **Потокобезопасность**:
    - `StringBuilder`, `StringJoiner` и `Collectors.joining` (в параллельных потоках) не потокобезопасны. Используйте `StringBuffer` или синхронизацию.
3. **Производительность `String.format`**:
    - Парсинг шаблона снижает производительность при большом количестве операций.
4. **Текстовые блоки**:
    - Не подходят для динамических модификаций из-за неизменяемости.
5. **Регулярные выражения**:
    - Жадные квантификаторы (`*`, `+`) могут вызвать "зависание" на больших строках.
    - **Решение**: Используйте не жадные квантификаторы (`*?`, `+?`) и тестируйте шаблоны.
### Лучшие практики

1. **Выбирайте подходящий инструмент**:
    - `String`: Для неизменяемых строк и пула строк.
    - `StringBuilder`: Для однопоточной конкатенации.
    - `StringBuffer`: Для многопоточной конкатенации.
    - `StringJoiner`/`Collectors.joining`: Для объединения строк с разделителями.
    - `String.format`/`formatted`: Для форматированных строк.
    - Text Blocks: Для многострочных шаблонов.
2. **Кэшируйте объекты**:
    - Храните `Pattern` и `StringJoiner` в статических полях для многократного использования.
3. **Минимизируйте создание объектов**:
    - Избегайте `new String("text")` и чрезмерного использования `intern()`.
4. **Проверяйте входные данные**:
    - Перед regex или `String.format` проверяйте строки на `null` или пустоту.
5. **Используйте современные возможности**:
    - В Java 15+ применяйте `String.formatted` и текстовые блоки для улучшенной читаемости.
## 11. Пример: Комбинированное использование

```java
import java.util.Arrays;
import java.util.List;
import java.util.regex.Pattern;
import java.util.stream.Collectors;

public class TextProcessor {
    private static final Pattern NUMBER_PATTERN = Pattern.compile("\\d+");

    public static String processComplexText(List<String> words) {
        // Используем StringJoiner для форматирования списка
        StringJoiner joiner = new StringJoiner(", ", "[", "]");
        words.forEach(joiner::add);

        // Используем Collectors.joining для фильтрации и объединения
        String filtered = words.stream()
            .filter(s -> s.length() > 3)
            .collect(Collectors.joining(";"));

        // Используем текстовый блок и String.formatted
        String template = """
            Исходный список: %s
            Отфильтрованные слова: %s
            Обработанные числа: %s
            """.formatted(
                joiner.toString(),
                filtered.isEmpty() ? "пусто" : filtered,
                NUMBER_PATTERN.matcher(joiner.toString()).replaceAll("#")
            );

        return template;
    }

    public static void main(String[] args) {
        List<String> words = Arrays.asList("Привет", "мир", "123", "Java");
        System.out.println(processComplexText(words));
        // Вывод:
        // Исходный список: [Привет, мир, 123, Java]
        // Отфильтрованные слова: Привет;Java
        // Обработанные числа: [Привет, мир, #, Java]
    }
}
```

## Senior Insights

### String Deduplication (G1 GC)

String Deduplication — механизм G1 GC для устранения дубликатов внутренних `byte[]` массивов у одинаковых строк.

```java
// Включение:
// -XX:+UseStringDeduplication  (требует G1 GC)
// -XX:+PrintStringDeduplicationStatistics  (статистика)

// Как работает:
// 1. GC инспектирует String объекты во время Minor GC
// 2. Вычисляет hash для каждого byte[] массива
// 3. Если находит дублирующиеся массивы — заменяет их ссылкой на один
// 4. Старые дубликаты освобождаются GC

// ВАЖНО: Работает только для byte[] (Java 9+ Compact Strings)
// String объекты остаются разными — только внутренний массив общий!

String s1 = new String("Hello World"); // byte[] #1
String s2 = new String("Hello World"); // byte[] #2
// После G1 String Deduplication:
// s1 и s2 разные объекты, но оба ссылаются на byte[] #1 (byte[] #2 GC-нут)
// s1 == s2 → false (разные String объекты)
// s1.equals(s2) → true (одинаковое содержимое)

// Когда применять:
// - Много String с одинаковым содержимым (логи, имена полей, коды)
// - Heap > 1GB с высокой долей String
// - Типичный выигрыш: 10-25% heap в enterprise приложениях
```

### Compact Strings (Java 9+) — внутренняя оптимизация

```java
// Java 8: String хранит char[] (всегда 2 байта/символ, UTF-16)
// Java 9+: String хранит byte[] с CODER флагом:
//   - LATIN1 (0): 1 байт/символ если все символы ASCII/Latin-1 (U+0000-U+00FF)
//   - UTF16  (1): 2 байта/символ если есть Unicode символы

// Практический эффект:
String ascii = "Hello World";     // LATIN1: 11 байт вместо 22 байт
String unicode = "Привет мир";    // UTF16: 18 байт (кириллица)
String mixed = "Hello Привет";    // UTF16: весь массив UTF16 из-за одного unicode символа

// Проверка CODER через reflection (не для production):
Field coderField = String.class.getDeclaredField("coder");
coderField.setAccessible(true);
byte coder = (byte) coderField.get(ascii);
System.out.println(coder == 0 ? "LATIN1" : "UTF16"); // LATIN1

// Влияние на производительность:
// - substring(), equals(), hashCode() - быстрее для LATIN1 строк
// - String.compareTo() использует SIMD векторизацию для LATIN1
// - Рекомендация: избегать смешивания кириллицы в ASCII-dominated данных
```

### String.intern() — осторожно!

```java
// BAD: intern() для всех строк - убивает производительность
for (String line : largeFile) {
    String interned = line.intern(); // КОНКУРЕНЦИЯ за String pool!
    // String pool — глобальная ConcurrentHashMap (до Java 8: PermGen, теперь Heap)
    // При высоком параллелизме — contention на pool lock!
}

// SENIOR WAY: intern() только для СТАБИЛЬНЫХ, ЧАСТО ПОВТОРЯЮЩИХСЯ строк
// Хорошие кандидаты: имена полей, коды состояний, страны, категории
// Плохие кандидаты: UUID, email, username (каждый уникален)

// Java 21+ альтернатива для кэширования строк:
// StringTable (символьная таблица JVM) — не трогать вручную
// Вместо intern(): использовать explicit HashMap<String, String>
Map<String, String> cache = new HashMap<>();
String intern(String s) { return cache.computeIfAbsent(s, k -> k); }
// Нет глобального lock, контролируемый размер
```

### Senior Interview Q&A

**Q1: Почему `String.hashCode()` кэшируется, и как это влияет на HashMap?**

> `String.hashCode()` вычисляется лениво при первом вызове и кэшируется в поле `int hash` (с Java 8 — дополнительный `boolean hashIsZero` для строк с hash=0). Это означает: первый `hashCode()` — O(n) по длине строки, последующие — O(1). HashMap/HashSet вызывают `hashCode()` при каждом `put()`/`get()`. Для горячих String ключей (заголовки HTTP, имена полей) кэш значительно ускоряет lookup. Подводный камень: после `String.intern()` hash кэш НЕ передаётся интернированной строке — она пересчитает при первом обращении.

**Q2: Как String Deduplication отличается от String Pool (`intern()`)?**

> `String.intern()`: программист явно просит JVM вернуть строку из String Pool (глобальный set). Два разных `String` объекта с одинаковым содержимым → один объект с одним `byte[]`. String Deduplication (G1): GC автоматически находит разные `String` объекты с одинаковым `byte[]` и заменяет дублирующиеся массивы одним. Объекты остаются разными, только внутренний массив общий. Deduplication не требует изменений в коде, работает в фоне во время GC, но: (1) работает только с G1; (2) не уменьшает число String объектов (только массивы); (3) добавляет небольшой overhead GC (~5-10%).

**Q3: Почему конкатенация строк в Java 9+ быстрее чем в Java 8?**

> Java 8: `"Hello " + name + "!"` компилируется в `new StringBuilder().append("Hello ").append(name).append("!").toString()`. Проблема: StringBuilder не знает финальный размер → возможен resize с копированием. Java 9+: `invokedynamic` + `StringConcatFactory.makeConcatWithConstants()`. Bootstrap метод анализирует шаблон конкатенации и генерирует оптимальный код: (1) вычисляет необходимый размер массива заранее; (2) аллоцирует `byte[]` точного размера; (3) копирует части напрямую без промежуточного StringBuilder. Для Latin-1 строк дополнительно использует SIMD. Benchmark: 15-30% быстрее для типичных конкатенаций.
