## Оператор `if-else`

Оператор `if-else` используется для выполнения условной логики, выбирая блок кода на основе истинности булевого выражения.

### Синтаксис

```java
if (условие) {
    // Выполняется, если условие true
} else if (другое_условие) {
    // Выполняется, если другое_условие true
} else {
    // Выполняется, если все условия false
}
```

- **Условие**: Должно быть булевым выражением (`true` или `false`).
- **Блоки**: `{}` обязательны для многострочных блоков, но могут быть опущены для одной строки.
- **Необязательные части**: `else` и `else if` могут отсутствовать.

```java
int number = 10;
if (number > 0) {
    System.out.println("Положительное число");
} else if (number == 0) {
    System.out.println("Ноль");
} else {
    System.out.println("Отрицательное число");
}
```

**Вывод**:

```
Положительное число
```
### Вложенные `if`

Вложенные `if` позволяют проверять дополнительные условия внутри блоков.

```java
int age = 20;
boolean hasLicense = true;
if (age >= 18) {
    if (hasLicense) {
        System.out.println("Можно водить");
    } else {
        System.out.println("Нужна лицензия");
    }
} else {
    System.out.println("Слишком молод");
}
```

**Заметка**: Глубокая вложенность ухудшает читаемость. Используйте логические операторы или методы для упрощения.
### Логические операторы

Операторы `&&`, `||`, `!` позволяют комбинировать условия:

- `&&` (И): `true`, если оба условия истинны.
- `||` (ИЛИ): `true`, если хотя бы одно условие истинно.
- `!` (НЕ): Инвертирует значение.

```java
int score = 85;
if (score >= 60 && score < 90) {
    System.out.println("Хорошо");
}
```
### Тернарный оператор

Тернарный оператор `?:` — это компактная альтернатива простому `if-else` для присваивания.

**Синтаксис**:

```java
переменная = (условие) ? значение_если_true : значение_если_false;
```

```java
int a = 10, b = 20;
int max = (a > b) ? a : b; // max = 20
```

**Заметка**: Используйте тернарный оператор только для простых выражений, чтобы сохранить читаемость.

## Оператор `switch`

Оператор `switch` выбирает блок кода на основе значения одного выражения. Это альтернатива цепочкам `if-else if`, особенно удобная для проверки на равенство.
### Классический `switch` (Java 8 и раньше)

**Синтаксис**:

```java
switch (выражение) {
    case значение1:
        // Код
        break;
    case значение2:
        // Код
        break;
    default:
        // Код по умолчанию
}
```

- **Выражение**: Должно возвращать совместимый тип (см. ниже).
- **break**: Прерывает выполнение, чтобы избежать "проваливания" (fall-through).
- **default**: Выполняется, если ни один `case` не совпал (необязательно).

```java
int day = 3;
switch (day) {
    case 1:
        System.out.println("Понедельник");
        break;
    case 2:
        System.out.println("Вторник");
        break;
    case 3:
        System.out.println("Среда");
        break;
    default:
        System.out.println("Выходной");
}
```

**Вывод**:

```
Среда
```

**Проваливание (fall-through)**:  

Без `break` выполнение продолжается в следующем `case`.

```java
int day = 1;
switch (day) {
    case 1:
        System.out.println("Будний день");
        // Без break — продолжает выполнение
    case 2:
        System.out.println("Рабочий день");
        break;
}
```

**Вывод**:

```
Будний день
Рабочий день
```
### Поддерживаемые типы (Java 8)

Классический `switch` поддерживает:

- Примитивы: `byte`, `short`, `int`, `char`.
- Обертки: `Byte`, `Short`, `Integer`, `Character` (с автобоксингом).
- `enum`.
- `String` (с Java 7).

**Пример с `String`**:

```java
String role = "admin";
switch (role) {
    case "admin":
        System.out.println("Права администратора");
        break;
    case "user":
        System.out.println("Обычный пользователь");
        break;
    default:
        System.out.println("Неизвестная роль");
}
```
### Современный `switch` (Java 12+)

#### `switch` как выражение (Java 14+)

`switch` может возвращать значение, используя стрелочный синтаксис (`->`) или `yield`.

**Синтаксис**:

```java
переменная = switch (выражение) {
    case значение1 -> выражение1;
    case значение2 -> выражение2;
    default -> выражение_по_умолчанию;
};
```

**Пример**:

```java
int day = 3;
String result = switch (day) {
    case 1 -> "Понедельник";
    case 2 -> "Вторник";
    case 3 -> "Среда";
    default -> "Неизвестно";
};
System.out.println(result); // Среда
```

#### Множественные метки

Можно указать несколько значений в одном `case`.

```java
String result = switch (day) {
    case 1, 2, 3, 4, 5 -> "Будний день";
    case 6, 7 -> "Выходной";
    default -> "Неизвестно";
};
```

#### Использование `yield`

Для сложных блоков в `switch`-выражении используется `yield`.

```java
String result = switch (day) {
    case 1, 2, 3, 4, 5 -> {
        String type = "Будний день";
        yield type;
    }
    case 6, 7 -> "Выходной";
    default -> {
        yield "Неизвестно";
    }
};
```
### Pattern Matching в `switch` (Java 17+, JEP 441)

Java 17+ вводит экспериментальную поддержку шаблонов (`pattern matching`) в `switch`, что позволяет проверять типы и извлекать данные.

```java
Object obj = "Hello";
String result = switch (obj) {
    case String s -> "Строка: " + s;
    case Integer i -> "Число: " + i;
    default -> "Неизвестный тип";
};
System.out.println(result); // Строка: Hello
```

**Заметка**: Pattern matching требует включения флага `--enable-preview` до Java 21.

## Внутренняя реализация в JVM

- **if-else**:
    
    - Компилируется в условные переходы (`if_icmpeq`, `if_icmpgt`, etc.) в байт-коде.
    - Условие вычисляется в стеке, затем JVM выполняет переход к нужной инструкции.
    - Производительность зависит от сложности условия и глубины вложенности.
- **switch**:
    
    - **Классический `switch`**: Компилируется в `tableswitch` (для последовательных значений) или `lookupswitch` (для нерегулярных значений).
    - **Tableswitch**: Быстрее, использует таблицу переходов (O(1)).
    - **Lookupswitch**: Медленнее, использует поиск (O(log n)).
    - **Современный `switch`**:
        - Стрелочный синтаксис устраняет fall-through, упрощая байт-код.
        - `switch`-выражения компилируются в более компактный код с `yield`.
    - **Pattern matching**: Генерирует инструкции для проверки типов (`instanceof`) и извлечения данных.

**Пример байт-кода для `if`**:

```java
int x = 10;
if (x > 5) { /* код */ }
```

Компилируется в:

```
iload_1        // Загрузка x
iconst_5       // Константа 5
if_icmple L1   // Если x <= 5, перейти к L1
; код для true
L1: ; код для false
```
## Производительность

- **if-else**:
    
    - Быстро для простых условий (O(1) для вычисления условия).
    - Деградирует при глубокой вложенности или сложных вычислениях условий.
    - JIT-компилятор может оптимизировать условия, встраивая константы.

- **switch**:
    
    - **Классический `switch`**: Быстрее для множества проверок на равенство благодаря `tableswitch`.
    - **Современный `switch`**: Стрелочный синтаксис и отсутствие fall-through улучшают читаемость и оптимизацию.
    - Pattern matching может быть медленнее из-за проверок типов.

**Рекомендация**:

- Используйте `switch` для множества проверок на равенство.
- Используйте `if-else` для сложных условий или диапазонов.
## Подводные камни

**Пропущенный `break` в `switch`**:

```java
int day = 1;
switch (day) {
	case 1:
		System.out.println("Понедельник");
		// Без break
	case 2:
		System.out.println("Вторник");
}
```

**Вывод**: `Понедельник\nВторник`  
**Решение**: Всегда используйте `break` или стрелочный синтаксис (`->`).

**Неправильное условие в `if`**:

```java
if (x = 5) { // Ошибка: присваивание вместо сравнения
	System.out.println("Всегда true");
}
```

**Решение**: Используйте `==` или сравнивайте с константой слева (`5 == x`).

**Глубокая вложенность `if`**:

- Ухудшает читаемость и производительность.
- **Решение**: Выносите логику в методы или используйте `switch`.

**Null в `switch` с `String`**:

```java
String s = null;
switch (s) { // NullPointerException
	case "test": break;
}
```

**Решение**: Проверяйте на `null` перед `switch`.

**Автобоксинг в `switch`**:

```java
Integer x = 10;
switch (x) { // Автоанбоксинг
case 10: break;
}
```

**Решение**: Учитывайте возможные `NullPointerException`.

## Лучшие практики

**Используйте `if-else` для сложных условий**:

- Подходит для диапазонов, логических выражений и неравенств.

```java
if (score >= 90 && score <= 100) {
	System.out.println("Отлично");
}
```

**Предпочитайте `switch` для проверки на равенство**:

- Особенно с `enum` или множественными значениями.

```java
enum Day { MONDAY, TUESDAY, WEDNESDAY }
Day day = Day.MONDAY;
switch (day) {
	case MONDAY -> System.out.println("Начало недели");
}
```

**Используйте стрелочный `switch` (Java 14+)**:

- Устраняет fall-through и улучшает читаемость.

```java
String result = switch (value) {
	case 1 -> "Один";
	default -> "Другое";
};
```

**Избегайте глубокой вложенности**:

- Выносите сложную логику в методы.

```java
boolean canDrive(int age, boolean hasLicense) {
	return age >= 18 && hasLicense;
}
if (canDrive(age, hasLicense)) {
	System.out.println("Можно водить");
}
```

**Проверяйте на `null` перед `switch` с `String`**:

```java
String s = getRole();
if (s != null) {
	switch (s) {
		case "admin" -> System.out.println("Админ");
	}
}
```

**Используйте `enum` для ограниченных наборов значений**:

```java
enum Role { ADMIN, USER }
Role role = Role.ADMIN;
switch (role) {
	case ADMIN -> System.out.println("Администратор");
	case USER -> System.out.println("Пользователь");
}
```

**Синхронизируйте в многопоточных приложениях**:

```java
synchronized (this) {
	if (sharedValue > 0) {
		sharedValue--;
	}
}
```
## Комплексное использование `if-else` и `switch`

```java
import java.util.concurrent.atomic.AtomicInteger;
import java.util.logging.Logger;

public class ConditionalExample {
    private static final Logger LOGGER = Logger.getLogger(ConditionalExample.class.getName());
    private final AtomicInteger counter = new AtomicInteger(0);

    enum Role { ADMIN, USER, GUEST }

    public String processRole(Role role, Integer score) {
        // if-else для сложной логики
        String accessLevel;
        if (score != null && score >= 90) {
            accessLevel = "Высокий";
        } else if (score != null && score >= 60) {
            accessLevel = "Средний";
        } else {
            accessLevel = "Низкий";
        }

        // switch для обработки роли
        String permissions = switch (role) {
            case ADMIN -> {
                counter.incrementAndGet();
                yield "Полный доступ";
            }
            case USER -> "Ограниченный доступ";
            case GUEST -> "Только просмотр";
        };

        return accessLevel + ": " + permissions;
    }

    public static void main(String[] args) {
        ConditionalExample example = new ConditionalExample();
        Role role = Role.ADMIN;
        Integer score = 95;

        // Многопоточная обработка
        Runnable task = () -> {
            String result = example.processRole(role, score);
            LOGGER.info("Результат: " + result + ", счетчик: " + example.counter.get());
        };
        new Thread(task).start();

        // Пример с null
        score = null;
        String result = example.processRole(Role.GUEST, score);
        LOGGER.info("Гость: " + result);

        // Тернарный оператор
        String status = (example.counter.get() > 0) ? "Активен" : "Неактивен";
        LOGGER.info("Статус: " + status);
    }
}
```

**Предполагаемый вывод**:

```
INFO: Результат: Высокий: Полный доступ, счетчик: 1
INFO: Гость: Низкий: Только просмотр
INFO: Статус: Активен
```
