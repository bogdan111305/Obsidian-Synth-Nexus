
Метод — это именованный блок кода внутри класса, который выполняет определенную задачу, может принимать параметры и возвращать результат. Методы являются частью объектно-ориентированного подхода Java и всегда принадлежат классу или интерфейсу.

### Синтаксис метода

```java
[модификаторы] возвращаемый_тип имяМетода(тип1 параметр1, тип2 параметр2, ...) [throws Исключение] {
    // Тело метода
    return значение; // Если возвращаемый тип не void
}
```

**Пример**:

```java
public int add(int a, int b) {
    return a + b;
}
```

- **Модификаторы**: `public`, `private`, `static`, `final`, etc.
- **Возвращаемый тип**: Примитив (`int`, `double`), объект (`String`, `List`), или `void`.
- **Имя метода**: Следует camelCase (например, `calculateSum`).
- **Параметры**: Могут быть любыми типами, включая примитивы и объекты.
- **Исключения**: Указываются через `throws` для проверяемых исключений.

### Виды методов

1. **Экземплярные методы**:
    
    - Привязаны к объекту класса.
    - Вызываются через экземпляр: `object.method()`.
    - Имеют доступ к полям экземпляра через `this`.
    
    **Пример**:
    
```java
class Calculator {
	int add(int a, int b) {
		return a + b;
	}
}
Calculator calc = new Calculator();
int sum = calc.add(2, 3); // 5
```
    
2. **Статические методы**:
    
    - Принадлежат классу, а не объекту.
    - Вызываются через имя класса: `ClassName.method()`.
    - Не имеют доступа к `this`.
    
    **Пример**:
    
```java
class MathUtils {
	static int multiply(int a, int b) {
		return a * b;
	}
}
int product = MathUtils.multiply(4, 5); // 20
```
    
3. **Абстрактные методы**:
    
    - Объявлены без реализации в абстрактных классах или интерфейсах.
    - Должны быть переопределены в подклассах.
    
    **Пример**:
    
```java
abstract class Shape {
	abstract double area();
}
class Circle extends Shape {
	double radius;
	@Override
	double area() {
		return Math.PI * radius * radius;
	}
}
```
    
4. **Методы по умолчанию (Java 8+)**:
    
    - Имеют реализацию в интерфейсах.
    - Позволяют добавлять функциональность без нарушения существующих реализаций.
    
    **Пример**:
    
```java
interface Logger {
	default void log(String message) {
		System.out.println("Log: " + message);
	}
}
class MyClass implements Logger {
	// log доступен без переопределения
}
```
    
5. **Статические методы в интерфейсах (Java 8+)**:
    
    - Принадлежат интерфейсу, вызываются через имя интерфейса.
    
    **Пример**:
    
```java
interface MathOps {
	static int max(int a, int b) {
		return a > b ? a : b;
	}
}
int max = MathOps.max(10, 20); // 20
```
### Передача параметров

- **По значению**: Java передает параметры по значению, включая ссылки на объекты.
- **Примитивы**: Передается копия значения.
- **Объекты**: Передается копия ссылки, что позволяет изменять содержимое объекта, но не саму ссылку.

```java
class Example {
    void modify(int num, StringBuilder sb) {
        num = 100; // Не влияет на внешнюю переменную
        sb.append("World"); // Изменяет объект
    }
}
StringBuilder sb = new StringBuilder("Hello");
int x = 10;
new Example().modify(x, sb);
System.out.println(x + ", " + sb); // 10, HelloWorld
```

## Перегрузка методов (Overloading)

Перегрузка — это определение нескольких методов с одинаковым именем, но разными сигнатурами (количество, типы или порядок параметров).

### Правила перегрузки

- Разные сигнатуры (параметры).
- Возвращаемый тип **не влияет** на перегрузку.
- Модификаторы доступа не влияют на перегрузку.

```java
class OverloadExample {
    public void print() {
        System.out.println("Без параметров");
    }

    public void print(String message) {
        System.out.println("Строка: " + message);
    }

    public void print(int number) {
        System.out.println("Число: " + number);
    }
}
```

**Вызов**:

```java
OverloadExample ex = new OverloadExample();
ex.print();         // Без параметров
ex.print("Hello");  // Строка: Hello
ex.print(42);       // Число: 42
```

### Автобоксинг и перегрузка

Автобоксинг может привести к неоднозначности.

```java
class Ambiguity {
    void method(int x) { System.out.println("int"); }
    void method(Integer x) { System.out.println("Integer"); }
}
new Ambiguity().method(5); // int (примитив предпочтительнее)
```

## Переопределение методов (Overriding)

Переопределение — это предоставление подклассом собственной реализации метода суперкласса или интерфейса.

### Правила переопределения

1. **Идентичная сигнатура**: Имя и параметры должны совпадать.
2. **Ковариантный возвращаемый тип**: Можно возвращать подтип оригинального типа.
3. **Модификаторы доступа**: Нельзя уменьшать видимость (например, `public` → `private`).
4. **Исключения**: Нельзя бросать новые или более общие проверяемые исключения.
5. **Аннотация `@Override`**: Рекомендуется для проверки корректности.
6. **Ограничения**:
    - `final` методы нельзя переопределять.
    - Статические методы **скрываются** (method hiding), а не переопределяются.

```java
class Animal {
    public void sound() {
        System.out.println("Some sound");
    }
}
class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Bark");
    }
}
Animal dog = new Dog();
dog.sound(); // Bark
```

### 3.2. Ковариантный возвращаемый тип

```java
class Animal {
    Animal getInstance() {
        return this;
    }
}
class Dog extends Animal {
    @Override
    Dog getInstance() { // Ковариантный тип
        return this;
    }
}
```

### 3.3. Скрытие статических методов (Method Hiding)

Статические методы не переопределяются, а скрываются.

```java
class Parent {
    static void greet() {
        System.out.println("Hello from Parent");
    }
}
class Child extends Parent {
    static void greet() {
        System.out.println("Hello from Child");
    }
}
Parent p = new Child();
p.greet(); // Hello from Parent
Child.greet(); // Hello from Child
```

## Лямбда-выражения (Java 8+)

Лямбда-выражения позволяют представлять методы как значения, что делает Java ближе к функциональному программированию.

### Синтаксис

```java
(параметры) -> выражение;
(параметры) -> { блок кода; return значение; };
```

```java
Function<Integer, Integer> square = x -> x * x;
System.out.println(square.apply(5)); // 25
```

### Функциональные интерфейсы

Лямбда-выражения реализуют функциональные интерфейсы (интерфейсы с одним абстрактным методом).

**Примеры интерфейсов**:

- `Function<T, R>`: Принимает `T`, возвращает `R`.
- `Consumer<T>`: Принимает `T`, ничего не возвращает.
- `Predicate<T>`: Принимает `T`, возвращает `boolean`.

```java
Predicate<Integer> isEven = n -> n % 2 == 0;
System.out.println(isEven.test(4)); // true
```

### Ссылки на методы

Ссылки на методы — это компактная альтернатива лямбда-выражениям для вызова существующих методов.

**Синтаксис**:

- `ClassName::staticMethod` (статический метод).
- `instance::instanceMethod` (экземплярный метод).
- `ClassName::instanceMethod` (экземплярный метод для первого аргумента).
- `ClassName::new` (конструктор).

```java
List<String> list = Arrays.asList("a", "b", "c");
list.forEach(System.out::println); // Ссылка на метод println
```

## Внутренняя реализация в JVM

- **Методы**:
    
    - Компилируются в байт-код, хранятся в Metaspace как часть класса.
    - Экземплярные методы используют `invokevirtual` для полиморфизма.
    - Статические методы используют `invokestatic`.
    - JIT-компилятор может встраивать (inline) часто вызываемые методы для оптимизации.
- **Перегрузка**:
    
    - Разрешается на этапе компиляции (static dispatch) на основе сигнатуры.
- **Переопределение**:
    
    - Реализуется через таблицу виртуальных методов (vtable) для полиморфизма (dynamic dispatch).
- **Лямбда-выражения**:
    
    - Компилируются в синтетические методы с использованием `invokedynamic`.
    - JVM создает реализацию функционального интерфейса через `LambdaMetafactory`.

**Пример байт-кода для метода**:

```java
public int add(int a, int b) { return a + b; }
```

Компилируется в:

```
iload_1        // Загрузка a
iload_2        // Загрузка b
iadd           // Сложение
ireturn        // Возврат результата
```

## Производительность

- **Экземплярные методы**:
    - Полиморфизм (через vtable) добавляет небольшие накладные расходы.
- **Статические методы**:
    - Быстрее, так как используют `invokestatic` без полиморфизма.
- **Лямбда-выражения**:
    - Накладные расходы на создание объекта функционального интерфейса.
    - JIT-компилятор оптимизирует частые вызовы.
- **Перегрузка**: Разрешается на этапе компиляции, не влияет на runtime.
- **Переопределение**: Полиморфизм добавляет минимальные затраты.

**Рекомендация**:

- Используйте статические методы для утилитных функций.
- Минимизируйте лямбда-выражения в критических по производительности участках.
## Подводные камни

1. **Перепутывание перегрузки и переопределения**:
    
```java
class Parent {
	void method(int x) {}
}
class Child extends Parent {
	void method(String x) {} // Перегрузка, а не переопределение
}
```
    
    **Решение**: Используйте `@Override` для проверки.
    
2. **Скрытие статических методов**:
    
```java
Parent p = new Child();
p.greet(); // Вызовет Parent.greet()
```
    
    **Решение**: Понимайте, что статические методы привязаны к типу ссылки.
    
3. **Null в лямбда-выражениях**:
    
```java
Function<String, Integer> len = s -> s.length();
len.apply(null); // NullPointerException
```
    
    **Решение**: Проверяйте входные данные.
    
4. **Автобоксинг в перегрузке**:
    
```java
void method(Number n) {}
void method(int n) {}
method(5); // Вызовет int-версию
```
    
    **Решение**: Учитывайте приоритет примитивов.
    
5. **Многопоточность без синхронизации**:
    
```java
int count = 0;
void increment() { count++; } // Гонка данных
```
    
    **Решение**: Используйте `synchronized` или `AtomicInteger`.
## Лучшие практики

1. **Используйте понятные имена методов**:
    
```java
int calculateSum(int a, int b); // Хорошо
int sum(int x, int y); // Плохо
```
    
2. **Применяйте `@Override`**:
    
    - Обеспечивает проверку корректности переопределения.
3. **Минимизируйте сложность методов**:
    
    - Разделяйте большие методы на меньшие.
4. **Используйте статические методы для утилит**:
    
```java
static int max(int a, int b) { return a > b ? a : b; }
```
    
5. **Предпочитайте методы по умолчанию в интерфейсах**:
    
    - Для обратной совместимости.
6. **Используйте лямбда-выражения для краткости**:
    
```java
list.forEach(s -> System.out.println(s));
```
    
7. **Синхронизируйте в многопоточных приложениях**:
    
```java
synchronized void modifySharedData() { /* код */ }
```
    
8. **Проверяйте входные данные в лямбда-выражениях**:
    
```java
Function<String, Integer> len = s -> (s != null) ? s.length() : 0;
```
## Комплексное использование методов и лямбда-выражений

```java
import java.util.Arrays;
import java.util.List;
import java.util.concurrent.atomic.AtomicInteger;
import java.util.function.Consumer;
import java.util.logging.Logger;

interface Processor {
    default void log(String message) {
        System.out.println("Лог: " + message);
    }
    void process(int value);
}

public class MethodExample implements Processor {
    private static final Logger LOGGER = Logger.getLogger(MethodExample.class.getName());
    private final AtomicInteger counter = new AtomicInteger(0);

    // Экземплярный метод
    public int add(int a, int b) {
        return a + b;
    }

    // Статический метод
    public static String format(String input) {
        return input.toUpperCase();
    }

    // Переопределение метода интерфейса
    @Override
    public void process(int value) {
        synchronized (this) {
            counter.addAndGet(value);
            log("Обработано: " + value + ", счетчик: " + counter.get());
        }
    }

    // Перегрузка метода
    public void process(String message) {
        log("Сообщение: " + message);
    }

    public static void main(String[] args) {
        MethodExample example = new MethodExample();
        
        // Вызов экземплярного метода
        int sum = example.add(5, 10);
        LOGGER.info("Сумма: " + sum);

        // Вызов статического метода
        String formatted = MethodExample.format("hello");
        LOGGER.info("Форматировано: " + formatted);

        // Использование лямбда-выражения
        List<Integer> numbers = Arrays.asList(1, 2, 3);
        Consumer<Integer> processor = n -> example.process(n);
        numbers.forEach(processor);

        // Ссылка на метод
        numbers.forEach(System.out::println);

        // Многопоточная обработка
        Runnable task = () -> example.process("Многопоточный вызов");
        new Thread(task).start();
    }
}
```

**Предполагаемый вывод**:

```
INFO: Сумма: 15
INFO: Форматировано: HELLO
Лог: Обработано: 1, счетчик: 1
Лог: Обработано: 2, счетчик: 3
Лог: Обработано: 3, счетчик: 6
1
2
3
Лог: Сообщение: Многопоточный вызов
```
