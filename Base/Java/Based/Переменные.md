Переменная — это именованный контейнер в памяти, который хранит значение определенного типа (примитивного или ссылочного). Каждая переменная имеет:

- **Имя**: Уникальный идентификатор (например, `x`, `counter`).
- **Тип**: Определяет, какие данные могут храниться (например, `int`, `String`).
- **Значение**: Данные, хранимые в переменной.
- **Область видимости**: Где переменная доступна.
- **Время жизни**: Как долго переменная существует.

Переменные в Java строго типизированы, что означает, что их тип фиксируется при объявлении и не может быть изменен.

## Классификация переменных

Переменные в Java классифицируются по **месту объявления**, **типу данных** и **модификаторам**

### По месту объявления

#### Локальные переменные

- **Объявление**: Внутри метода, конструктора или блока кода (например, `{}`).
- **Хранение**: В **стеке** (stack) JVM, в рамках текущего вызова метода.
- **Инициализация**: Не инициализируются по умолчанию, требуют явного присваивания.
- **Время жизни**: Существует только во время выполнения блока.

```java
void method() {
	int x = 10; // Локальная переменная
	System.out.println(x); // 10
} // x уничтожается при выходе из метода
```

#### Поля экземпляра (instance variables)

- **Объявление**: Внутри класса, но вне методов, без ключевого слова `static`.
- **Хранение**: В **куче** (heap), внутри объекта.
- **Инициализация**: Автоматически инициализируются значениями по умолчанию (`0` для чисел, `null` для объектов, `false` для `boolean`).
- **Время жизни**: Существует, пока жив объект (до сборки мусора).

```java
public class MyClass {
	int x; // Поле экземпляра
	public MyClass() {
		System.out.println(x); // 0 (по умолчанию)
		x = 20;
	}
}
```

#### Статические переменные (static variables)

- **Объявление**: Внутри класса с ключевым словом `static`.
- **Хранение**: В **Metaspace** (в Java 8+; ранее — PermGen).
- **Инициализация**: Инициализируются значениями по умолчанию или при загрузке класса.
- **Время жизни**: Существует, пока класс загружен в JVM.

```java
public class MyClass {
	static int counter = 0; // Статическая переменная
	public static void increment() {
		counter++;
	}
}
```
### По типу данных

#### Примитивные типы

- Хранят значение напрямую в памяти.
- Фиксированный размер, независимый от платформы (благодаря стандартизации JVM).
- Типы: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`.

|Тип|Размер|Диапазон|Значение по умолчанию|
|---|---|---|---|
|`byte`|1 байт|-128..127|0|
|`short`|2 байта|-32,768..32,767|0|
|`int`|4 байта|-2^31..2^31-1|0|
|`long`|8 байт|-2^63..2^63-1|0L|
|`float`|4 байта|~±3.4E38|0.0f|
|`double`|8 байт|~±1.7E308|0.0|
|`char`|2 байта|0..65,535 (Unicode)|'\u0000'|
|`boolean`|1 бит (реализация JVM-зависима)|true/false|false|

```java
int number = 42;
char letter = 'A';
boolean flag = true;
```

#### Ссылочные типы

- Хранят ссылку на объект в куче.
- Включают классы (`String`, `List`), интерфейсы, массивы и пользовательские типы.
- Значение по умолчанию: `null`.

```java
String s = "Hello"; // s — ссылка на объект String
List<Integer> list = new ArrayList<>(); // list — ссылка на объект ArrayList
```

### По модификаторам

#### `final`

- Переменная инициализируется один раз и не может быть переназначена.
- Для ссылочных типов фиксирует ссылку, но не содержимое объекта.
- Применяется к локальным переменным, полям экземпляра и статическим полям.

**Пример**:

```java
final int x = 10;
// x = 20; // Ошибка компиляции
final List<String> list = new ArrayList<>();
list.add("Hello"); // Разрешено: изменяется содержимое объекта
// list = new ArrayList<>(); // Ошибка компиляции
```

#### `volatile`

- Обеспечивает видимость изменений переменной между потоками.
- Предотвращает кэширование значения в локальной памяти потока.
- Не гарантирует атомарность операций.

**Пример**:

```java
public class SharedResource {
    volatile boolean flag = false;

    void setFlag() {
        flag = true; // Видно всем потокам
    }
}
```

#### `transient`

- Исключает поле из сериализации (см. `java.io.Serializable`).
- При десериализации поле получает значение по умолчанию.

**Пример**:

```java
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    String name;
    transient String password; // Не сериализуется
}
```

## Хранение и инициализация

### Хранение в памяти JVM

- **Стек (Stack)**: Локальные переменные и ссылки на объекты хранятся в стеке каждого потока. Стек быстрый, но ограничен по размеру.
- **Куча (Heap)**: Объекты (включая поля экземпляра) хранятся в куче, управляемой сборщиком мусора.
- **Metaspace**: Статические переменные и метаданные классов хранятся в Metaspace (в Java 8+ заменил PermGen).

### Инициализация

|Вид переменной|Где хранится|Инициализация по умолчанию|Жизненный цикл|
|---|---|---|---|
|Локальная|Стек|Нет (ошибка, если не инициализирована)|До выхода из метода/блока|
|Поле экземпляра|Куча|Да (0, null, false)|Пока жив объект|
|Статическая|Metaspace|Да (0, null, false)|Пока загружен класс|

```java
public class Example {
    int instanceVar; // 0
    static String staticVar; // null

    void method() {
        int localVar; // Не инициализирована
        // System.out.println(localVar); // Ошибка компиляции
    }
}
```

## Область видимости и жизненный цикл

### Область видимости

- **Локальные переменные**: Видимы только внутри блока, метода или конструктора.
- **Поля экземпляра**: Доступны через `this` внутри класса, если не ограничены модификаторами доступа (`private`, `protected`, `public`).
- **Статические переменные**: Доступны через имя класса (`ClassName.var`) или `this` (в нестатическом контексте).

**Пример**:

```java
public class ScopeExample {
    int instanceVar = 10;
    static int staticVar = 20;

    void method() {
        int localVar = 30;
        System.out.println(localVar); // 30
        System.out.println(this.instanceVar); // 10
        System.out.println(ScopeExample.staticVar); // 20
    }
}
```
### Жизненный цикл

- **Локальные переменные**: Уничтожаются при выходе из блока.
- **Поля экземпляра**: Уничтожаются, когда объект становится недоступным и собирается сборщиком мусора.
- **Статические переменные**: Уничтожаются при выгрузке класса из JVM (редко, обычно при завершении программы).

## Shadowing (Затенение переменных)

**Shadowing** — это ситуация, когда переменная во внутренней области видимости (например, параметр метода или локальная переменная) имеет то же имя, что и переменная во внешней области видимости (например, поле класса), и "затеняет" её, делая недоступной под этим именем.

### Пример с параметром метода

```java
public class Person {
    String name = "Default";

    public void setName(String name) {
        this.name = name; // this ссылается на поле класса
        // name = name; // Присваивает параметру, а не полю
    }

    public static void main(String[] args) {
        Person p = new Person();
        p.setName("Алексей");
        System.out.println(p.name); // Алексей
    }
}
```

### Shadowing в блоках

Java **не допускает** затенение локальных переменных в одной области видимости:

```java
void method() {
    int x = 10;
    {
        int x = 20; // Ошибка компиляции: переменная x уже определена
    }
}
```

Однако затенение допустимо между разными методами или параметрами:

```java
public class ShadowExample {
    int x = 10;

    void method(int x) { // Параметр затеняет поле
        System.out.println(x); // Параметр
        System.out.println(this.x); // Поле
    }
}
```

### Shadowing в циклах

Затенение в одной области видимости не допускается:

```java
void method() {
    int i = 5;
    for (int i = 0; i < 10; i++) { // Ошибка: i уже определена
        System.out.println(i);
    }
}
```

**Решение**: Используйте разные имена или вложенные блоки:

```java
void method() {
    int i = 5;
    {
        for (int j = 0; j < 10; j++) {
            System.out.println(j);
        }
    }
}
```

### Shadowing в лямбда-выражениях

Лямбда-выражения (Java 8+) не могут объявлять переменные, затеняющие внешние переменные:

```java
String msg = "Hello";
Runnable r = () -> {
    // String msg = "Hi"; // Ошибка: переменная msg уже определена
    System.out.println(msg); // OK: использует внешнюю msg
};
```
## Effectively Final

**Effectively final** — это переменная, которая не объявлена как `final`, но фактически не изменяется после инициализации. Этот термин важен для лямбда-выражений, анонимных классов и вложенных классов, где переменные из внешнего контекста должны быть неизменяемыми.
### Пример effectively final

```java
public void printMessage() {
    String msg = "Hello"; // Effectively final
    Runnable r = () -> System.out.println(msg); // OK
    r.run();
}
```
### Нарушение effectively final

```java
public void printMessage() {
    String msg = "Hello";
    msg = "Hi"; // Переприсваивание
    Runnable r = () -> System.out.println(msg); // Ошибка компиляции
}
```

**Пример с анонимным классом**:

```java
public void example() {
    int count = 0; // Effectively final
    Runnable r = new Runnable() {
        @Override
        public void run() {
            System.out.println(count); // OK
        }
    };
    // count = 1; // Ошибка: count перестанет быть effectively final
}
```
## Модификаторы доступа

Переменные могут иметь модификаторы доступа, которые влияют на их видимость:

- **`public`**: Доступна везде.
- **`protected`**: Доступна в пакете и подклассах.
- **`default` (package-private)**: Доступна только в пакете, если модификатор не указан.
- **`private`**: Доступна только внутри класса.

**Пример**:

```java
public class AccessExample {
    public int publicVar = 1;
    protected int protectedVar = 2;
    int defaultVar = 3; // package-private
    private int privateVar = 4;
}
```

**Заметка**: Модификаторы доступа применяются только к полям (экземпляра или статическим), но не к локальным переменным.
## Переменные в многопоточной среде

### Потокобезопасность

- **Локальные переменные**: Потокобезопасны, так как каждый поток имеет свой стек.
- **Поля экземпляра**: Не потокобезопасны, если объект используется несколькими потоками.
- **Статические переменные**: Не потокобезопасны, так как общие для всех экземпляров.

### Использование `volatile`

`volatile` обеспечивает видимость изменений переменной между потоками, но не атомарность.

```java
public class Counter {
    volatile int count = 0;

    void increment() {
        count++; // Не атомарно, может привести к гонке данных
    }
}
```

**Решение для атомарности**: Используйте `synchronized`, `AtomicInteger` или `Lock`.

```java
import java.util.concurrent.atomic.AtomicInteger;

public class SafeCounter {
    private final AtomicInteger count = new AtomicInteger(0);

    void increment() {
        count.incrementAndGet(); // Атомарная операция
    }
}
```
## Внутренняя реализация в JVM

- **Стек**: Локальные переменные и ссылки хранятся в стеке каждого потока. Размер стека ограничен (настраивается через `-Xss`).
- **Куча**: Объекты (включая поля экземпляра) хранятся в куче, разделенной на области (Young Generation, Old Generation).
- **Metaspace**: Статические переменные и метаданные классов хранятся в Metaspace, которая не ограничена PermGen (Java 8+).
- **Сборка мусора**: Поля экземпляра удаляются, когда объект становится недоступным. Статические переменные живут до выгрузки класса.

**Пример в памяти**:

```java
public class Example {
    static String staticField = "Static"; // Metaspace
    String instanceField = "Instance"; // Heap

    void method() {
        int localVar = 42; // Stack
    }
}
```
## Подводные камни

### Неинициализированные локальные переменные:

```java
int x;
// System.out.println(x); // Ошибка компиляции
```
- **Решение**: Всегда инициализируйте локальные переменные.
### Shadowing:

```java
String name = "Default";
void setName(String name) {
	name = "New"; // Изменяет параметр, а не поле
}
```
- **Решение**: Используйте `this` для доступа к полям.
### Злоупотребление `static`:
- Статические переменные увеличивают потребление памяти, так как живут до выгрузки класса.
- **Решение**: Используйте статические переменные только для общих данных.
### Многопоточность:
- Без `volatile` или синхронизации изменения переменной могут быть не видны другим потокам.
- **Решение**: Используйте `volatile`, `synchronized` или атомарные классы.
### Лямбда и effectively final:
- Переменные, изменяемые после захвата в лямбде, вызывают ошибку.
- **Решение**: Убедитесь, что переменные effectively final, или используйте массивы/объекты для обхода.

## Комплексное использование переменных

```java
import java.io.Serializable;
import java.util.concurrent.atomic.AtomicInteger;

public class Counter implements Serializable {
    private static final long serialVersionUID = 1L;
    private static final AtomicInteger instanceCount = new AtomicInteger(0); // Статическая переменная
    private final String id; // Поле экземпляра, неизменяемое
    private transient int tempValue; // Не сериализуется
    private volatile boolean active = true; // Для многопоточности

    public Counter(String id) {
        this.id = id;
        instanceCount.incrementAndGet();
    }

    public void process() {
        int localCounter = 0; // Локальная переменная
        Runnable task = () -> {
            // localCounter = 1; // Ошибка: localCounter должен быть effectively final
            System.out.println("ID: " + id + ", Active: " + active);
        };
        task.run();
        localCounter++; // OK: не влияет на лямбду
    }

    public static int getInstanceCount() {
        return instanceCount.get();
    }

    public static void main(String[] args) {
        Counter c1 = new Counter("C1");
        Counter c2 = new Counter("C2");
        c1.process();
        System.out.println("Количество экземпляров: " + Counter.getInstanceCount());

        // Сериализация
        try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("counter.ser"))) {
            oos.writeObject(c1);
        } catch (IOException e) {
            e.printStackTrace();
        }

        // Десериализация
        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("counter.ser"))) {
            Counter restored = (Counter) ois.readObject();
            System.out.println("Восстановлено: " + restored.id + ", tempValue: " + restored.tempValue);
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```

**Вывод**:

```
ID: C1, Active: true
Количество экземпляров: 2
Восстановлено: C1, tempValue: 0
```
