Абстрактный класс — это класс, который **нельзя инстанцировать** напрямую (нельзя создать объект с помощью `new AbstractClass()`), так как он частично или полностью не реализован. Он обозначается ключевым словом `abstract` и служит базой для других классов через наследование.

**Основные характеристики**:

- Может содержать **абстрактные методы** (без реализации) и **конкретные методы** (с реализацией).
- Может иметь **поля** (переменные состояния), конструкторы и методы.
- Наследники обязаны либо реализовать все абстрактные методы, либо сами быть абстрактными.
- Поддерживает **полиморфизм**, позволяя обрабатывать объекты наследников через тип базового класса.

**Базовый абстрактный класс**:

```java
public abstract class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    // Абстрактный метод
    public abstract void makeSound();

    // Конкретный метод
    public void sleep() {
        System.out.println(name + " спит");
    }
}
```

**Наследник**:

```java
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public void makeSound() {
        System.out.println("Гав!");
    }
}
```

**Использование**:

```java
Animal dog = new Dog("Бобик");
dog.makeSound(); // Выведет: Гав!
dog.sleep();     // Выведет: Бобик спит
```

## 2. Особенности абстрактных классов

Абстрактные классы обладают следующими характеристиками:

**Запрет на инстанцирование**:

- Попытка создать объект (`new Animal()`) вызовет ошибку компиляции.
- Это защищает от создания объектов, которые логически неполны.

**Абстрактные и конкретные методы**:

- Абстрактные методы объявляются без тела (`public abstract void method();`) и требуют реализации в наследниках.
- Конкретные методы содержат реализацию, которую наследники могут использовать или переопределять.

**Поля и конструкторы**:

- Абстрактные классы могут содержать поля (например, `name`) и конструкторы для инициализации.
- Наследники вызывают конструктор базового класса через `super()`.

**Наследование**:

- Класс может наследовать только один абстрактный класс (Java не поддерживает множественное наследование классов).
- Наследник должен реализовать все абстрактные методы или быть объявленным `abstract`.

**Реализация интерфейсов**:

- Абстрактный класс может реализовывать интерфейсы, предоставляя частичную или полную реализацию их методов.

**Полиморфизм**:

- Объекты наследников можно хранить в переменных типа абстрактного класса, что упрощает обработку разных типов.

```java
//Пример с интерфейсом:

public interface Movable {
    void move();
}

public abstract class Animal implements Movable {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public abstract void makeSound();
}
```

## 3. Абстрактные методы

Абстрактные методы — это методы без реализации, объявленные с ключевым словом `abstract`. Они задают контракт, который должны выполнить наследники.

**Синтаксис**:

```java
public abstract void makeSound();
```

**Особенности**:
- Не содержат тела (без `{}`).
- Должны быть реализованы в неабстрактных наследниках.
- Могут быть `public`, `protected` или с пакетом доступа, но не `private`.

```java
//Пример реализации:

public class Cat extends Animal {
	public Cat(String name) {
		super(name);
	}

	@Override
	public void makeSound() {
		System.out.println("Мяу!");
	}

	@Override
	public void move() {
		System.out.println(name + " идет");
	}
}
```
## 4. Зачем нужны абстрактные классы?

Абстрактные классы решают следующие задачи:

1. **Общий шаблон**:
    
    - Задают структуру и поведение для группы классов (например, все животные издают звук, но конкретный звук зависит от вида).
2. **Запрет создания неполных объектов**:
    
    - Логически неполный класс (например, абстрактный `Animal`) не должен быть инстанцирован.
3. **Повторное использование кода**:
    
    - Конкретные методы (например, `sleep()`) устраняют дублирование кода в наследниках.
4. **Полиморфизм**:
    
    - Позволяют работать с объектами разных типов через общий тип:
        
        ```java
        Animal[] animals = {new Dog("Бобик"), new Cat("Мурка")};
        for (Animal animal : animals) {
            animal.makeSound(); // Полиморфный вызов
        }
        ```
        
5. **Частичная реализация**:
    
    - Абстрактный класс может реализовать часть интерфейса или логики, оставляя детали наследникам.

**Сценарий использования**:

- Моделирование иерархии транспортных средств:

```java
public abstract class Vehicle {
	protected String brand;
	protected int speed;

	public Vehicle(String brand, int speed) {
		this.brand = brand;
		this.speed = speed;
	}

	public abstract void startEngine();

	public void stop() {
		System.out.println(brand + " остановился");
	}
}

public class Car extends Vehicle {
	public Car(String brand, int speed) {
		super(brand, speed);
	}

	@Override
	public void startEngine() {
		System.out.println(brand + " завел двигатель");
	}
}
```
## 5. Внутренняя реализация в JVM

- **Абстрактный класс**:
    
    - В байт-коде помечается флагом `ACC_ABSTRACT` в заголовке класса.
    - Нельзя создать объект через `new` (JVM проверяет флаг).
    - Абстрактные методы имеют флаг `ACC_ABSTRACT` и не содержат тела.
- **Наследование**:
    
    - JVM использует таблицу виртуальных методов (vtable) для полиморфных вызовов.
    - Конкретные методы наследников переопределяют записи в vtable.
    - Вызов метода (`invokevirtual`) разрешает метод на основе фактического типа объекта.
- **Память**:
    
    - Абстрактный класс хранит поля в объекте наследника.
    - Размер объекта зависит от полей базового класса и наследника.
    - Конструкторы вызываются через `invokespecial`, начиная с базового класса.

**Пример байт-кода (упрощенно)**:

```java
public abstract class Animal {
    public abstract void makeSound();
}
```

Байт-код:

```
.class public abstract Animal
  .method public abstract makeSound()V
  .end method
```

- **Производительность**:
    - Вызовы методов через полиморфизм (`invokevirtual`) оптимизируются JIT-компилятором (инлайн-кэширование).
    - Накладные расходы минимальны, сравнимы с обычными классами.

## 6. Производительность

- **Создание объектов**:
    
    - Конструкторы абстрактных классов вызываются в наследниках, что не добавляет значительных накладных расходов.
    - Размер объекта зависит от полей (аналогично обычным классам).
- **Вызов методов**:
    
    - Конкретные методы: O(1), как в обычных классах.
    - Полиморфные вызовы: O(1) с JIT-оптимизациями (vtable lookup).
    - Абстрактные методы: Реализация в наследниках, производительность зависит от их кода.
- **Память**:
    
    - Абстрактные классы не создают собственных объектов, но поля базового класса увеличивают размер объектов наследников.
    - Множественные уровни наследования могут усложнить оптимизацию JVM.

**Рекомендация**:

- Минимизируйте количество полей в абстрактных классах.
- Избегайте глубоких иерархий наследования для упрощения JIT-оптимизаций.

## 8. Сравнение с интерфейсами

|Характеристика|Абстрактный класс|Интерфейс|
|---|---|---|
|**Методы**|Абстрактные и конкретные|Только абстрактные (до Java 8), дефолтные/статические (Java 8+)|
|**Поля**|Могут содержать состояние|Только `public static final` константы|
|**Конструкторы**|Поддерживаются|Не поддерживаются|
|**Наследование**|Один класс|Множество интерфейсов|
|**Модификаторы**|`public`, `protected`, пакетный доступ|Только `public` (до Java 9), `private` методы (Java 9+)|
|**Когда использовать**|Общая реализация + состояние|Контракт без состояния, множественное наследование|

**Пример выбора**:

- **Абстрактный класс**: Моделирование иерархии с общим состоянием (например, `Animal` с полем `name`).
- **Интерфейс**: Определение контракта без реализации (например, `Runnable`, `Comparable`).

**С Java 8+**:

- Интерфейсы с `default`-методами частично пересекаются с абстрактными классами.
- Используйте абстрактные классы, если нужно состояние или сложная общая логика.

## 9. Современные возможности (Java 8+)

1. **Интеграция с интерфейсами**:
    
    - Абстрактные классы могут реализовывать интерфейсы с `default`-методами:
        
        ```java
        public interface Eatable {
            default void eat() {
                System.out.println("Ест еду");
            }
        }
        
        public abstract class Animal implements Eatable {
            public abstract void makeSound();
        }
        ```
        
2. **Стримы и коллекции**:
    
    - Абстрактные классы могут использоваться в коллекциях с полиморфным перебором:
        
        ```java
        List<Animal> animals = Arrays.asList(new Dog("Бобик"), new Cat("Мурка"));
        animals.stream().forEach(Animal::makeSound);
        ```
        
3. **Sealed Classes (Java 17+)**:
    
    - Абстрактные классы могут быть `sealed`, ограничивая круг наследников:
        
        ```java
        public sealed abstract class Animal permits Dog, Cat {
            public abstract void makeSound();
        }
        ```
        

## 10. Альтернативы абстрактным классам

1. **Интерфейсы с `default`-методами**:
    
    - Подходят для контрактов без состояния.
    - Пример: `List` с `default`-методом `sort`.
2. **Композиция вместо наследования**:
    
    - Используйте композицию, если наследование усложняет структуру:
        
        ```java
        public class AnimalBehavior {
            public void sleep(String name) {
                System.out.println(name + " спит");
            }
        }
        
        public class Dog {
            private String name;
            private AnimalBehavior behavior = new AnimalBehavior();
        
            public Dog(String name) {
                this.name = name;
            }
        
            public void makeSound() {
                System.out.println("Гав!");
            }
        
            public void sleep() {
                behavior.sleep(name);
            }
        }
        ```
        
3. **Функциональные интерфейсы**:
    
    - Для простых операций используйте лямбда-выражения вместо абстрактных классов:
        
        ```java
        Consumer<String> sound = s -> System.out.println(s + ": Гав!");
        sound.accept("Бобик");
        ```
        

**Рекомендация**: Используйте абстрактные классы, если требуется общее состояние и сложная логика. Для простых контрактов предпочтите интерфейсы или композицию.

## 11. Подводные камни

1. **Глубокое наследование**:
    
    - Многоуровневые иерархии усложняют код и оптимизацию JVM.
    - **Решение**: Ограничивайте глубину наследования (2–3 уровня).
2. **Неправильное использование состояния**:
    
    - Поля в абстрактном классе могут привести к состоянию гонки в многопоточных приложениях.
    - **Решение**: Используйте синхронизацию или неизменяемые поля.
3. **Переопределение конкретных методов**:
    
    - Наследники могут неправильно переопределять методы, нарушая логику.
    - **Решение**: Используйте `final` для методов, которые не должны переопределяться:
        
        ```java
        public abstract class Animal {
            public final void sleep() {
                System.out.println("Спит");
            }
        }
        ```
        
4. **Сравнение с интерфейсами**:
    
    - Неправильный выбор между абстрактным классом и интерфейсом усложняет архитектуру.
    - **Решение**: Используйте интерфейсы для контрактов, абстрактные классы для состояния.

## 12. Лучшие практики

1. **Задавайте минимально необходимую функциональность**:
    
    - Абстрактные методы должны быть минимальным контрактом.
    - Конкретные методы — для общей логики.
2. **Используйте полиморфизм**:
    
    - Храните объекты в коллекциях типа абстрактного класса:
        
        ```java
        List<Animal> animals = new ArrayList<>();
        animals.add(new Dog("Бобик"));
        animals.add(new Cat("Мурка"));
        ```
        
3. **Ограничивайте наследование**:
    
    - Используйте `sealed` классы (Java 17+) или композицию.
4. **Обеспечивайте потокобезопасность**:
    
    - Синхронизируйте доступ к полям или используйте потокобезопасные структуры.
5. **Документируйте методы**:
    
    - Указывайте в Javadoc, какие методы должны переопределять наследники:
        
        ```java
        /**
         * Издает звук, специфичный для животного.
         */
        public abstract void makeSound();
        ```
        
6. **Избегайте глубоких иерархий**:
    
    - Предпочитайте композицию или интерфейсы для сложных систем.

## 13. Пример: Комплексное использование

```java
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;
import java.util.logging.Logger;

// Интерфейс для дополнительного поведения
interface Eatable {
    default void eat() {
        System.out.println("Ест еду");
    }
}

// Абстрактный класс
public abstract sealed class Animal implements Eatable permits Dog, Cat {
    protected String name;
    protected int age;
    private static final Logger LOGGER = Logger.getLogger(Animal.class.getName());

    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Абстрактный метод
    public abstract void makeSound();

    // Конкретный метод
    public void sleep() {
        LOGGER.info(name + " спит");
    }

    // Геттер
    public String getName() {
        return name;
    }
}

// Наследник 1
public final class Dog extends Animal {
    public Dog(String name, int age) {
        super(name, age);
    }

    @Override
    public void makeSound() {
        System.out.println(name + ": Гав!");
    }
}

// Наследник 2
public final class Cat extends Animal {
    public Cat(String name, int age) {
        super(name, age);
    }

    @Override
    public void makeSound() {
        System.out.println(name + ": Мяу!");
    }

    @Override
    public void eat() {
        System.out.println(name + ": Ест рыбу");
    }
}

// Демонстрация
public class AbstractClassExample {
    private static final Logger LOGGER = Logger.getLogger(AbstractClassExample.class.getName());

    public static void main(String[] args) {
        // Полиморфизм
        List<Animal> animals = new ArrayList<>();
        animals.add(new Dog("Бобик", 3));
        animals.add(new Cat("Мурка", 2));

        // Перебор с использованием стрима
        animals.stream()
               .forEach(animal -> {
                   LOGGER.info("Животное: " + animal.getName());
                   animal.makeSound();
                   animal.sleep();
                   animal.eat();
               });

        // Многопоточность
        ConcurrentHashMap<String, Animal> animalMap = new ConcurrentHashMap<>();
        animalMap.put("Бобик", new Dog("Бобик", 3));
        new Thread(() -> animalMap.put("Мурка", new Cat("Мурка", 2))).start();

        // Проверка в другом потоке
        new Thread(() -> {
            animalMap.forEach((name, animal) -> {
                LOGGER.info("Потокобезопасный доступ: " + name);
                animal.makeSound();
            });
        }).start();
    }
}
```

**Предполагаемый вывод**:

```
INFO: Животное: Бобик
Бобик: Гав!
INFO: Бобик спит
Бобик: Ест еду
INFO: Животное: Мурка
Мурка: Мяу!
INFO: Мурка спит
Мурка: Ест рыбу
INFO: Потокобезопасный доступ: Бобик
Бобик: Гав!
INFO: Потокобезопасный доступ: Мурка
Мурка: Мяу!
```
