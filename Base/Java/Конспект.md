## 1. Основы

### 1.1. Примитивные типы данных
**Целые числа:**
- `byte`: -128 до 127 (8 бит, 1 байт)
- `short`: -32,768 до 32,767 (16 бит, 2 байта)  
- `int`: -2,147,483,648 до 2,147,483,647 (32 бита, 4 байта)
- `long`: -9,223,372,036,854,775,808 до 9,223,372,036,854,775,807 (64 бита, 8 байт)

**Числа с плавающей точкой:**
- `float`: ±3.4E+38 (32 бита, точность ~7 цифр)
- `double`: ±1.7E+308 (64 бита, точность ~15 цифр)

**Логический и символьный:**
- `boolean`: true/false (JVM-зависимый размер)
- `char`: Unicode символы 0-65,535 (16 бит, 2 байта)

**Особенности:**
- Значения по умолчанию: 0 для чисел, false для boolean, '\u0000' для char
- Автоупаковка/автораспаковка с wrapper-классами
- Переполнение целых типов происходит циклически

### 1.2. Переменные
- **Локальные**: объявляются внутри методов, должны быть инициализированы
- **Поля экземпляра**: автоматически получают значения по умолчанию
- **Статические поля**: принадлежат классу, не экземпляру
- `final` переменные: константы, присваиваются только один раз
- Shadowing: локальная переменная скрывает поле класса

### 1.3. Массивы
- Объекты в Java, хранятся в куче
- Фиксированный размер после создания
- Индексация с 0, длина через `.length`
- Могут быть многомерными (массивы массивов)
- Инициализация: `int[] arr = {1, 2, 3}` или `new int[5]`

### 1.4. Управляющие конструкции
**Условные операторы:**
- `if-else`: базовые условные конструкции
- Тернарный оператор `condition ? value1 : value2`
- `switch` (поддерживает String с Java 7, выражения с Java 14)

**Циклы:**
- `for`: классический цикл с счетчиком
- `while`: цикл с предусловием
- `do-while`: цикл с постусловием
- `for-each`: упрощенный цикл для коллекций и массивов
- `break`, `continue`: управление выполнением циклов

### 1.5. Методы и функции
- Методы принадлежат классам, статические - классу
- Перегрузка (overloading): одинаковые имена, разные параметры
- Переопределение (overriding): изменение поведения в наследниках
- Переменное количество аргументов (varargs): `method(int... args)`
- Возвращаемые типы: примитивы, объекты или void

### 1.6. Структура памяти Java
**Куча (Heap):**
- Young Generation: Eden (новые объекты), S0/S1 (выжившие объекты)
- Old Generation (Tenured): долгоживущие объекты
- Metaspace (с Java 8): метаданные классов

**Не-куча:**
- Стек: локальные переменные, параметры методов
- PC Register: указатель на текущую инструкцию
- Native Method Stack: для native методов

### 1.7. Сборка и выполнение
- Компиляция: `.java` файлы → `.class` файлы (байт-код)
- JVM загружает классы через ClassLoader
- JIT-компилятор оптимизирует "горячий" код во время выполнения
- Сборщик мусора автоматически управляет памятью

## 2. ООП и базовые конструкции

### 2.1. Принципы ООП
**Инкапсуляция:**
- Сокрытие внутренней реализации
- Доступ к данным через public методы (геттеры/сеттеры)
- Защита данных от некорректного изменения
- Использование модификаторов доступа

**Наследование:**
- `extends` для классов, `implements` для интерфейсов
- Единственное наследование классов, множественное интерфейсов
- Переопределение методов с аннотацией `@Override`
- Конструкторы не наследуются

**Полиморфизм:**
- Один интерфейс - разные реализации
- Раннее связывание (статические методы) и позднее (виртуальные)
- Приведение типов: upcasting (автоматическое) и downcasting (явное)
- `instanceof` для проверки типа

**Абстракция:**
- Абстрактные классы с ключевым словом `abstract`
- Интерфейсы для определения контрактов
- Скрытие сложности реализации

### 2.2. Модификаторы доступа
- `public`: доступен отовсюду
- `protected`: доступен в пакете и наследникам
- `default` (package-private): доступен только в пакете
- `private`: доступен только в текущем классе

### 2.3. Классы и объекты
- Класс - шаблон/чертеж, объект - экземпляр
- Конструкторы для инициализации объектов
- `this` для ссылки на текущий объект
- `super` для обращения к родительскому классу
- Статические блоки инициализации выполняются при загрузке класса

### 2.4. Интерфейсы
- Контракты поведения без реализации (до Java 8)
- `default` методы (Java 8+): методы с реализацией по умолчанию
- `static` методы в интерфейсах
- Функциональные интерфейсы: один абстрактный метод
- Множественное наследование поведения

### 2.5. Enums
- Типобезопасные константы
- Могут содержать поля, методы, конструкторы
- Автоматически наследуют от `java.lang.Enum`
- Методы: `values()`, `valueOf()`, `ordinal()`, `name()`

### 2.6. Вложенные классы
- **Статические**: не имеют доступа к внешнему экземпляру
- **Внутренние**: имеют доступ к полям внешнего класса
- **Локальные**: объявляются внутри методов
- **Анонимные**: для одноразового использования без имени

### 2.7. Generic (Обобщения)
- Параметризованные типы для типобезопасности
- Wildcards: `?` (любой тип), `? extends T` (верхняя граница), `? super T` (нижняя граница)
- Type erasure: стирание типов во время компиляции
- Ограничения: нельзя создавать массивы generic типов

### 2.8. Static
- Принадлежит классу, не экземпляру
- Статические методы не могут обращаться к нестатическим полям
- Инициализация при первом обращении к классу
- Статические импорты для удобства

### 2.9. Современные возможности Java
**Java 8+:**
- Лямбда-выражения: `(param) -> expression`
- Stream API для функциональной обработки данных
- Optional для избежания NullPointerException
- Ссылки на методы: `Class::method`

**Java 9+:**
- Модульная система (JPMS)
- Factory методы для коллекций: `List.of()`, `Set.of()`, `Map.of()`
- try-with-resources с effectively final переменными

**Java 14+:**
- Records для data-классов
- Pattern matching для instanceof
- Switch expressions с `yield`

**Java 17+:**
- Sealed классы: ограничение иерархии наследования
- Text blocks для многострочных строк

## 3. Коллекции и работа с данными

### 3.1. Иерархия коллекций
```
Collection<E>
├── List<E> (упорядоченная, с дубликатами)
│   ├── ArrayList<E> (динамический массив)
│   ├── LinkedList<E> (двусвязный список)
│   └── Vector<E> (синхронизированный, устаревший)
├── Set<E> (без дубликатов)
│   ├── HashSet<E> (хеш-таблица)
│   ├── LinkedHashSet<E> (сохраняет порядок вставки)
│   └── TreeSet<E> (сортированное множество)
└── Queue<E> (очередь FIFO)
    ├── PriorityQueue<E> (куча с приоритетами)
    ├── ArrayDeque<E> (двусторонняя очередь)
    └── LinkedList<E> (также реализует Deque)

Map<K,V> (пары ключ-значение, не наследует Collection)
├── HashMap<K,V> (хеш-таблица)
├── LinkedHashMap<K,V> (сохраняет порядок)
├── TreeMap<K,V> (сортированная карта)
└── Hashtable<K,V> (синхронизированная, устаревшая)
```

### 3.2. Производительность коллекций (Big-O)
**ArrayList:**
- Доступ по индексу: O(1)
- Добавление в конец: O(1) амортизированное
- Вставка/удаление в середине: O(n)

**LinkedList:**
- Доступ по индексу: O(n)
- Добавление/удаление на концах: O(1)
- Поиск элемента: O(n)

**HashMap:**
- get/put/remove: O(1) среднее, O(n) худший случай
- В Java 8+ использует деревья при коллизиях (O(log n))

**TreeMap/TreeSet:**
- Все операции: O(log n)
- Использует красно-черное дерево для балансировки

### 3.3. Stream API
**Архитектура:**
- Источник данных → Промежуточные операции → Терминальная операция
- Ленивое выполнение: вычисления начинаются только при терминальной операции

**Промежуточные операции (возвращают Stream):**
- `filter(Predicate)`: фильтрация элементов
- `map(Function)`: преобразование элементов
- `sorted()`: сортировка
- `distinct()`: удаление дубликатов
- `limit(n)`: ограничение количества элементов

**Терминальные операции (запускают выполнение):**
- `collect(Collector)`: сбор в коллекцию
- `forEach(Consumer)`: выполнение действия для каждого элемента
- `reduce(BinaryOperator)`: свертка к одному значению
- `count()`: подсчет элементов

**Параллельные потоки:**
- `parallelStream()` для многопоточной обработки
- Эффективны для CPU-интенсивных задач на больших данных
- Используют ForkJoinPool.commonPool()

### 3.4. Функциональное программирование
**Основные функциональные интерфейсы:**
- `Predicate<T>`: принимает T, возвращает boolean (test())
- `Function<T,R>`: принимает T, возвращает R (apply())
- `Consumer<T>`: принимает T, ничего не возвращает (accept())
- `Supplier<T>`: не принимает аргументов, возвращает T (get())

**Лямбда-выражения:**
- Краткий синтаксис для анонимных функций
- Захват переменных (effectively final)
- Ссылки на методы: `Class::method`, `object::method`, `Class::new`

## 4. Работа со строками

### 4.1. String
- **Неизменяемость**: объекты String нельзя изменить после создания
- **Пул строк**: оптимизация памяти через переиспользование
- **Кодировка**: UTF-16, compact strings в Java 9+ (Latin-1 или UTF-16)
- **Основные методы**: `length()`, `charAt()`, `substring()`, `indexOf()`, `contains()`

### 4.2. StringBuilder/StringBuffer
- **StringBuilder**: изменяемые строки, не потокобезопасный (быстрее)
- **StringBuffer**: потокобезопасный аналог (медленнее из-за синхронизации)
- **Методы**: `append()`, `insert()`, `delete()`, `reverse()`, `toString()`
- **Применение**: эффективная конкатенация в циклах

### 4.3. Современные возможности
- **String.format()**: форматирование с placeholders (`%s`, `%d`, `%f`)
- **String.formatted()** (Java 15+): метод экземпляра для форматирования
- **Text blocks** (Java 15+): многострочные строки с `"""`
- **StringJoiner**: объединение строк с разделителями
- **Регулярные выражения**: Pattern и Matcher для сложного поиска

## 5. Дата и время

### 5.1. Современный API (Java 8+)
**Основные классы:**
- `LocalDate`: дата без времени (2024-01-15)
- `LocalTime`: время без даты (14:30:00)
- `LocalDateTime`: дата и время без часового пояса
- `ZonedDateTime`: дата, время и часовой пояс
- `Instant`: точка времени в UTC (timestamp)

**Работа с интервалами:**
- `Duration`: временные интервалы (часы, минуты, секунды)
- `Period`: календарные интервалы (годы, месяцы, дни)

**Форматирование:**
- `DateTimeFormatter`: парсинг и форматирование дат
- Предопределенные форматтеры: `ISO_LOCAL_DATE`, `ISO_DATE_TIME`
- Кастомные паттерны: `"dd.MM.yyyy HH:mm:ss"`

### 5.2. Устаревший API
- `java.util.Date` и `Calendar`: избегать в новом коде
- **Проблемы**: мутабельность, неинтуитивный API, проблемы с потокобезопасностью
- **Конвертация**: через `Instant` при работе с legacy кодом

## 6. Ввод-вывод и сериализация

### 6.1. I/O потоки
**Байтовые потоки (для бинарных данных):**
- `InputStream`/`OutputStream`: базовые абстрактные классы
- `FileInputStream`/`FileOutputStream`: работа с файлами
- `BufferedInputStream`/`BufferedOutputStream`: буферизация для производительности

**Символьные потоки (для текста):**
- `Reader`/`Writer`: базовые абстрактные классы
- `FileReader`/`FileWriter`: текстовые файлы
- `BufferedReader`/`BufferedWriter`: буферизация
- `InputStreamReader`/`OutputStreamWriter`: мост между байтовыми и символьными потоками

### 6.2. NIO.2 (Java 7+)
- `Path`: представление пути к файлу/директории
- `Files`: утилитный класс с статическими методами
- **Преимущества**: более простой API, лучшая производительность
- **Методы**: `Files.readString()`, `Files.writeString()`, `Files.copy()`, `Files.walk()`

### 6.3. Сериализация
**Serializable:**
- Маркерный интерфейс для автоматической сериализации
- `serialVersionUID`: контроль совместимости версий
- `transient`: исключение полей из сериализации

**Externalizable:**
- Полный контроль через `writeExternal()`/`readExternal()`
- Лучшая производительность, но больше кода

**Безопасность:**
- `ObjectInputFilter` (Java 9+) для фильтрации классов
- **Альтернативы**: JSON (Jackson, Gson), XML, Protocol Buffers

## 7. Исключения

### 7.1. Иерархия исключений
```
Throwable
├── Error (критические ошибки JVM)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
└── Exception
    ├── Checked Exceptions (обязательная обработка)
    │   ├── IOException
    │   ├── SQLException
    │   └── ClassNotFoundException
    └── RuntimeException (unchecked)
        ├── NullPointerException
        ├── IllegalArgumentException
        ├── ArrayIndexOutOfBoundsException
        └── ConcurrentModificationException
```

### 7.2. Обработка исключений
**try-catch-finally:**
```java
try {
    // код, который может выбросить исключение
} catch (SpecificException e) {
    // обработка конкретного исключения
} catch (Exception e) {
    // обработка общего исключения
} finally {
    // код, который выполняется всегда
}
```

**try-with-resources (Java 7+):**
- Автоматическое закрытие ресурсов, реализующих `AutoCloseable`
- Предотвращает утечки ресурсов
- Может использовать несколько ресурсов

**Создание собственных исключений:**
- Наследование от `Exception` (checked) или `RuntimeException` (unchecked)
- Передача причины через конструктор или `initCause()`

### 7.3. Лучшие практики
- Ловить конкретные исключения, избегать `catch (Exception e)`
- Логировать исключения с достаточным контекстом
- Не игнорировать исключения (пустой catch блок)
- Использовать исключения только для исключительных ситуаций

## 9. Многопоточность

### 9.1. Создание потоков
**Способы создания:**
```java
// 1. Наследование Thread
class MyThread extends Thread {
    public void run() { /* код потока */ }
}

// 2. Реализация Runnable
Thread thread = new Thread(() -> { /* код потока */ });

// 3. ExecutorService (рекомендуемый)
ExecutorService executor = Executors.newFixedThreadPool(4);
executor.submit(() -> { /* код задачи */ });

// 4. Виртуальные потоки (Java 21+)
Thread.ofVirtual().start(() -> { /* код потока */ });
```

### 9.2. Состояния потоков
- `NEW`: поток создан, но не запущен
- `RUNNABLE`: выполняется или готов к выполнению
- `BLOCKED`: ожидает монитор для входа в synchronized блок
- `WAITING`: ожидает уведомления (wait(), join() без timeout)
- `TIMED_WAITING`: ожидает с таймаутом (sleep(), wait(timeout))
- `TERMINATED`: поток завершил выполнение

### 9.3. Синхронизация
**synchronized:**
- Обеспечивает взаимное исключение (mutex) и видимость изменений
- Может блокировать методы целиком или отдельные блоки кода
- Использует мониторы объектов для блокировки
- Может привести к deadlock при неправильном использовании

**volatile:**
- Гарантирует видимость изменений переменной всем потокам
- Предотвращает кэширование в регистрах CPU
- НЕ обеспечивает атомарность составных операций (например, increment)

**Атомарные классы:**
- `AtomicInteger`, `AtomicLong`, `AtomicReference`, `AtomicBoolean`
- Lock-free операции через CAS (Compare-And-Swap)
- Высокая производительность без блокировок
- Методы: `get()`, `set()`, `compareAndSet()`, `incrementAndGet()`

### 9.4. Продвинутая синхронизация (java.util.concurrent)
**ReentrantLock:**
- Более гибкая альтернатива synchronized
- Поддержка `tryLock()` для избежания блокировок
- Справедливые блокировки (fairness)
- Условные переменные через `Condition`

**Семафоры и барьеры:**
- `Semaphore`: ограничение количества потоков, имеющих доступ к ресурсу
- `CountDownLatch`: ожидание завершения фиксированного числа событий
- `CyclicBarrier`: синхронизация группы потоков в определенной точке

### 9.5. Пулы потоков
**ExecutorService:**
- `newFixedThreadPool(n)`: фиксированное число потоков
- `newCachedThreadPool()`: динамическое создание потоков
- `newSingleThreadExecutor()`: один поток для последовательного выполнения
- `newScheduledThreadPool(n)`: планирование задач по времени

**Future и CompletableFuture:**
- `Future<T>`: результат асинхронной задачи
- `CompletableFuture<T>`: асинхронное программирование с цепочками операций
- Методы: `thenApply()`, `thenCompose()`, `exceptionally()`, `allOf()`, `anyOf()`

### 9.6. Современные возможности (Java 21+)
**Виртуальные потоки (Project Loom):**
- Легковесные потоки, управляемые JVM (не ОС)
- Масштабируемость до миллионов потоков
- Идеальны для I/O-интенсивных задач
- Используют carrier threads для выполнения

**Structured Concurrency:**
- `StructuredTaskScope` для управления группами связанных задач
- Автоматическая отмена дочерних задач при ошибке
- Структурированный подход к параллельному выполнению

### 9.7. Потокобезопасные коллекции
- `ConcurrentHashMap`: lock-free хеш-таблица с сегментированной блокировкой
- `CopyOnWriteArrayList`: оптимизирована для частого чтения
- `BlockingQueue`: очереди с блокировкой для паттерна producer-consumer
- `ConcurrentSkipListMap`: потокобезопасная сортированная карта

### 9.8. Подводные камни
- **Race conditions**: конкуренция за общие ресурсы без синхронизации
- **Deadlock**: взаимная блокировка двух или более потоков
- **Livelock**: потоки активно мешают друг другу прогрессировать
- **Starvation**: поток не получает доступ к ресурсам длительное время

### 9.9. Лучшие практики
- Использовать неизменяемые объекты где возможно
- Минимизировать область synchronized блоков
- Предпочитать высокоуровневые примитивы (ExecutorService, atomic классы)
- Избегать блокировок в одинаковом порядке для предотвращения deadlock
- Тестировать многопоточный код на race conditions и deadlock
- Применять виртуальные потоки для I/O-bound задач
