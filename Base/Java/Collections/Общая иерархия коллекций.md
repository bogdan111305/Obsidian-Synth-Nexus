## 1. Основы коллекций

Фреймворк коллекций в Java начинается с двух корневых интерфейсов:

- **`java.util.Collection<E>`**: Представляет группу объектов (элементов), таких как списки, множества или очереди. Поддерживает итерацию и общие операции (добавление, удаление, проверка наличия).
- **`java.util.Map<K, V>`**: Хранит пары ключ-значение, где каждый ключ уникален. Не является подтипом `Collection`, но интегрируется с ним через методы вроде `entrySet()`.

## 2. Основные интерфейсы

Ниже приведены ключевые интерфейсы фреймворка коллекций и их назначение:

|Интерфейс|Назначение|
|---|---|
|`Collection`|Корневой интерфейс для всех коллекций, поддерживает базовые операции|
|`List`|Упорядоченная коллекция с доступом по индексу, допускает дубликаты|
|`Set`|Множество уникальных элементов, не допускает дубликаты|
|`Queue`|Очередь (обычно FIFO), для добавления/удаления элементов|
|`Deque`|Двусторонняя очередь, поддерживает добавление/удаление с обоих концов|
|`SortedSet`|`Set` с элементами, отсортированными по `Comparable` или `Comparator`|
|`NavigableSet`|Расширение `SortedSet` с методами для навигации (например, `ceiling`, `floor`)|
|`Map`|Хранит пары ключ-значение, ключи уникальны|
|`SortedMap`|`Map` с ключами, отсортированными по `Comparable` или `Comparator`|
|`NavigableMap`|Расширение `SortedMap` с методами для навигации (например, `ceilingKey`)|

## 4. Классы-реализации и их особенности

### 4.1. `List`

- **`ArrayList<E>`**:
    
    - **Структура**: Динамический массив.
    - **Особенности**: Быстрый доступ по индексу (O(1)), медленное добавление/удаление в середину (O(n)).
    - **Использование**: Общее назначение, когда важен быстрый доступ по индексу.
- **`LinkedList<E>`**:
    
    - **Структура**: Двусвязный список.
    - **Особенности**: Быстрое добавление/удаление в начало/конец (O(1)), медленный доступ по индексу (O(n)). Также реализует `Deque`.
    - **Использование**: Когда важны операции с концами списка или очередь.
- **`Vector<E>`**:
    
    - **Структура**: Синхронизированный динамический массив (устаревший).
    - **Особенности**: Потокобезопасный, но медленнее `ArrayList` из-за синхронизации.
    - **Использование**: В многопоточных приложениях (лучше использовать `Collections.synchronizedList`).
- **`Stack<E>`**:
    
    - **Структура**: Расширяет `Vector`, реализует стек (LIFO).
    - **Особенности**: Устаревший, предпочтительнее использовать `ArrayDeque` для стека.
    - **Использование**: Для задач, требующих стека.

### 4.2. `Set`

- **`HashSet<E>`**:
    
    - **Структура**: Хэш-таблица на базе `HashMap`.
    - **Особенности**: Быстрые операции добавления/поиска/удаления (O(1) в среднем), не сохраняет порядок.
    - **Использование**: Когда важна уникальность и производительность.
- **`LinkedHashSet<E>`**:
    
    - **Структура**: Хэш-таблица с двусвязным списком.
    - **Особенности**: Сохраняет порядок добавления, чуть медленнее `HashSet`.
    - **Использование**: Когда нужен порядок вставки.
- **`TreeSet<E>`**:
    
    - **Структура**: Красно-черное дерево.
    - **Особенности**: Элементы сортируются (O(log n) для операций), требует `Comparable` или `Comparator`.
    - **Использование**: Для упорядоченных множеств.

### 4.3. `Queue` / `Deque`

- **`PriorityQueue<E>`**:
    
    - **Структура**: Куча (heap).
    - **Особенности**: Элементы извлекаются по приоритету (O(log n) для добавления/удаления).
    - **Использование**: Для задач с приоритетами (например, планировщик задач).
- **`ArrayDeque<E>`**:
    
    - **Структура**: Динамический массив с кольцевым буфером.
    - **Особенности**: Быстрые операции с обоих концов (O(1)), не синхронизирован.
    - **Использование**: Для стеков и очередей.
- **`LinkedList<E>`**:
    
    - **Особенности**: Реализует как `List`, так и `Deque`, универсальна, но медленнее `ArrayDeque` для очередей.
    - **Использование**: Когда требуется гибкость.

### 4.4. `Map`

- **`HashMap<K, V>`**:
    
    - **Структура**: Хэш-таблица.
    - **Особенности**: Быстрые операции (O(1) в среднем), не сохраняет порядок, допускает `null`.
    - **Использование**: Общее назначение для пар ключ-значение.
- **`LinkedHashMap<K, V>`**:
    
    - **Структура**: Хэш-таблица с двусвязным списком.
    - **Особенности**: Сохраняет порядок вставки или доступа, чуть медленнее `HashMap`.
    - **Использование**: Для порядка или LRU-кэша.
- **`TreeMap<K, V>`**:
    
    - **Структура**: Красно-черное дерево.
    - **Особенности**: Ключи сортируются (O(log n)), требует `Comparable` или `Comparator`.
    - **Использование**: Для упорядоченных словарей.
- **`Hashtable<K, V>`**:
    
    - **Структура**: Синхронизированная хэш-таблица (устаревшая).
    - **Особенности**: Потокобезопасная, не допускает `null`.
    - **Использование**: В многопоточных приложениях (лучше `ConcurrentHashMap`).
- **`Properties`**:
    
    - **Структура**: Расширяет `Hashtable`.
    - **Особенности**: Для работы с конфигурационными файлами (ключ и значение — строки).
    - **Использование**: Для хранения настроек.

## 5. Отличия `Collection` и `Map`

|Характеристика|`Collection`|`Map`|
|---|---|---|
|**Тип элементов**|Отдельные элементы|Пары ключ-значение|
|**Итерация**|Через `Iterator` или `for-each`|Через `entrySet()`, `keySet()`, `values()`|
|**Примеры**|`List`, `Set`, `Queue`|`HashMap`, `TreeMap`, `ConcurrentHashMap`|
|**Дубликаты**|Зависит от реализации (`List` — да, `Set` — нет)|Ключи уникальны, значения могут повторяться|

## 6. Производительность (Big-O нотация)

Производительность операций зависит от реализации. Ниже приведены основные операции и их временная сложность:

|Реализация|Добавить (add/put)|Удалить (remove)|Поиск (contains/get)|Итерация|
|---|---|---|---|---|
|**ArrayList**|O(1) (конец), O(n) (середина)|O(n)|O(n)|O(n)|
|**LinkedList**|O(1) (конец), O(n) (по индексу)|O(1) (если известен узел), O(n)|O(n)|O(n)|
|**Vector**|O(1) (конец), O(n) (середина)|O(n)|O(n)|O(n)|
|**HashSet**|O(1) (среднее)|O(1) (среднее)|O(1) (среднее)|O(n)|
|**LinkedHashSet**|O(1) (среднее)|O(1) (среднее)|O(1) (среднее)|O(n)|
|**TreeSet**|O(log n)|O(log n)|O(log n)|O(n)|
|**PriorityQueue**|O(log n)|O(log n)|O(n)|O(n)|
|**ArrayDeque**|O(1) (концы)|O(1) (концы)|O(n)|O(n)|
|**HashMap**|O(1) (среднее)|O(1) (среднее)|O(1) (среднее)|O(n)|
|**LinkedHashMap**|O(1) (среднее)|O(1) (среднее)|O(1) (среднее)|O(n)|
|**TreeMap**|O(log n)|O(log n)|O(log n)|O(n)|
|**Hashtable**|O(1) (среднее)|O(1) (среднее)|O(1) (среднее)|O(n)|

**Примечания**:

- O(1) для хэш-таблиц предполагает хорошую хэш-функцию и низкую вероятность коллизий.
- O(n) для `ArrayList` при добавлении/удалении в середину из-за сдвига элементов.
- `TreeSet` и `TreeMap` используют красно-черное дерево, что дает O(log n).

## 7. Внутренняя реализация в JVM

- **`ArrayList`**:
    
    - Хранит элементы в массиве (`Object[]`).
    - При превышении размера массив увеличивается (копирование, O(n)).
    - JIT-компилятор оптимизирует доступ по индексу.
- **`LinkedList`**:
    
    - Узлы (`Node`) с ссылками на предыдущий и следующий элементы.
    - Выделение памяти для каждого узла увеличивает накладные расходы.
- **`HashSet` / `HashMap`**:
    
    - Хэш-таблица с массивом бакетов.
    - В Java 8+ при большом числе коллизий в бакете используется сбалансированное дерево (O(log n) вместо O(n)).
    - Хэш-код вычисляется через `hashCode()`.
- **`TreeSet` / `TreeMap`**:
    
    - Красно-черное дерево, обеспечивающее сортировку и балансировку.
    - Требует `Comparable` или `Comparator` для элементов/ключей.
- **`PriorityQueue`**:
    
    - Бинарная куча (массив с родительско-дочерними связями).
    - Балансировка кучи — O(log n).
- **`ArrayDeque`**:
    
    - Кольцевой буфер на основе массива.
    - Эффективное управление памятью для операций с концами.
- **`Vector` / `Hashtable`**:
    
    - Устаревшие, используют синхронизацию на уровне методов (`synchronized`).

**Байт-код**:  
Операции с коллекциями (например, `add`, `get`) компилируются в вызовы методов (`invokevirtual`, `invokeinterface`). Итерация через `for-each` использует `Iterator` или `Spliterator` для стримов.

## 8. Многопоточность

Большинство коллекций (`ArrayList`, `HashMap`, `TreeSet`, и т.д.) **не потокобезопасны**. Для многопоточной работы используйте:

- **Синхронизированные обертки**:
    
    ```java
    List<E> syncList = Collections.synchronizedList(new ArrayList<>());
    Map<K, V> syncMap = Collections.synchronizedMap(new HashMap<>());
    ```
    
- **Потокобезопасные коллекции (Java 5+)**:
    
    - `ConcurrentHashMap`: Высокопроизводительная хэш-таблица с частичной блокировкой.
    - `CopyOnWriteArrayList`: Копия массива при каждой модификации, подходит для частого чтения.
    - `ConcurrentLinkedQueue`: Потокобезопасная очередь.

**Пример**:

```java
import java.util.concurrent.ConcurrentHashMap;

Map<String, Integer> map = new ConcurrentHashMap<>();
new Thread(() -> map.put("key", 1)).start();
new Thread(() -> System.out.println(map.get("key"))).start();
```

- **Параллельные стримы**:  
    Коллекции поддерживают `parallelStream()` для многопоточной обработки.
    
    ```java
    List<Integer> list = Arrays.asList(1, 2, 3);
    list.parallelStream().forEach(System.out::println);
    ```
    

**Ограничение**: Параллельные стримы требуют осторожности, чтобы избежать гонок данных.

## 9. Современные возможности (Java 8+)

### 9.1. Стримы

`Stream API` позволяет выполнять функциональные операции над коллекциями.

**Пример**:

```java
List<String> list = Arrays.asList("apple", "banana", "cherry");
list.stream()
    .filter(s -> s.startsWith("a"))
    .map(String::toUpperCase)
    .forEach(System.out::println); // APPLE
```

- **Преимущества**: Декларативный стиль, поддержка параллельной обработки.
- **Недостатки**: Накладные расходы на создание потока, медленнее для простых операций.

### 9.2. Неизменяемые коллекции (Java 9+)

Методы `List.of()`, `Set.of()`, `Map.of()` создают неизменяемые коллекции.

**Пример**:

```java
List<String> immutableList = List.of("A", "B", "C");
immutableList.add("D"); // UnsupportedOperationException
```

- **Использование**: Для безопасной передачи данных без модификации.

### 9.3. Factory-методы для коллекций (Java 9+)

Метод `Map.ofEntries()` и другие для создания коллекций с множеством элементов.

**Пример**:

```java
Map<String, Integer> map = Map.ofEntries(
    Map.entry("A", 1),
    Map.entry("B", 2)
);
```

## 10. Подводные камни

1. **Модификация коллекции при итерации**:
    
    ```java
    List<String> list = new ArrayList<>(Arrays.asList("A", "B"));
    for (String s : list) {
        list.remove(s); // ConcurrentModificationException
    }
    ```
    
    **Решение**: Используйте `Iterator` или стримы.
    
2. **Null в коллекциях**:
    
    - `HashMap` допускает `null` ключи/значения, `Hashtable` — нет.
    - `TreeSet` и `TreeMap` не допускают `null`, если требуется `Comparable`.
3. **Неэффективный выбор реализации**:
    
    - Использование `LinkedList` для частого доступа по индексу (O(n) вместо O(1) в `ArrayList`).
    - **Решение**: Выбирайте реализацию по задаче.
4. **Гонки данных в многопоточности**:
    
    ```java
    List<Integer> list = new ArrayList<>();
    new Thread(() -> list.add(1)).start();
    new Thread(() -> list.add(2)).start(); // Возможна ошибка
    ```
    
    **Решение**: Используйте `ConcurrentHashMap` или синхронизированные обертки.
    
5. **Переполнение хэш-таблицы**:
    
    - Плохая хэш-функция в `HashMap`/`HashSet` приводит к O(n) вместо O(1).
    - **Решение**: Переопределяйте `hashCode()` и `equals()` корректно.

## 11. Лучшие практики

1. **Выбирайте правильную реализацию**:
    
    - `ArrayList` для быстрого доступа по индексу.
    - `LinkedList` или `ArrayDeque` для операций с концами.
    - `HashSet`/`HashMap` для быстрого поиска.
    - `TreeSet`/`TreeMap` для сортировки.
2. **Используйте стримы для сложных операций**:
    
    ```java
    List<Integer> numbers = Arrays.asList(1, 2, 3);
    int sum = numbers.stream().mapToInt(i -> i).sum();
    ```
    
3. **Применяйте неизменяемые коллекции**:
    
    ```java
    Set<String> set = Set.of("A", "B");
    ```
    
4. **Обеспечивайте потокобезопасность**:
    
    ```java
    Map<String, Integer> map = new ConcurrentHashMap<>();
    ```
    
5. **Переопределяйте `hashCode` и `equals` для пользовательских объектов**:
    
    ```java
    class Key {
        String id;
        @Override public int hashCode() { return id.hashCode(); }
        @Override public boolean equals(Object o) { /* логика */ }
    }
    ```
    
6. **Используйте итераторы для модификации**:
    
    ```java
    Iterator<String> iterator = list.iterator();
    while (iterator.hasNext()) {
        if (iterator.next().startsWith("A")) iterator.remove();
    }
    ```
    
7. **Избегайте устаревших классов**:
    
    - Предпочитайте `ArrayDeque` вместо `Stack`, `ConcurrentHashMap` вместо `Hashtable`.

## 12. Пример: Комплексное использование коллекций

```java
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;
import java.util.stream.Collectors;
import java.util.logging.Logger;

public class CollectionsExample {
    private static final Logger LOGGER = Logger.getLogger(CollectionsExample.class.getName());

    public static void main(String[] args) {
        // List: ArrayList
        List<String> list = new ArrayList<>(Arrays.asList("apple", "banana", "cherry"));
        LOGGER.info("ArrayList: " + list);
        list.add(1, "date");
        LOGGER.info("После добавления: " + list);

        // Set: TreeSet
        Set<String> set = new TreeSet<>(list);
        LOGGER.info("TreeSet (отсортирован): " + set);

        // Queue: ArrayDeque
        Deque<String> queue = new ArrayDeque<>();
        queue.offer("first");
        queue.offer("second");
        LOGGER.info("ArrayDeque: " + queue);
        LOGGER.info("Извлечено: " + queue.poll());

        // Map: ConcurrentHashMap
        Map<String, Integer> map = new ConcurrentHashMap<>();
        map.put("apple", 1);
        map.put("banana", 2);
        new Thread(() -> map.put("cherry", 3)).start();
        LOGGER.info("ConcurrentHashMap: " + map);

        // Стримы
        List<String> filtered = list.stream()
                                   .filter(s -> s.startsWith("a"))
                                   .map(String::toUpperCase)
                                   .collect(Collectors.toList());
        LOGGER.info("Фильтрация и преобразование: " + filtered);

        // Неизменяемая коллекция
        Set<String> immutableSet = Set.of("X", "Y");
        LOGGER.info("Неизменяемый Set: " + immutableSet);

        // Многопоточная обработка
        List<Integer> numbers = IntStream.range(0, 5).boxed().collect(Collectors.toList());
        numbers.parallelStream().forEach(n -> LOGGER.info("Параллельный стрим: " + n));
    }
}
```

**Предполагаемый вывод** (порядок для `parallelStream` может варьироваться):

```
INFO: ArrayList: [apple, banana, cherry]
INFO: После добавления: [apple, date, banana, cherry]
INFO: TreeSet (отсортирован): [apple, banana, cherry, date]
INFO: ArrayDeque: [first, second]
INFO: Извлечено: first
INFO: ConcurrentHashMap: {apple=1, banana=2, cherry=3}
INFO: Фильтрация и преобразование: [APPLE]
INFO: Неизменяемый Set: [X, Y]
INFO: Параллельный стрим: 0
INFO: Параллельный стрим: 1
INFO: Параллельный стрим: 2
INFO: Параллельный стрим: 3
INFO: Параллельный стрим: 4
```

## 13. Заключение

Фреймворк коллекций в Java предоставляет гибкие и мощные инструменты для работы с данными. Интерфейсы `Collection` и `Map` формируют основу иерархии, а их реализации (`ArrayList`, `HashMap`, `TreeSet`, и т.д.) покрывают различные сценарии. Понимание внутренней реализации, производительности (Big-O), многопоточности и современных возможностей (стримы, неизменяемые коллекции) позволяет выбирать подходящую структуру данных. Соблюдение лучших практик и учет подводных камней, таких как `ConcurrentModificationException` или неправильный `hashCode()`, обеспечивает надежность и эффективность кода.
