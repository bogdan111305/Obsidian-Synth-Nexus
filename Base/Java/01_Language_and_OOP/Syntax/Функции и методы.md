# Функции и методы в Java

> [!QUOTE] Суть
> **Метод** — именованный блок кода внутри класса, выполняющий задачу. Может принимать параметры и возвращать результат. В Java методы всегда принадлежат классу/интерфейсу. Параметры передаются **по значению** (для объектов — копия ссылки).
## Синтаксис метода

```java
[модификаторы] возвращаемый_тип имяМетода(тип1 параметр1, тип2 параметр2, ...) [throws Исключение] {
    // Тело метода
    return значение; // Если возвращаемый тип не void
}
```

- **Модификаторы**: `public`, `private`, `static`, `final`, etc.
- **Возвращаемый тип**: Примитив (`int`, `double`), объект (`String`, `List`), или `void`.
- **Имя метода**: Следует camelCase (например, `calculateSum`).
- **Параметры**: Могут быть любыми типами, включая примитивы и объекты.
- **Исключения**: Указываются через `throws` для проверяемых исключений.

```java
public int add(int a, int b) {
    return a + b;
}
```
## Виды методов

### Экземплярные методы

- Привязаны к объекту класса.
- Вызываются через экземпляр: `object.method()`.
- Имеют доступ к полям экземпляра через `this`.

```java
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
}
Calculator calc = new Calculator();
int sum = calc.add(2, 3); // 5
```
### Статические методы
- Принадлежат классу, а не объекту.
- Вызываются через имя класса: `ClassName.method()`.
- Не имеют доступа к `this`.

```java
class MathUtils {
    static int multiply(int a, int b) {
        return a * b;
    }
}
int product = MathUtils.multiply(4, 5); // 20
```
### Абстрактные методы
- Объявлены без реализации в абстрактных классах или интерфейсах.
- Должны быть переопределены в подклассах.

```java
abstract class Shape {
    abstract double area();
}
class Circle extends Shape {
    double radius;
    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}
```
### Методы по умолчанию (default, Java 8+)
- Имеют реализацию в интерфейсах.
- Позволяют добавлять функциональность без нарушения существующих реализаций.

```java
interface Logger {
    default void log(String message) {
        System.out.println("Log: " + message);
    }
}
class MyClass implements Logger {
    // log доступен без переопределения
}
```
### Статические методы в интерфейсах (Java 8+)
- Принадлежат интерфейсу, вызываются через имя интерфейса.

```java
interface MathOps {
    static int max(int a, int b) {
        return a > b ? a : b;
    }
}
int max = MathOps.max(10, 20); // 20
```
## Передача параметров

- **По значению**: Java всегда передаёт параметры по значению, включая ссылки на объекты.
- **Примитивы**: Передаётся копия значения.
- **Объекты**: Передаётся копия ссылки, что позволяет изменять содержимое объекта, но не саму ссылку.

```java
class Example {
    void modify(int num, StringBuilder sb) {
        num = 100; // Не влияет на внешнюю переменную
        sb.append("World"); // Изменяет объект
    }
}
StringBuilder sb = new StringBuilder("Hello");
int x = 10;
new Example().modify(x, sb);
System.out.println(x + ", " + sb); // 10, HelloWorld
```
> Для примитивов — всегда копия значения. Для объектов — копия ссылки, но объект можно менять внутри метода.
## Перегрузка методов (Overloading)

- Определение нескольких методов с одинаковым именем, но разными сигнатурами (количество, типы или порядок параметров).
- Возвращаемый тип **не влияет** на перегрузку.
- Модификаторы доступа не влияют на перегрузку.

```java
class OverloadExample {
    public void print() {
        System.out.println("Без параметров");
    }
    public void print(String message) {
        System.out.println("Строка: " + message);
    }
    public void print(int number) {
        System.out.println("Число: " + number);
    }
}
OverloadExample ex = new OverloadExample();
ex.print();         // Без параметров
ex.print("Hello");  // Строка: Hello
ex.print(42);       // Число: 42
```
### Автобоксинг и перегрузка

 `Автобоксинг может привести к неоднозначности`
 
> Если у вас есть перегруженные методы с параметрами-примитивами и обёртками, компилятор иногда не может однозначно выбрать, какой из методов вызвать.
> 
> Избегайте перегрузки, где различие только между примитивом и обёрткой, чтобы не было путаницы и ошибок компиляции.

```java
class Ambiguity {
    void method(int x) { System.out.println("int"); }
    void method(Integer x) { System.out.println("Integer"); }
}
new Ambiguity().method(5); // int (примитив предпочтительнее)
```
## Переопределение методов (Overriding)

`Подкласс предоставляет свою реализацию метода суперкласса или интерфейса`
### Правила переопределения

1. **Сигнатура**: Имя и параметры должны совпадать.
2. **Тип возврата**: Допустим подтип оригинального типа.
3. **Доступ**: Нельзя уменьшать видимость (например, public → private).
4. **Исключения**: Запрещены новые или более общие проверяемые исключения.
5. **Аннотация**: Используйте @Override для проверки.
6. **Ограничения**:
    - final методы не переопределяются.
    - Статические методы скрываются, а не переопределяются.

```java
class Animal {
    public void sound() {
        System.out.println("Some sound");
    }
}
class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Bark");
    }
}
Animal dog = new Dog();
dog.sound(); // Bark
```

### Ковариантный возвращаемый тип

> Ковариантный возвращаемый тип — это возможность в переопределённом методе возвращать не точно тот же тип, что и в базовом классе, а его подтип (более конкретный тип).
>
> Позволяет сделать API удобнее: если вы переопределяете метод и точно знаете, что возвращаете более конкретный тип, вы можете это явно указать, и не придётся делать лишние приведения типов.
> 
> В базовом классе метод возвращает "животное" (Animal), а в подклассе — "собаку" (Dog). Собака — это тоже животное, но более конкретно.
>
```java
class Animal {
    Animal getInstance() {
        return this;
    }
}
class Dog extends Animal {
    @Override
    Dog getInstance() { // Ковариантный тип
        return this;
    }
}
```
> Теперь при вызове getInstance() у Dog вы сразу получаете Dog, а не просто Animal.

> [!WARNING] Ловушка: Method Hiding (скрытие статических методов)
> Статические методы **не переопределяются** — они **скрываются**. При вызове через переменную суперкласса (`Parent p = new Child(); p.greet()`) выполнится метод **Parent**, а не Child. Это ломает ожидаемый полиморфизм.

### Скрытие статических методов (Method Hiding)

`Статические методы не переопределяются, а скрываются`

```java
class Parent {
    static void greet() {
        System.out.println("Hello from Parent");
    }
}
class Child extends Parent {
    static void greet() {
        System.out.println("Hello from Child");
    }
}
Parent p = new Child();
p.greet(); // Hello from Parent
Child.greet(); // Hello from Child
```

## 6. Лямбда-выражения и ссылки на методы (Java 8+)

> [!INFO] Лямбды под капотом
> Лямбда-выражение реализует функциональный интерфейс, но **не создаёт отдельный `.class`-файл**. JVM использует `invokedynamic` + `LambdaMetafactory` для генерации реализации на лету. Это быстрее и компактнее анонимных классов.
>
> **Лямбда-выражения — это не анонимные классы!**
> - Лямбда реализует функциональный интерфейс, но не создаёт отдельный .class-файл, как анонимный класс.
> - JVM использует invokedynamic и LambdaMetafactory для генерации нужной реализации интерфейса на лету.
> - Лямбда быстрее и компактнее, не имеет собственного this и не может иметь нестатических полей.

| Лямбда-выражение                | Анонимный класс                |
|----------------------------------|-------------------------------|
| Реализует функциональный интерфейс| Реализует любой интерфейс/класс|
| Не создаёт отдельный .class-файл | Создаёт отдельный .class-файл  |
| Нет собственного this            | Есть собственный this          |
| Быстрее и компактнее             | Медленнее, больше памяти       |
| Не может иметь нестатических полей| Может иметь свои поля          |
- Лямбда-выражения позволяют передавать поведение (функции) как параметры, делают код компактнее и удобнее для работы с коллекциями и потоками.
- Лямбда реализует функциональный интерфейс (интерфейс с одним абстрактным методом).

 **Синтаксис:**
```java
(параметры) -> выражение;
(параметры) -> { блок кода; return значение; };
```

```java
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
names.stream().filter(s -> s.startsWith("A")).forEach(System.out::println); // Alice
```

- Лямбда-выражения короче и читаемее, чем анонимные классы, но не имеют собственного this и не могут иметь нестатических полей.

 **Сравнение:**
```java
Runnable r1 = () -> System.out.println("lambda");
Runnable r2 = new Runnable() {
    public void run() { System.out.println("anon"); }
};
```

- Основные стандартные интерфейсы (java.util.function):
    - `Function<T, R>` — принимает T, возвращает R
    - `Consumer<T>` — принимает T, ничего не возвращает
    - `Predicate<T>` — принимает T, возвращает boolean
    - `Supplier<T>` — не принимает параметров, возвращает T

- Ссылки на методы — компактная альтернатива лямбдам для вызова существующих методов:
    - `ClassName::staticMethod`
    - `instance::instanceMethod`
    - `ClassName::instanceMethod`
    - `ClassName::new`
