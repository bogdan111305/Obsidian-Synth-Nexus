В современной Java (версии 10–22) акцент сместился на **читаемость кода** и **снижение когнитивной нагрузки**. Инструменты `var`, **Text Blocks** и **Unnamed Variables** позволяют убрать визуальный шум, оставляя на виду только суть алгоритма.
## `var`: Локальный вывод типов (Java 10+)

`var` — это не динамическая типизация (как в JS), а **синтаксический сахар** для статической типизации. Компилятор сам смотрит на правую часть выражения и подставляет нужный тип в байт-код.

### Где разрешен и запрещен `var`

|**Можно ✅**|**Нельзя ❌**|
|---|---|
|Локальные переменные внутри методов|Поля класса (instance/static variables)|
|Переменные цикла `for` и `for-each`|Параметры методов|
|Ресурс в `try-with-resources`|Возвращаемые типы методов|
|Параметры лямбда-выражений (Java 11+)|Инициализация `null` или без инициализации|

### Особенности вывода типов

Если вы пишете `var list = new ArrayList<>();`, компилятор выведет тип `ArrayList<Object>`. Чтобы сохранить типизацию, нужно указывать тип справа: `var list = new ArrayList<String>();`.

> **Важно:** `var` — это не ключевое слово, а «зарезервированное имя типа». Это значит, что код `int var = 5;` технически скомпилируется (для обратной совместимости), но делать так — плохой тон.

## Text Blocks: Многострочные строки (Java 15+)

Текстовые блоки решают проблему «лесенки» из плюсов и символов `\n`, делая JSON, SQL и HTML внутри Java-кода читаемыми.

### Правила оформления

1. **Открывающие кавычки:** Должны заканчиваться переводом строки. Текст не может начинаться на той же строке, что и `"""`.
    
2. **Управление отступами (Incidental Whitespace):** Компилятор автоматически убирает «лишние» общие отступы слева, ориентируясь на самую левую позицию текста или закрывающих кавычек `"""`.

### Магия экранирования

- `\"` больше не нужно экранировать.
    
- `\` в конце строки — соединяет текущую строку со следующей (удаляет перенос).
    
- `\s` — гарантирует сохранение пробела (по умолчанию trailing spaces в конце строк текстового блока обрезаются).

## Unnamed Variables и Patterns (Java 22+)

Символ подчеркивания `_` официально стал «безымянной переменной». Это способ сказать компилятору и коллегам: «Мне нужно здесь что-то объявить по правилам синтаксиса, но я не собираюсь это использовать».

### Примеры использования

- **В блоках catch:** `catch (Exception _)` — если нам важен сам факт ошибки, а не её детали.
    
- **В циклах:** `for (var _ : list)` — если нужно просто выполнить действие N раз.
    
- **В деструктуризации Records:**

```java
if (obj instanceof Order(var id, _, var status)) { 
    // Нам важны только id и статус, остальное (_) игнорируем
}
```

## Резюме: Правила «хорошего тона»

1. **Принцип `var`:** Используйте его только тогда, когда тип **очевиден** из правой части (например, `var user = new User()`). Не используйте `var`, если справа стоит вызов метода с неясным возвращаемым типом (`var result = service.process()`).

2. **Форматирование Text Blocks:** Положение закрывающих кавычек `"""` имеет значение. Если сдвинуть их левее текста, у всего блока появится отступ.

3. **Unnamed `_`:** Используйте его всегда, когда переменная не используется. Это исключает предупреждения компилятора "Variable is never used" и делает код чище.

## Вопросы на интервью (Interview Prep)

1. **Является ли `var` ключевым словом?** (Ответ: Нет, это зарезервированное имя типа).
    
2. **Можно ли объявить `var x;` а потом присвоить значение?** (Ответ: Нет, инициализация обязательна сразу).
    
3. **Как Text Blocks обрабатывают переносы строк Windows (CRLF) vs Unix (LF)?** (Ответ: Компилятор нормализует все переносы к `LF`).
    
4. **В чем преимущество `_` перед обычным именем переменной `unused`?** (Ответ: `_` не создает привязки в памяти, позволяет объявлять несколько таких переменных в одной области видимости и явно выражает намерение игнорировать данные).
