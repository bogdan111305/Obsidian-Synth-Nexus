## Примитивные типы Java (главное)

### Восемь примитивных типов

| Группа | Тип | Размер | Диапазон | Значение по умолчанию |
|--------|-----|--------|----------|----------------------|
| **Целые** | `byte` | 8 бит | -128 .. 127 | 0 |
| | `short` | 16 бит | -32 768 .. 32 767 | 0 |
| | `int` | 32 бита | -2³¹ .. 2³¹-1 (~ ±2 млрд) | 0 |
| | `long` | 64 бита | -2⁶³ .. 2⁶³-1 | 0L |
| **С плавающей точкой** | `float` | 32 бита | ~±3.4×10³⁸ (6‑7 знаков) | 0.0f |
| | `double` | 64 бита | ~±1.8×10³⁰⁸ (15‑16 знаков) | 0.0d |
| **Символ** | `char` | 16 бит | 0 .. 65 535 (Unicode) | `'\u0000'` |
| **Логический** | `boolean` | JVM‑зависим (логически 1 бит) | `true` / `false` | `false` |

> Размеры фиксированы и не зависят от платформы.

### Особенности каждого типа

- **`boolean`**  
  В JVM обычно занимает 1 байт (в массивах — тоже 1 байт на элемент). Нельзя привести к `int` или наоборот.

- **`char`**  
  Единственный беззнаковый тип. Хранит Unicode в UTF-16. Символы вне BMP (эмодзи) занимают два `char` (суррогатная пара).

- **`float` / `double` (IEEE 754)**  
  **Ограниченная точность** — не сравнивайте через `==`:
  ```java
  0.1 + 0.2 == 0.3; // false
  ```
  Для точных расчётов (деньги) — `BigDecimal`.

  **Специальные значения:**  
  - `NaN` (Not a Number) — результат 0.0/0.0. Проверка: `Double.isNaN(x)`, не `x == Double.NaN`.  
  - `±Infinity` — результат 1.0/0.0.

- **Целые типы (`byte`, `short`, `int`, `long`)**  
  Хранятся в **дополнительном коде** (two’s complement):  
  - Отрицательное число = инверсия битов положительного + 1.  
  - Нет «двух нулей».  
  - Арифметика зациклена: переполнение перебрасывает на противоположный край (`127 + 1 = -128` для `byte`).  

  **Деление и остаток:**  
  - Целочисленное деление отбрасывает дробную часть (`5/2 = 2`).  
  - Остаток `%` может быть отрицательным: `-5 % 2 = -1` (знак делимого).

### Приведение типов

- **Неявное (widening)** — автоматически от меньшего к большему:  
  `byte → short → int → long → float → double`, `char → int`. Без потери данных.
- **Явное (narrowing)** — вручную `(тип)`. Потеря дробной части или переполнение.
  ```java
  double d = 9.78;
  int i = (int) d;  // 9
  ```
- `boolean` не приводится ни к чему (даже к `int`).

### Автобоксинг, анбоксинг и кэширование

- **Автобоксинг** — примитив → обёртка (`int → Integer`). Вызывает `Integer.valueOf()`.  
- **Анбоксинг** — обёртка → примитив (`Integer → int`). Вызывает `intValue()`.

```java
int x = 5;
Integer boxed = x;      // автобоксинг
int y = boxed;          // анбоксинг
```

**Кэширование обёрток:**  
Для `Integer`, `Byte`, `Short`, `Long`, `Character` кэшируются значения **-128..127**.  
```java
Integer a = 100, b = 100;
a == b;   // true (один объект из кэша)

Integer c = 1000, d = 1000;
c == d;   // false (разные объекты)
```

**Производительность:**  
Каждый автобоксинг создаёт объект в куче. В циклах это тормозит и нагружает GC.  
Используйте примитивы или `IntStream.range` (без боксинга).

### Битовые операции и сдвиги (часто в вопросах)

| Операция | Что делает | Пример (`int a = 5;` 0101) |
|----------|------------|----------------------------|
| `&` | И | `5 & 3` = 1 (0101 & 0011 = 0001) |
| `\|` | ИЛИ | `5 \| 3` = 7 (0101 \| 0011 = 0111) |
| `^` | XOR | `5 ^ 3` = 6 (0101 ^ 0011 = 0110) |
| `~` | НЕ (инверсия) | `~5` = -6 (все биты инвертированы) |
| `<<` | сдвиг влево (умножение на 2ⁿ) | `5 << 1` = 10 |
| `>>` | сдвиг вправо (знаковый) | `-5 >> 1` = -3 (знак сохраняется) |
| `>>>` | сдвиг вправо без знака (заполняет 0) | `-5 >>> 1` = 2147483645 |

> **Разница `>>` и `>>>`:** `>>` сохраняет знаковый бит (для отрицательных чисел заполняет 1), `>>>` всегда заполняет 0.

### Что ещё важно (из вопросов собеседования)

- **Литералы:**  
  `int` = `10`; `long` = `10L`; `float` = `10.0f`; `double` = `10.0d`;  
  шестнадцатеричные: `0xFF`; двоичные: `0b1010`; восьмеричные (редко): `012`.

- **Переполнение не вызывает ошибок** — проверяйте вручную:
  ```java
  int a = Integer.MAX_VALUE + 1; // -2147483648
  ```

- **`BigDecimal`** — для точных денежных расчётов (не используйте `float`/`double`).

- **Округление:** `Math.round()` для `float`/`double`, `Math.floor()`/`ceil()`.

- **`instanceof` с примитивами нельзя** — только с объектами. Обёртки работают: `Integer i = 5; i instanceof Integer` → `true`.

- **Тернарный оператор `? :`** с примитивами и обёртками может приводить к анбоксингу/автобоксингу с неожиданностями.

- **Многопоточность:**  
  Для примитивов нет проблем видимости без `volatile` (кроме `long` и `double`, которые неатомарны на 32-битных JVM — используйте `volatile` или атомики).

## Переменные

> **Переменная** — именованный контейнер, хранящий значение определённого типа (примитив или ссылку).

### Виды переменных и их хранение

| Вид | Где хранится | Значение по умолчанию | Время жизни |
|-----|--------------|----------------------|-------------|
| **Локальная** | Стек потока | ❌ Нет (ошибка компиляции без инициализации) | До выхода из блока |
| **Поле экземпляра** | Куча (внутри объекта) | ✅ `0` / `null` / `false` | Пока жив объект |
| **Статическое поле** | Metaspace (Java 8+) | ✅ `0` / `null` / `false` | Пока загружен класс |

```java
class Example {
    int instanceVar;       // 0
    static int staticVar;  // 0
    void method() {
        int localVar;      // не инициализирована → ошибка
    }
}
```

### Примитивные vs ссылочные переменные

- **Примитивная** хранит значение непосредственно (само число, символ, логическое значение).
- **Ссылочная** хранит адрес (ссылку) на объект в куче. Значение по умолчанию — `null`.

```java
int a = 10;           // примитив
String s = "Hello";   // ссылка
```

### Передача в методы (pass‑by‑value)

Java всегда передаёт **копию значения**:
- Для примитивов — копию самого числа.
- Для ссылок — копию ссылки (объект остаётся тот же).

```java
void method(int x) { x = 20; }          // не меняет переменную снаружи
void method(StringBuilder sb) { sb.append("!"); } // меняет объект
void method(StringBuilder sb) { sb = new StringBuilder(); } // не меняет внешнюю ссылку
```

### Effectively final

Переменная, которая не меняется после инициализации (даже без `final`).  
Лямбды и анонимные классы могут захватывать только такие переменные.

```java
String msg = "hello";
// msg = "bye";   // раскомментировать → ошибка
Runnable r = () -> System.out.println(msg); // OK
```

**Почему:** лямбда «запоминает» значение при создании. Если переменная изменится позже — неясно, какое брать. Плюс лямбда может работать в другом потоке → гонка данных.

### Shadowing (затенение)

Внутренняя область видимости (параметр, локальная) «затеняет» внешнюю переменную с тем же именем.

```java
class Foo {
    int x = 10;
    void method(int x) {
        System.out.println(x);      // параметр
        System.out.println(this.x); // поле
    }
}
```

- Доступ к затенённому полю — через `this`.
- В одной области видимости (например, внутри метода) нельзя объявить две переменные с одинаковым именем — это ошибка компиляции, а не shadowing.

### Модификаторы переменных

| Модификатор | Что делает |
|-------------|------------|
| `final` | Ссылку нельзя переприсвоить. Для примитивов — значение фиксировано. Для ссылок — объект можно изменить. |
| `volatile` | Обеспечивает видимость изменений между потоками, но **не атомарность** (например, `volatile int count; count++` не атомарен). |
| `transient` | Исключает поле из стандартной Java‑сериализации. |

```java
final List<String> list = new ArrayList<>();
list.add("OK");        // разрешено
list = new ArrayList(); // ошибка

volatile boolean flag = false;
transient String pwd = "secret";
```

### Модификаторы доступа (для полей)

| Модификатор | Доступ в классе | Доступ в пакете | Доступ в подклассе (другой пакет) | Доступ везде |
|-------------|----------------|----------------|----------------------------------|--------------|
| `private`   | ✅ | ❌ | ❌ | ❌ |
| _package‑private_ (без модификатора) | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ (через наследование) | ❌ |
| `public`    | ✅ | ✅ | ✅ | ✅ |

> Начиная с Java 9, модули могут дополнительно ограничивать экспорт пакетов.

### Дополнительные важные моменты (из собеседований)

- **`var`** — локальная переменная с выводом типа (Java 10+). Тип определяется компилятором по инициализатору.
  ```java
  var list = new ArrayList<String>(); // тип ArrayList<String>
  // var x; // ошибка — нужна инициализация
  ```

- **`ThreadLocal`** — переменная, имеющая своё значение для каждого потока.
  ```java
  ThreadLocal<Integer> threadLocal = ThreadLocal.withInitial(() -> 0);
  ```

- **Pattern matching для `instanceof`** — можно сразу объявить переменную нужного типа.
  ```java
  if (obj instanceof String s) {
      System.out.println(s.length()); // s уже String
  }
  ```

- **Константы** — `static final` поля с именами в UPPER_CASE. Компилятор может встроить их значение.
  ```java
  static final double PI = 3.14159;
  ```

## Массивы

### Что такое массив

- **Объект** в куче (heap), хранит фиксированное количество элементов одного типа.
- Переменная массива — **ссылка** в стеке (может быть `null`).
- Длина задаётся при создании и не меняется (`arr.length` — поле, не метод).

```java
int[] a = new int[5];          // элементы = 0
int[] b = {10, 20, 30};        // короткая запись
String[] s = new String[3];    // элементы = null
```

### Доступ и перебор

- Индексы от `0` до `length-1`. Выход за границы → `ArrayIndexOutOfBoundsException`.
- Перебор: `for` по индексу или `for-each` (только чтение).

```java
for (int i = 0; i < arr.length; i++) { ... }
for (int x : arr) { ... }
```

### Многомерные и зубчатые (jagged)

- Двумерный массив = массив массивов. Строки могут быть разной длины.

```java
int[][] matrix = new int[3][4];        // прямоугольный 3x4
int[][] jagged = new int[3][];          // строки разной длины
jagged[0] = new int[2];
jagged[1] = new int[5];
```

### Полезные методы `java.util.Arrays`

| Операция | Пример | Примечание |
|----------|--------|-------------|
| Сортировка | `Arrays.sort(arr)` | Для примитивов — Dual-Pivot Quicksort, для объектов — Timsort |
| Поиск | `Arrays.binarySearch(sortedArr, key)` | Массив **должен быть отсортирован**, иначе неверный результат |
| Копирование | `Arrays.copyOf(arr, newLen)` | Обрезает или дополняет нулями/`null` |
| Копирование части | `Arrays.copyOfRange(arr, from, to)` | `to` не включается |
| Заполнение | `Arrays.fill(arr, value)` | |
| Сравнение | `Arrays.equals(a, b)` | Поэлементно |
| Глубокое сравнение | `Arrays.deepEquals(a2d, b2d)` | Для многомерных |
| Строковое представление | `Arrays.toString(arr)` | `"[1, 2, 3]"` |
| Глубокий вывод | `Arrays.deepToString(arr2d)` | Для многомерных |

### Копирование массивов

- `clone()`, `Arrays.copyOf()`, `System.arraycopy()` (самый быстрый, нативны).
- Для **примитивов** — глубокая копия (значения).
- Для **ссылочных типов** — поверхностная копия (копируются ссылки, объекты общие).

```java
Person[] people = {new Person("Alice")};
Person[] copy = people.clone();
copy[0].setName("Bob");   // меняет того же Person → people[0] тоже изменился
```

### Ковариантность массивов (ловушка)

- Массивы ковариантны: `String[]` можно присвоить `Object[]`.
- Это приводит к ошибке времени выполнения: `ArrayStoreException`.

```java
String[] strings = new String[3];
Object[] objects = strings;   // компилируется
objects[0] = 42;              // компилируется, но в рантайме ArrayStoreException
```

> **Важно:** Обобщения (`List<String>`) **не ковариантны** — такую ошибку ловит компилятор.

### Производительность (Big‑O)

| Операция | Сложность |
|----------|-----------|
| Чтение/запись по индексу | O(1) |
| Поиск значения | O(n) |
| Вставка/удаление (кроме конца) | O(n) |
| Копирование | O(n) |
| Сортировка | O(n log n) |

### Важные дополнения (часто забывают)

- **`System.arraycopy(src, srcPos, dest, destPos, length)`** — самый эффективный способ копирования.
- **`Arrays.asList(arr)`** — возвращает `List` фиксированного размера, **не работает с примитивами** (для `int[]` вернёт `List<int[]>`).
- **Varargs** — это скрытый массив:
  ```java
  void method(String... args) { // args — массив String[]
  ```
- **Массивы и коллекции**:
  - Массивы хранят примитивы эффективнее, но не умеют динамически менять размер.
  - `ArrayList` внутри использует массив и автоматически расширяется.
- **Массив объектов** изначально заполнен `null` — обращение к элементу без проверки → `NullPointerException`.
- **Многомерные массивы в памяти** — массив массивов, строки могут быть независимыми объектами.

## Управляющие конструкции (главное)

### `if-else` и логические операторы

```java
if (условие) { ... } 
else if (другое) { ... } 
else { ... }
```

**Логические операторы**:
  `&&` и `||` — **short‑circuit** (левая часть определяет результат, правая не вычисляется).
### Тернарный оператор

```java
результат = (условие) ? значение1 : значение2;
```

- Только для простых присваиваний. Не злоупотребляйте вложенностью.
- Оба выражения должны быть совместимы по типу.

### 3. `switch` — классический (Java ≤ 8)

```java
switch (выражение) {
    case 1:
        // код
        break;
    case 2:
        // код
        break;
    default:
        // код
}
```

- **Поддерживаемые типы:** `byte`, `short`, `int`, `char`, `String` (с Java 7), `enum`, а также их обёртки (автобоксинг).
- **`break`** обязателен, иначе **fall‑through** (проваливание) — выполнение продолжается в следующий `case`.
- **`default`** — необязательно, выполняется при отсутствии совпадений.

```java
int day = 1;
switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        System.out.println("Будний день");
        break;
    default:
        System.out.println("Выходной");
}
```

### 4. Современный `switch` (Java 12–17+)

#### a) Стрелочный синтаксис (без `break`, без fall‑through)
```java
switch (day) {
    case 1, 2, 3, 4, 5 -> System.out.println("Будний день");
    case 6, 7 -> System.out.println("Выходной");
}
```

#### b) `switch` как выражение (Java 14+)
```java
String result = switch (day) {
    case 1, 2, 3, 4, 5 -> "Будний";
    case 6, 7 -> "Выходной";
    default -> "Неизвестно";
};
```

#### c) `yield` для возврата из блока
```java
int days = switch (month) {
    case 2 -> 28;
    case 4, 6, 9, 11 -> 30;
    default -> {
        int d = 31;
        yield d;
    }
};
```

#### d) Pattern matching в `switch` (Java 17+ / 21 без preview)
```java
Object obj = "Hello";
String msg = switch (obj) {
    case String s -> "Строка: " + s;
    case Integer i -> "Число: " + i;
    case null -> "null";
    default -> "Неизвестно";
};
```
> **Важно:** порядок `case` важен (более конкретные типы раньше общих). `null` можно обработать явно.

### 5. Внутренняя реализация JVM и производительность

| Конструкция | Байт-код | Сложность |
|-------------|----------|------------|
| `if-else` | Условные переходы (`if_icmpeq`, `if_icmpgt` и т.д.) | O(1) на условие, но цепочка O(n) |
| `switch` (плотные значения) | `tableswitch` — таблица переходов | O(1) |
| `switch` (разреженные значения) | `lookupswitch` — бинарный поиск по ключам | O(log n) |

- **Вывод:** `switch` на многих `case` эффективнее цепочки `if-else`.
- Современный `switch`-выражение генерирует более компактный байт-код без fall‑through.

### 6. Что ещё важно (из собеседований)

- **Short‑circuit:** в `if (a != null && a.equals(b))` второе условие не вызовет NPE.
- **Guard clauses** (защитные проверки) — упрощают вложенность:
  ```java
  if (age < 18) return "Доступ запрещён";
  // дальше логика для взрослых
  ```
- **`switch` с `enum`** — компилятор проверяет полноту охвата (если нет `default`, должны быть все значения).
- **`switch (null)`** — раньше бросал `NullPointerException`. В pattern matching можно обработать явно.
- **Условная компиляция** не поддерживается напрямую (в отличие от C/C++). Используйте `static final boolean DEBUG = false; if (DEBUG) { ... }` — такой код вырезается компилятором(недостижимый код).
- **Многопоточность:** условные операторы не атомарны. Проверка и действие должны быть синхронизированы (например, `if (!map.containsKey(key)) map.put(key, value)` — не потокобезопасно).

## Циклы и рекурсия (главное)

### 1. Виды циклов

| Цикл | Когда использовать | Особенность |
|------|--------------------|--------------|
| `for` | Количество итераций известно или легко вычислимо | Инициализация, условие, шаг — в одной строке |
| `while` | Количество итераций неизвестно, проверка в начале | Может не выполниться ни разу |
| `do-while` | Тело нужно выполнить хотя бы один раз | Проверка после итерации |
| `for-each` | Перебор всех элементов массива или `Iterable` | Не даёт доступа к индексу, нельзя удалять |

```java
// for
for (int i = 0; i < 5; i++) { ... }

// while
while (условие) { ... }

// do-while
do { ... } while (условие);

// for-each
for (String s : list) { ... }
```

### 2. `break` и `continue`, метки

- `break` — выход из цикла.
- `continue` — переход к следующей итерации.
- **Метки** — для выхода из вложенных циклов.

```java
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break outer; // выход из внешнего цикла
    }
}
```

### 3. `for-each` под капотом

- Для массивов — обычный индексный цикл.
- Для `Iterable` (коллекции) — превращается в `while` с итератором:

```java
for (String s : list) { ... }
// эквивалентно:
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    ...
}
```

**Ограничения `for-each`:**
- Нельзя удалять элементы (кроме `it.remove()` через явный итератор, но в `for-each` его нет) → `ConcurrentModificationException`.
- Нельзя модифицировать коллекцию во время обхода (кроме потокобезопасных или через `removeIf`).
- Нет доступа к индексу.

```java
// Удаление через for-each — ошибка
for (String s : list) {
    if (s.isEmpty()) list.remove(s); // ConcurrentModificationException
}
// Правильно:
list.removeIf(String::isEmpty);
```

### 4. Stream API и функциональная итерация

```java
list.stream().forEach(s -> System.out.println(s));
list.parallelStream().forEach(s -> ...); // параллельно, порядок не гарантирован
```

- `forEach` в стримах — терминальная операция.
- Для простых циклов на небольших коллекциях обычный `for` быстрее.
- Stream удобен для цепочек `filter`/`map`/`reduce`.

### 5. Рекурсия

- Метод вызывает сам себя.
- **Обязательно:** условие выхода (base case), иначе `StackOverflowError`.

```java
long factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

**Проблемы рекурсии в Java:**
- **Нет оптимизации хвостовой рекурсии** (в отличие от Scala/Kotlin/Scheme). Глубокая рекурсия (тысячи вызовов) → переполнение стека (обычно ~512 КБ на поток).
- Каждый вызов — новый фрейм в стеке.

**Решение:** итерация с явным стеком (`Deque`, `Stack`) или преобразование в цикл.

```java
// Обход дерева без рекурсии
Deque<Node> stack = new ArrayDeque<>();
stack.push(root);
while (!stack.isEmpty()) {
    Node node = stack.pop();
    // обработка
    if (node.right != null) stack.push(node.right);
    if (node.left != null) stack.push(node.left);
}
```

**Мемоизация** (кэширование результатов) — для рекурсии с перекрывающимися подзадачами (Фибоначчи, ДП):

```java
Map<Integer, Long> cache = new HashMap<>();
long fib(int n) {
    if (n <= 1) return n;
    if (cache.containsKey(n)) return cache.get(n);
    long res = fib(n-1) + fib(n-2);
    cache.put(n, res);
    return res;
}
```

### 6. Производительность и внутренняя реализация

| Конструкция | Байт-код / реализация | Примечание |
|-------------|------------------------|-------------|
| `for` / `while` | Условные переходы (`goto` в байт-коде) | JIT оптимизирует |
| `for-each` (массив) | Индексный цикл | Эффективно |
| `for-each` (коллекция) | Вызов `iterator()` и `hasNext()`/`next()` | Немного медленнее, но удобно |
| `stream().forEach()` | `Spliterator` + лямбда | Для больших данных, параллельная обработка |
| Рекурсия | Фреймы в стеке, нет TCO | Ограниченная глубина |

### 7. Дополнительные важные моменты (из собеседований)

- **Бесконечный цикл:** `for(;;) { }` или `while(true) { }`. Остановка — `break` или `return`.
- **Вложенные циклы:** сложность O(n*m), следите за производительностью.
- **`Iterator` и `Iterable`:** любой класс, реализующий `Iterable`, можно использовать в `for-each`.
- **Обработка исключений:** если в цикле выброшено исключение, цикл прерывается (если не перехвачено внутри).
- **Многопоточность:** при обходе коллекции, которая изменяется из другого потока, может быть `ConcurrentModificationException` (даже вне `for-each`). Используйте `CopyOnWriteArrayList` или синхронизацию.
- **`yield` в циклах?** Нет, `yield` — только для `switch`-выражений (Java 14+).
- **Рекурсия vs итерация:** итерация всегда предпочтительнее для больших глубин, рекурсия — для естественно рекурсивных структур (деревья, графы) при небольшой глубине.
