# Модификаторы доступа, пакеты и импорты

> `private` → только класс; package-private → пакет; `protected` → пакет + подклассы; `public` → везде. JPMS (Java 9+) добавляет module-level visibility поверх этих правил — `public` без `exports` не виден снаружи модуля.

## Модификаторы доступа

| Модификатор | В классе | В пакете | В подклассах | Вне пакета |
|---|---|---|---|---|
| `private` | ✅ | ❌ | ❌ | ❌ |
| package-private | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ (только через наследование) |
| `public` | ✅ | ✅ | ✅ | ✅ |

```java
public class Example {
    private int a;    // только этот класс
    int b;            // package-private
    protected int c;  // пакет + наследники
    public int d;     // везде
}
```

### Нюанс `protected` — только через тип подкласса

Доступ к `protected` члену из другого пакета возможен **только через ссылку типа подкласса**, не через ссылку суперкласса:

```java
// Пакет com.base:
public class Base {
    protected int value = 42;
}

// Пакет com.derived:
public class Derived extends Base {
    void method() {
        System.out.println(value);      // OK: через this (Derived)
        System.out.println(this.value); // OK

        Base b = new Base();
        System.out.println(b.value);   // COMPILE ERROR — другой пакет + не через подкласс
    }
}
```

### Модификаторы для классов верхнего уровня

Классы верхнего уровня (не nested) могут быть только `public` или package-private:
- `private class Foo {}` — compile error для top-level
- `protected class Foo {}` — compile error для top-level

Для nested/inner классов доступны все четыре.

## JPMS — Module System (Java 9+)

JPMS добавляет модульный уровень видимости поверх пакетных правил:

```
module com.myapp {
    requires com.lib;          // зависимость
    exports com.myapp.api;     // пакет виден всем
    exports com.myapp.internal to com.test; // только для com.test
    opens com.myapp.dto;       // для Reflection (runtime)
    opens com.myapp.dto to com.jackson; // только для Jackson
}
```

**Ключевые понятия:**
- `exports` — пакет доступен при compile-time и runtime
- `opens` — пакет доступен через Reflection в runtime (даже приватные члены)
- `requires transitive` — транзитивная зависимость (нужна, если ты реэкспортируешь типы из другого модуля в своём API)

**Эффект на `public`:**
```java
// Без JPMS: public = везде
// С JPMS: public в неэкспортированном пакете = невидим снаружи модуля

// module-info.java не содержит: exports com.myapp.internal
// → com.myapp.internal.Helper — compile error из другого модуля,
//   даже если Helper.method() объявлен public!
```

## Пакеты

```java
// Объявление — первая строка файла:
package com.company.project.module;

// Структура директорий должна совпадать:
// src/com/company/project/module/MyClass.java
```

**Конвенция именования:** reverse domain name в нижнем регистре:
- `com.google.guava.collect`
- `org.springframework.boot.autoconfigure`
- Не использовать: `utils`, `common`, `misc` как корень — слишком широко

**Default package** (без `package`):
- Класс нельзя импортировать из другого пакета
- Нельзя использовать в модульной системе
- Только для quick experiments — не для production

## Импорт

```java
import java.util.List;                  // точечный импорт (предпочтительно)
import java.util.*;                     // wildcard — все классы пакета (но не подпакетов!)
import static java.lang.Math.sqrt;     // статический импорт метода
import static java.lang.Math.*;        // все static-члены

// java.lang импортируется автоматически: String, Integer, Math и т.д.
```

**Wildcard `*` не включает подпакеты:**
```java
import java.util.*;   // List, Map, ArrayList — да
                      // java.util.concurrent.ConcurrentHashMap — нет, нужен отдельный import
```

**Конфликт имён:**
```java
import java.util.Date;
import java.sql.Date;  // compile error — неоднозначность!

// Решение: одно из имён используем полностью:
import java.util.Date;
// java.sql.Date используем как java.sql.Date везде в коде
```

## Подводные камни

- **`protected` ≠ "доступен в подклассе из любого места"** — из другого пакета доступ только через ссылку типа подкласса (`this`/`super`), не через ссылку суперкласса.
- **`public` в модуле без `exports`** — в JPMS public-член закрытого пакета недоступен снаружи модуля. Reflection без `opens` тоже бросит `InaccessibleObjectException`.
- **Wildcard `*` ≠ подпакеты** — `import java.util.*` не включает `java.util.concurrent.*`.
- **Default package нельзя импортировать** — класс в default package недоступен из любого именованного пакета.
- **Constant Interface Antipattern** — хранить константы в интерфейсе (`interface Constants { int MAX = 100; }`) — антипаттерн: засоряет namespace реализующих классов. Используй `final class Constants` или `enum`.
- **`private` static-метод в интерфейсе** (Java 9+) и внутренних классах не путать: `private` в interface видим только внутри того же интерфейса, в том числе в `default`-методах.
