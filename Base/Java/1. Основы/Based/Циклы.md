> **Циклы** позволяют выполнять повторяющиеся действия, пока выполняется условие. В Java есть циклы `while`, `do-while`, `for`, `for-each`, а также функциональные циклы через Stream API. Для рекурсивных задач используется рекурсия.
## Цикл `while` и `do-while`

### Цикл `while`
- Выполняет блок кода, пока условие истинно (`true`).
- Подходит для случаев, когда количество итераций заранее неизвестно.

```java
int i = 0;
while (i < 5) {
    System.out.println("i = " + i);
    i++;
}
```
### Цикл `do-while`
- Условие проверяется **после** итерации, тело выполнится хотя бы один раз.
```java
int i = 0;
do {
    System.out.println("i = " + i);
    i++;
} while (i < 5);
```

> **Совет:** Используйте `while` для неизвестного числа итераций, `do-while` — если нужно выполнить хотя бы раз.

---

## 2. Цикл `for` и `for-each`

### 2.1 Цикл `for`
- Удобен, когда количество итераций известно или выражается переменной.

> **Синтаксис:**
```java
for (инициализация; условие; шаг) {
    // Тело цикла
}
```

> **Пример:**
```java
for (int i = 0; i < 5; i++) {
    System.out.println("i = " + i);
}
```

- Можно использовать обратный цикл, несколько переменных, цикл без тела, бесконечный цикл (`for (;;) { ... }`).

### 2.2 Цикл `for-each` (Java 5+)
- Предназначен для перебора коллекций или массивов.

> **Синтаксис:**
```java
for (Тип элемент : коллекция) {
    // Тело цикла
}
```

> **Пример:**
```java
List<String> list = List.of("A", "B", "C");
for (String s : list) {
    System.out.println(s);
}
```

> **Совет:** Используйте `for-each` для коллекций и массивов, если не нужен индекс.

---

## 3. Stream API: `stream().forEach()`

- Позволяет выполнять действия для каждого элемента коллекции в функциональном стиле.
- Удобен для сложных цепочек операций (`filter`, `map`, ...), поддерживает параллельную обработку.

> **Пример:**
```java
List<String> list = List.of("A", "B", "C");
list.stream().forEach(s -> System.out.println("Элемент: " + s));
```

- Для параллельной обработки используйте `parallelStream().forEach()`, но порядок не гарантирован.
- Для упорядоченной обработки используйте `forEachOrdered()`.

> **Совет:** Для простых операций на небольших коллекциях обычные циклы быстрее. Stream API удобен для сложных преобразований и больших данных.

## 4. Рекурсия

- Метод вызывает сам себя для решения подзадачи.
- Подходит для задач с естественной рекурсивной структурой (обход деревьев, факториал, Фибоначчи).

> **Пример:**
```java
public static long factorial(int n) {
    if (n == 0 || n == 1) return 1;
    return n * factorial(n - 1);
}
```

### Мемоизация

> **Мемоизация** — это техника кэширования результатов рекурсивных вызовов для ускорения вычислений и избежания повторных операций.

> **Пример:**
```java
import java.util.HashMap;
import java.util.Map;

Map<Integer, Long> memo = new HashMap<>();
long fib(int n) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    long result = fib(n - 1) + fib(n - 2);
    memo.put(n, result);
    return result;
}
```
> **Совет:** Используйте мемоизацию для рекурсивных задач с перекрывающимися подзадачами (например, Фибоначчи, динамическое программирование).

- **Хвостовая рекурсия**: вызов метода — последняя операция. В Java не оптимизируется.
- **Итеративная альтернатива**: часто быстрее и экономичнее по памяти.

---

## 5. Внутренняя реализация и производительность

- **while/for**: Компилируются в условные переходы и оптимизируются JIT-компилятором.
- **for-each**: Для массивов — индексный цикл, для коллекций — итератор.
- **stream().forEach()**: Использует Spliterator и лямбды, медленнее для простых операций, но удобен для сложных цепочек.
- **Рекурсия**: Каждый вызов — новая стековая рамка, без оптимизации хвостовой рекурсии.

> **Рекомендация:**
> - Используйте `for` для счётных циклов.
> - Используйте `while` для непредсказуемых итераций.
> - Используйте `for-each` или `stream().forEach()` для коллекций.
> - Заменяйте рекурсию итерацией для больших данных.

## 7. Подводные камни (чек-лист)

- [ ] **Бесконечный цикл**
    - Пример:
    ```java
    while (true) { ... }
    ```
    > **Решение:** Всегда предусматривайте условие выхода.
- [ ] **Ошибки в условии цикла**
    - Пример:
    ```java
    for (int i = 0; i >= 0; i++) { ... }
    ```
    > **Решение:** Проверяйте границы и условия.
- [ ] **Модификация коллекции в for-each/stream**
    - Пример:
    ```java
    for (String s : list) { list.remove(s); }
    ```
    > **Решение:** Используйте итератор или collect.
- [ ] **Гонка данных в parallelStream().forEach()**
    - Пример:
    ```java
    long[] sum = {0};
    list.parallelStream().forEach(n -> sum[0] += n);
    ```
    > **Решение:** Используйте reduce или sum.
- [ ] **StackOverflowError в рекурсии**
    - Пример:
    ```java
    public static void recurse(int n) { recurse(n); }
    ```
    > **Решение:** Всегда задавайте базовое условие.
- [ ] **Неэффективная рекурсия**
    - Пример: Фибоначчи без мемоизации.
    > **Решение:** Используйте мемоизацию или итерацию.

## 8. Лучшие практики

1. **Выбирайте подходящий цикл**: `for` — для известного числа итераций, `while` — для непредсказуемых, `for-each`/`stream` — для коллекций.
2. **Используйте стримы для выразительности**:
    ```java
    list.stream().filter(n -> n % 2 == 0).forEach(System.out::println);
    ```
3. **Задавайте явное условие выхода**:
    ```java
    while (condition) { if (exitCondition) break; }
    ```
4. **Предпочитайте итерацию для глубоких рекурсий**.
5. **Используйте мемоизацию для рекурсивных задач**.

## 9. Пример: Комплексное использование циклов, стримов и рекурсии

> **Пример класса, иллюстрирующего работу с разными циклами, стримами, рекурсией, мемоизацией и многопоточностью:**

```java
import java.util.Arrays;
import java.util.List;
import java.util.Map;
import java.util.HashMap;
import java.util.concurrent.atomic.AtomicLong;
import java.util.logging.Logger;
import java.util.stream.IntStream;

public class LoopRecursionExample {
    private static final Logger LOGGER = Logger.getLogger(LoopRecursionExample.class.getName());
    private static final Map<Integer, Long> memo = new HashMap<>();
    private static final AtomicLong counter = new AtomicLong(0);

    // Рекурсивный факториал с проверкой
    public static long factorial(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be non-negative");
        if (n == 0 || n == 1) return 1;
        return n * factorial(n - 1);
    }

    // Итеративный Фибоначчи
    public static long fibIterative(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be non-negative");
        if (n == 0) return 0;
        if (n == 1) return 1;
        long a = 0, b = 1;
        for (int i = 2; i <= n; i++) {
            long next = a + b;
            a = b;
            b = next;
            counter.incrementAndGet(); // Потокобезопасный счетчик
        }
        return b;
    }

    // Фибоначчи с мемоизацией
    public static long fibMemoized(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be non-negative");
        if (n == 0) return 0;
        if (n == 1) return 1;
        if (memo.containsKey(n)) return memo.get(n);
        long result = fibMemoized(n - 1) + fibMemoized(n - 2);
        memo.put(n, result);
        return result;
    }

    // Обработка списка с помощью стрима
    public static void processList(List<String> list) {
        list.stream().forEach(s -> {
            synchronized (LoopRecursionExample.class) {
                LOGGER.info("Элемент: " + s);
            }
        });
    }

    // Параллельная обработка с parallelStream()
    public static long parallelSum(List<Integer> numbers) {
        return numbers.parallelStream().mapToLong(i -> i).sum();
    }

    public static void main(String[] args) {
        // Цикл while
        int i = 0;
        while (i < 3) {
            LOGGER.info("while: i = " + i);
            i++;
        }

        // Цикл for
        for (int j = 0; j < 3; j++) {
            LOGGER.info("for: j = " + j);
        }

        // for-each
        List<String> list = List.of("A", "B", "C");
        for (String s : list) {
            LOGGER.info("for-each: " + s);
        }

        // stream().forEach()
        processList(list);

        // parallelStream().forEach()
        List<Integer> numbers = IntStream.range(0, 10).boxed().toList();
        numbers.parallelStream().forEach(n -> LOGGER.info("parallelStream: " + n));

        // Рекурсия: факториал
        LOGGER.info("Факториал(5): " + factorial(5));

        // Итеративный Фибоначчи
        LOGGER.info("Фибоначчи(10): " + fibIterative(10));

        // Рекурсия с мемоизацией
        LOGGER.info("Фибоначчи с мемоизацией(10): " + fibMemoized(10));

        // Многопоточная обработка
        Runnable task = () -> {
            long result = parallelSum(numbers);
            LOGGER.info("Параллельная сумма: " + result + ", счетчик: " + counter.get());
        };
        new Thread(task).start();
    }
}
```
