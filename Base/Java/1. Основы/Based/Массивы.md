# Массивы в Java

> **Массив** — это объект, который хранит упорядоченный набор элементов одного типа (примитивного или ссылочного). Каждый элемент доступен по целочисленному индексу, начиная с `0`. **Длина массива фиксируется при создании и не может быть изменена, но значения элементов можно менять.**

- **Фиксированная длина**: Размер массива определяется при создании и не может быть изменён.
- **Однотипность**: Все элементы имеют одинаковый тип (например, `int`, `String`).
- **Хранение**: Массивы всегда хранятся в **куче** (heap), даже для примитивных типов.
- **Индексация**: Доступ к элементам осуществляется за O(1) по индексу.
## Объявление, создание и инициализация

### Объявление

> **Пример:**
```java
int[] numbers;      // Предпочтительный стиль
int numbers2[];     // Допустимо, но реже используется
```
> **Важно:** Переменная массива — это **ссылка**, которая изначально равна `null`, пока массив не создан.

### 2.2 Создание (выделение памяти)

> **Пример:**
```java
int[] numbers = new int[5]; // Массив из 5 элементов, все равны 0
```

- Примитивы: `0` (для чисел), `false` (для `boolean`), `\u0000` (для `char`).
- Ссылочные типы: `null`.

### 2.3 Инициализация значениями

> **Пример:**
```java
int[] nums = {10, 20, 30}; // Массив из 3 элементов
String[] words = {"hello", "world"};
```
> **Важно:** Формат `{...}` нельзя использовать отдельно от объявления.

```java
int[] nums;
nums = {1, 2, 3}; // Ошибка компиляции
// Правильно:
nums = new int[]{1, 2, 3};
```

## 3. Работа с массивами

### 3.1 Доступ к элементам

> **Пример:**
```java
int[] nums = {10, 20, 30};
int first = nums[0]; // 10
nums[1] = 99;        // nums = {10, 99, 30}
```
> **Ошибка:** Доступ за пределами массива вызывает `ArrayIndexOutOfBoundsException`.

```java
int x = nums[3]; // Ошибка: индекс вне границ
```

### 3.2 Получение длины

> **Пример:**
```java
int len = nums.length; // 3
```

### 3.3 Перебор массива

#### Классический цикл `for`
```java
for (int i = 0; i < nums.length; i++) {
    System.out.println(nums[i]);
}
```
#### Цикл `for-each` (Java 5+)
```java
for (int num : nums) {
    System.out.println(num);
}
```
> **Совет:** `for-each` удобен для чтения, но не позволяет изменять индекс или массив напрямую.

### 3.4 Многомерные массивы

#### Двумерный массив
```java
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};
System.out.println(matrix[1][2]); // 6
```
#### Создание вручную
```java
int[][] matrix = new int[2][3]; // 2 строки, 3 столбца
matrix[0][0] = 1;
```
#### Зубчатый (jagged) массив
```java
int[][] jagged = new int[2][];
jagged[0] = new int[3]; // {0, 0, 0}
jagged[1] = new int[5]; // {0, 0, 0, 0, 0}
```

### 3.5 Массивы объектов

> **Пример:**
```java
String[] messages = new String[2];
messages[0] = "Hi";
messages[1] = "Bye";
```
> **Важно:** Массив объектов изначально заполнен `null`.

## 4. API `java.util.Arrays`

Класс `Arrays` предоставляет утилитные методы для работы с массивами.

| Операция         | Пример кода                                   | Описание                       |
|------------------|-----------------------------------------------|---------------------------------|
| Сортировка       | Arrays.sort(a);                               | Быстрая сортировка             |
| Вывод            | Arrays.toString(a);                           | Преобразование в строку        |
| Копирование      | Arrays.copyOf(a, a.length);                   | Копия массива                  |
| Копия диапазона  | Arrays.copyOfRange(a, 1, 3);                  | Копия части массива            |
| Заполнение       | Arrays.fill(a, 42);                           | Все элементы = 42              |
| Сравнение        | Arrays.equals(a, b);                          | Поверхностное сравнение        |
| Бинарный поиск   | Arrays.binarySearch(sorted, 2);               | Поиск в отсортированном массиве|
| Глубокое сравнение| Arrays.deepEquals(matrix1, matrix2);         | Для многомерных массивов       |
| Глубокий вывод   | Arrays.deepToString(matrix1);                 | Для многомерных массивов       |

## 5. Клонирование и копирование массивов

> **Пример:**
```java
int[] original = {1, 2, 3};
int[] clone = original.clone(); // Новая копия
clone[0] = 99; // Не влияет на original
```

- Для примитивных типов: создается **глубокая копия** (значения копируются).
- Для ссылочных типов: создается **поверхностная копия** (копируются ссылки, а не объекты).

> **Пример:**
```java
String[] original = {"a", "b"};
String[] clone = original.clone();
clone[0] = "z"; // Не влияет на original
original[0].concat("x"); // Влияет на оба, так как ссылка на объект общая
```

## 6. Внутренняя реализация в JVM

```mermaid
graph TD;
  A[Переменная массива (ссылка в стеке)] -->|указывает на| B[Объект-массив в куче];
  B -->|содержит| C[Заголовок: тип, длина];
  B -->|содержит| D[Данные: элементы];
```

- **Хранение**: Массивы — это объекты, всегда хранятся в **куче** (heap), даже для примитивных типов.
- **Структура**:
    - Заголовок: Содержит метаданные (тип элементов, длина).
    - Данные: Последовательность элементов (значения для примитивов, ссылки для объектов).
- **Ссылки**: Переменная массива — это ссылка в стеке, указывающая на объект в куче.
- **Сборка мусора**: Массив удаляется из памяти, когда становится недоступным.

> **Пример:**
```java
int[] arr = new int[3]; // arr — ссылка в стеке, массив — в куче
```

## 7. Производительность

| Операция                | Время (Big-O) |
|-------------------------|---------------|
| Чтение по индексу       | O(1)          |
| Запись по индексу       | O(1)          |
| Поиск значения          | O(n)          |
| Вставка/удаление        | O(n)          |
| Копирование             | O(n)          |
| Сортировка (`sort`)     | O(n log n)    |

- **Преимущества**:
    - Быстрый доступ по индексу (O(1)).
    - Компактное хранение для примитивов.
- **Недостатки**:
    - Фиксированная длина требует создания нового массива для изменения размера.
    - Вставка/удаление требуют сдвига элементов.

## 8. Многопоточность и массивы

- **Потокобезопасность**:
    - Массивы не являются потокобезопасными. Одновременный доступ из нескольких потоков может привести к гонке данных.
    - **Решение**: Используйте `synchronized` или классы из `java.util.concurrent`.

> **Пример:**
```java
int[] sharedArray = new int[10];
synchronized (sharedArray) {
    sharedArray[0]++;
}
```

- **Атомарные операции**:
    - Для примитивных массивов используйте `AtomicIntegerArray`:

> **Пример:**
```java
import java.util.concurrent.atomic.AtomicIntegerArray;
AtomicIntegerArray atomicArray = new AtomicIntegerArray(10);
atomicArray.incrementAndGet(0); // Атомарное увеличение
```

- **Volatile массивы**:
    - Ключевое слово `volatile` для массива обеспечивает видимость ссылки, но не элементов.
    - **Решение**: Используйте `AtomicIntegerArray` или `AtomicReferenceArray` для ссылочных типов.

## 9. Массивы vs `ArrayList`

| Характеристика   | Массив         | `ArrayList`                |
|------------------|---------------|----------------------------|
| Размер           | Фиксированный | Динамический               |
| Тип элементов    | Примитивы/Объекты | Только объекты (автоупаковка для примитивов) |
| Производительность| Быстрее       | Медленнее (управление размером, автоупаковка) |
| Функциональность | Ограниченная (`Arrays`) | Богатый API (`add`, `remove`, etc.) |
| Память           | Компактнее     | Больше накладных расходов  |

> **Когда использовать:**
> - **Массивы**: Для фиксированного размера, примитивных типов, высокой производительности.
> - **`ArrayList`**: Для динамического размера, удобных операций добавления/удаления.
