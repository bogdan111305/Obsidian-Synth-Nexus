# Функции и методы в Scala

## Обзор

Scala объединяет функциональное и объектно-ориентированное программирование. Функции являются первоклассными объектами, что означает, что их можно передавать как параметры, возвращать из других функций и присваивать переменным.

## Определение функций

### Базовый синтаксис

```scala
// Простая функция
def add(x: Int, y: Int): Int = x + y

// Функция без параметров
def getCurrentTime(): Long = System.currentTimeMillis()

// Функция без возвращаемого значения
def printMessage(msg: String): Unit = println(msg)

// Функция с выведением типа
def multiply(x: Int, y: Int) = x * y
```

### Функции с блоками

```scala
def calculateArea(radius: Double): Double = {
  val pi = 3.14159
  pi * radius * radius
}

def processUser(user: User): UserInfo = {
  val permissions = getUserPermissions(user)
  val settings = getUserSettings(user)
  UserInfo(user, permissions, settings)
}
```

## Параметры функций

### Именованные параметры

```scala
def createUser(name: String, age: Int, email: String): User = {
  User(name, age, email)
}

// Вызов с именованными параметрами
val user = createUser(
  name = "Alice",
  age = 25,
  email = "alice@example.com"
)
```

### Параметры по умолчанию

```scala
def greet(name: String, greeting: String = "Hello"): String = {
  s"$greeting, $name!"
}

// Использование
val msg1 = greet("Alice") // "Hello, Alice!"
val msg2 = greet("Bob", "Hi") // "Hi, Bob!"
```

### Множественные списки параметров

```scala
def processData(data: List[Int])(f: Int => Int): List[Int] = {
  data.map(f)
}

// Использование
val numbers = List(1, 2, 3, 4, 5)
val doubled = processData(numbers)(_ * 2)
```

### Varargs (переменное количество параметров)

```scala
def sum(numbers: Int*): Int = {
  numbers.sum
}

// Использование
val result1 = sum(1, 2, 3, 4, 5)
val result2 = sum(10, 20)
val result3 = sum() // 0
```

## Возвращаемые значения

### Явное указание типа

```scala
def getString(): String = "Hello"
def getNumber(): Int = 42
def getBoolean(): Boolean = true
```

### Выведение типа

```scala
def add(x: Int, y: Int) = x + y // Тип выводится как Int
def getList() = List(1, 2, 3) // Тип выводится как List[Int]
```

### Unit (эквивалент void)

```scala
def printMessage(msg: String): Unit = println(msg)
def doSomething(): Unit = {
  // Действия без возвращаемого значения
}

// Unit можно опустить
def printMessage(msg: String) = println(msg)
```

## Локальные функции

```scala
def factorial(n: Int): Int = {
  def factorialHelper(n: Int, acc: Int): Int = {
    if (n <= 1) acc
    else factorialHelper(n - 1, n * acc)
  }
  
  factorialHelper(n, 1)
}
```

## Функции высшего порядка

### Функции как параметры

```scala
def applyFunction(f: Int => Int, x: Int): Int = f(x)
def applyPredicate(p: String => Boolean, s: String): Boolean = p(s)

// Использование
val result = applyFunction(x => x * x, 5) // 25
val isLong = applyPredicate(_.length > 5, "Hello World") // true
```

### Функции как возвращаемые значения

```scala
def createMultiplier(factor: Int): Int => Int = {
  (x: Int) => x * factor
}

// Использование
val double = createMultiplier(2)
val triple = createMultiplier(3)
val result1 = double(5) // 10
val result2 = triple(5) // 15
```

## Анонимные функции (лямбды)

```scala
// Базовый синтаксис
val add = (x: Int, y: Int) => x + y
val square = (x: Int) => x * x
val isEven = (x: Int) => x % 2 == 0

// С подчеркиванием
val double = (_: Int) * 2
val addFive = (_: Int) + 5

// Использование
val result1 = add(3, 4) // 7
val result2 = square(5) // 25
val result3 = isEven(6) // true
```

## Частичное применение функций

```scala
def add(x: Int, y: Int): Int = x + y

// Частичное применение
val addFive = add(5, _: Int)
val result = addFive(3) // 8

// Альтернативный способ
val addFiveAlt = add(5, _)
```

## Каррирование

```scala
// Каррированная функция
def addCurried(x: Int)(y: Int): Int = x + y

// Использование
val addFive = addCurried(5)_
val result = addFive(3) // 8

// Автоматическое каррирование
def add(x: Int, y: Int): Int = x + y
val addFive = add(5, _)
```

## Методы в классах

### Определение методов

```scala
class Calculator {
  def add(x: Int, y: Int): Int = x + y
  
  def multiply(x: Int, y: Int): Int = x * y
  
  def divide(x: Int, y: Int): Double = {
    if (y == 0) throw new IllegalArgumentException("Division by zero")
    x.toDouble / y
  }
}

// Использование
val calc = new Calculator()
val result = calc.add(5, 3) // 8
```

### Методы с побочными эффектами

```scala
class Logger {
  private var messages: List[String] = List()
  
  def log(message: String): Unit = {
    messages = message :: messages
    println(s"[LOG] $message")
  }
  
  def getMessages: List[String] = messages.reverse
}
```

## Функциональные объекты

```scala
object MathUtils {
  def factorial(n: Int): Int = {
    if (n <= 1) 1
    else n * factorial(n - 1)
  }
  
  def fibonacci(n: Int): Int = {
    if (n <= 1) n
    else fibonacci(n - 1) + fibonacci(n - 2)
  }
}

// Использование
val fact5 = MathUtils.factorial(5) // 120
val fib10 = MathUtils.fibonacci(10) // 55
```

## Типичные ошибки

### 1. Неправильное использование Unit

```scala
// Ошибка - функция возвращает Unit, но ожидается Int
def process(x: Int): Unit = {
  println(x)
  // Нет возвращаемого значения
}

val result: Int = process(5) // Ошибка компиляции

// Правильно
def process(x: Int): Int = {
  println(x)
  x * 2
}
```

### 2. Неправильное каррирование

```scala
// Ошибка - неправильный синтаксис
def add(x: Int, y: Int): Int = x + y
val addFive = add(5) // Ошибка компиляции

// Правильно
val addFive = add(5, _)
```

### 3. Забывание типов параметров

```scala
// Ошибка - неоднозначность типов
def process(x) = x * 2 // Ошибка компиляции

// Правильно
def process(x: Int) = x * 2
```

## Лучшие практики

### 1. Используйте функциональный стиль

```scala
// Хорошо
def processList(list: List[Int]): List[Int] = {
  list.filter(_ > 0).map(_ * 2)
}

// Избегайте
def processList(list: List[Int]): List[Int] = {
  var result = List[Int]()
  for (item <- list) {
    if (item > 0) {
      result = (item * 2) :: result
    }
  }
  result.reverse
}
```

### 2. Делайте функции чистыми

```scala
// Хорошо - чистая функция
def add(x: Int, y: Int): Int = x + y

// Избегайте - побочные эффекты
def addWithLogging(x: Int, y: Int): Int = {
  println(s"Adding $x and $y")
  x + y
}
```

### 3. Используйте значимые имена

```scala
// Хорошо
def calculateArea(radius: Double): Double = {
  math.Pi * radius * radius
}

// Избегайте
def calc(r: Double): Double = {
  3.14 * r * r
}
```
