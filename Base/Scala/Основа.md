##### **Scala — это ООП язык**, так как каждое значение в нём — это объект.
##### **Scala — это ФП язык**, так как каждая функция в нём — это значение.
#### Вывод: **Каждая функция в Scala - объект**

---
##### **`val` — неизменяемая переменная**

- `val` используется для объявления **неизменяемой** переменной. После присвоения значения его нельзя изменить.
    
- Это похоже на `final` в Java или `const` в JavaScript.
    
- Используется для объявления констант или значений, которые не должны меняться.
---
##### **Ленивые переменные (`lazy val`)**

- `lazy val` — это переменная, значение которой вычисляется только при первом обращении к ней.
    
- Полезно для оптимизации, когда вычисление значения дорогостоящее или не всегда требуется
---
##### **`var` — изменяемая переменная**

- `var` используется для объявления **изменяемой** переменной. Её значение можно изменять после инициализации.
    
- Используется, когда значение переменной должно меняться в процессе выполнения программы.
---
##### **Блоки кода `{}` в Scala**

- Группы выражений, заключенные в `{}`.
    
- Они возвращают значение последнего выражения, вычисления внутри блока.
    
- Переменные, объявленные внутри блока, видны только внутри этого блока.
    
- Блоки кода широко используются для организации кода, определения функций, вычисления значений и выполнения побочных эффектов.
---
##### **Условные выражения (`if`, `else if`, `else`)**

Условные выражения позволяют выполнять код в зависимости от выполнения определённых условий.

##### Пример:

``` Scala
val number = 10

if (number > 0) {
  println("Число положительное")
} else if (number < 0) {
  println("Число отрицательное")
} else {
  println("Число равно нулю")
}
```

- В Scala `if` является выражением, то есть возвращает значение. Это позволяет использовать его для присваивания:
    
``` Scala
val result = if (number > 0) "Положительное" else "Отрицательное или ноль"
println(result)
 ```
---
##### **Циклы (`while`, `do-while`)**

Циклы используются для многократного выполнения блока кода.
##### `while`:

``` Scala
var i = 0
while (i < 5) {
  println(i)
  i += 1
}
```
##### `do-while`:

``` Scala
var j = 0
do {
  println(j)
  j += 1
} while (j < 5)
```

- В Scala циклы используются реже, чем в императивных языках, так как предпочтение отдается функциональным подходам, таким как рекурсия или методы высшего порядка (например, `map`, `filter`).
---
##### **Цикл `for`**

Цикл `for` в Scala более мощный, чем в других языках. Он может использоваться для итерации по коллекциям, генерации последовательностей и фильтрации.

##### Итерация по диапазону:

``` Scala
for (i <- 1 to 5) {
  println(i)
}
```
##### Итерация по коллекции:

``` Scala
val fruits = List("apple", "banana", "orange")
for (fruit <- fruits) {
  println(fruit)
}
```
##### Фильтрация в `for`:

``` Scala
for (i <- 1 to 10 if i % 2 == 0) {
  println(i) // Выведет четные числа от 1 до 10
}
```
##### Генерация коллекции с помощью `yield`:

``` Scala
val squares = for (i <- 1 to 5) yield i * i
println(squares) // Vector(1, 4, 9, 16, 25)
```
---
##### **Сопоставление с образцом (`match`)**

Сопоставление с образцом (pattern matching) — это мощная конструкция, которая позволяет сравнивать значение с набором шаблонов и выполнять соответствующий код.
##### Пример:

``` Scala
val day = "Monday"

val result = day match {
  case "Monday" => "Начало недели"
  case "Friday" => "Конец недели"
  case _ => "Обычный день" // Шаблон по умолчанию
}

println(result) // Начало недели
```
- `match` можно использовать с любыми типами данных, включая case-классы, кортежи и коллекции.
---
##### **Рекурсия**

Рекурсия — это функциональный подход к повторению операций. В Scala рекурсия часто используется вместо циклов.
##### Пример:

``` Scala
def factorial(n: Int): Int = {
  if (n <= 1) 1
  else n * factorial(n - 1)
}

println(factorial(5)) // 120
```
- Для оптимизации хвостовой рекурсии используется аннотация `@tailrec` (из пакета `scala.annotation.tailrec`).
    
---
##### **Исключения (`try`, `catch`, `finally`)**

Scala поддерживает обработку исключений с помощью конструкций `try`, `catch` и `finally`.
##### Пример:

``` Scala
try {
  val result = 10 / 0
} catch {
  case e: ArithmeticException => println("Деление на ноль!")
} finally {
  println("Этот блок выполнится в любом случае")
}
```
---
##### **`for`-выражения (монадические comprehension)**

`for`-выражения в Scala могут использоваться для работы с монадическими типами, такими как `Option`, `Try`, `Future` и коллекции.
##### Пример с `Option`:

``` Scala
val maybeNumber: Option[Int] = Some(42)
val maybeResult = for {
  number <- maybeNumber
} yield number * 2

println(maybeResult) // Some(84)
```
---
##### **`return`**

В Scala использование `return` для возврата из функции не рекомендуется. Вместо этого используется значение последнего выражения в функции.
##### Пример (без `return`):

``` Scala
def add(a: Int, b: Int): Int = {
  a + b
}

println(add(2, 3)) // 5
```

##### **`def` как метод, `val` как функция-значение**

##### `def` — метод

В Scala `def` используется для определения методов. Метод — это не функция, а часть класса или объекта. Методы могут принимать параметры и возвращать значения.

``` Scala
def add(a: Int, b: Int): Int = {
  a + b
}
```
Здесь `add` — это метод, который принимает два аргумента и возвращает их сумму.

##### `val` — функция-значение

Функции в Scala являются объектами, и их можно присваивать переменным с помощью `val`. Такие функции называются **функциями-значениями**.

``` Scala
val addFunction: (Int, Int) => Int = (a, b) => a + b
```
Здесь `addFunction` — это переменная, которая хранит функцию, принимающую два аргумента и возвращающую их сумму.

---
##### **Преобразования функций**

Функции в Scala можно преобразовывать и комбинировать. Например, можно создать новую функцию на основе существующей:

``` Scala
val multiplyByTwo: Int => Int = (x: Int) => x * 2
val addOne: Int => Int = (x: Int) => x + 1

// Композиция функций
val composedFunction: Int => Int = multiplyByTwo andThen addOne

val result = composedFunction(3) // (3 * 2) + 1 = 7
println(result) // Вывод: 7
```
Здесь `andThen` используется для композиции функций: сначала применяется `multiplyByTwo`, затем `addOne`.

---
##### **Композиция функций в Scala**

Композиция функций — это создание новой функции путем объединения двух или более функций. В Scala есть два основных способа композиции:
##### a) `andThen`

Функция `andThen` применяет первую функцию, а затем передает результат второй функции.

``` Scala
val f: Int => Int = (x: Int) => x + 1
val g: Int => Int = (x: Int) => x * 2

val composed: Int => Int = f andThen g

val result = composed(3) // (3 + 1) * 2 = 8
println(result) // Вывод: 8
```
##### b) `compose`

Функция `compose` применяет вторую функцию, а затем передает результат первой функции.

``` Scala
val f: Int => Int = (x: Int) => x + 1
val g: Int => Int = (x: Int) => x * 2

val composed: Int => Int = f compose g

val result = composed(3) // (3 * 2) + 1 = 7
println(result) // Вывод: 7
```
---
##### **Композиция функций в Java**

В Java нет встроенной поддержки функций как объектов первого класса (до Java 8). Однако, начиная с Java 8, появились функциональные интерфейсы и лямбда-выражения, которые позволяют работать с функциями.
##### Пример композиции в Java:

``` java
import java.util.function.Function;

public class Main {
    public static void main(String[] args) {
        Function<Integer, Integer> f = x -> x + 1;
        Function<Integer, Integer> g = x -> x * 2;

        // Композиция функций
        Function<Integer, Integer> composed = f.andThen(g);

        int result = composed.apply(3); // (3 + 1) * 2 = 8
        System.out.println(result); // Вывод: 8
    }
}
```
Здесь `andThen` используется для композиции функций, аналогично Scala.
##### **Реализация анонимной функции**

##### **В Scala**

В Scala анонимная функция реализуется как объект, соответствующий функциональному типу. Функциональные типы в Scala представлены как `A => B`, где `A` — тип входного параметра, а `B` — тип возвращаемого значения.
##### Пример:
``` Scala
val add = (a: Int, b: Int) => a + b
```
Здесь:

- `(a: Int, b: Int)` — параметры функции.
- `a + b` — тело функции.
- `add` — переменная, которая хранит анонимную функцию.

Под капотом Scala создает объект, реализующий функциональный тип `Function2[Int, Int, Int]` (для функции с двумя аргументами). Этот объект имеет метод `apply`, который выполняет логику функции.

---
##### **В Java**

В Java анонимные функции реализуются через лямбда-выражения, которые являются синтаксическим сахаром для анонимных классов, реализующих функциональные интерфейсы (например, `Function`, `Consumer`, `Predicate` и т.д.).
##### Пример:
``` java
Function<Integer, Integer> square = x -> x * x;
```
Здесь:
- `x -> x * x` — лямбда-выражение.
- `square` — переменная, которая хранит анонимную функцию.

Под капотом Java создает экземпляр анонимного класса, реализующего интерфейс `Function`, и переопределяет его метод `apply`.

---

#### **Что представляет собой анонимная функция?**

Анонимная функция состоит из:

1. **Параметров**: Список входных параметров (может быть пустым).
2. **Тела функции**: Логика, которая выполняется при вызове функции.
3. **Контекста**: Анонимные функции могут захватывать переменные из окружающего контекста (замыкания).
---

##### **Как работает анонимная функция под капотом?**

##### В Scala:

4. Анонимная функция компилируется в объект, реализующий один из функциональных трейтов (`Function0`, `Function1`, `Function2`, и т.д., в зависимости от количества аргументов).
5. Этот объект имеет метод `apply`, который выполняет логику функции.
6. Когда вы вызываете анонимную функцию, на самом деле вызывается метод `apply` этого объекта.
##### Пример:

``` Scala
val add = (a: Int, b: Int) => a + b
```
Под капотом это преобразуется в:

``` Scala
val add = new Function2[Int, Int, Int] {
  def apply(a: Int, b: Int): Int = a + b
}
```
##### В Java:

7. Лямбда-выражение компилируется в экземпляр анонимного класса, реализующего функциональный интерфейс.
8. Этот класс переопределяет метод интерфейса (например, `apply` для `Function`).
9. Когда вы вызываете лямбда-выражение, на самом деле вызывается переопределенный метод.
##### Пример:
``` java
Function<Integer, Integer> square = x -> x * x;
```
Под капотом это преобразуется в:
``` java
Function<Integer, Integer> square = new Function<Integer, Integer>() {
  @Override
  public Integer apply(Integer x) {
    return x * x;
  }
};
```
---
##### **Замыкания (Closures)**

Анонимные функции могут захватывать переменные из окружающего контекста. Такие функции называются **замыканиями**.
##### Пример в Scala:
``` Scala
val factor = 10
val multiplier = (x: Int) => x * factor

println(multiplier(5)) // Вывод: 50
```
Здесь `factor` — переменная из внешнего контекста, которая захватывается анонимной функцией.
##### Пример в Java:
``` java
int factor = 10;
Function<Integer, Integer> multiplier = x -> x * factor;

System.out.println(multiplier.apply(5)); // Вывод: 50
```
