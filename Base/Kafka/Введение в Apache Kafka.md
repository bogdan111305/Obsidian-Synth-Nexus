# Введение в Apache Kafka: распределённая система обмена сообщениями

> [!QUOTE] Apache Kafka — это распределённый брокер сообщений для потоковой обработки данных. Хранит сообщения на диске, гарантирует порядок внутри партиции, обеспечивает высокую пропускную способность и горизонтальную масштабируемость.

## Что такое Apache Kafka

**Apache Kafka** — распределённая платформа потоковой передачи данных (distributed streaming platform). Предназначена для обработки потоков данных в реальном времени.

### Ключевые характеристики:
- **Высокая пропускная способность**: миллионы сообщений в секунду на одном узле
- **Отказоустойчивость**: репликация данных между брокерами
- **Горизонтальная масштабируемость**: добавление брокеров без остановки системы
- **Сохраняемость**: данные хранятся на диске, не теряются при перезапуске
- **Распределённость**: кластерная архитектура из нескольких серверов

> [!INFO] Kafka хранит данные на диске и поддерживает настройку времени хранения:
> ```
> log.retention.hours=168   # 7 дней по умолчанию
> log.retention.bytes=1073741824  # 1GB
> ```

## Архитектура Kafka

### Основные компоненты

#### 1. Broker (Брокер)
Сервер Kafka, который хранит данные и обрабатывает запросы от producer'ов и consumer'ов.

```properties
# server.properties (минимум)
broker.id=0
listeners=PLAINTEXT://localhost:9092
log.dirs=/tmp/kafka-logs
```

#### 2. Topic (Топик)
Категория или канал для сообщений. Создаётся с указанием партиций и фактора репликации.

```bash
kafka-topics.sh --create --topic my-topic \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 2
```

#### 3. Partition (Партиция)
Упорядоченная последовательность сообщений. Каждое сообщение имеет уникальный `offset` внутри партиции.

#### 4. Producer (Продюсер)
Отправляет сообщения в топики. Может указывать ключ — тогда сообщения с одним ключом всегда попадают в одну партицию.

```java
KafkaProducer<String, String> producer = new KafkaProducer<>(props);
ProducerRecord<String, String> record = new ProducerRecord<>("my-topic", "key", "value");
producer.send(record);
```

#### 5. Consumer (Консьюмер)
Читает сообщения из топиков. Отслеживает прогресс через `offset`.

```java
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("my-topic"));
ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
```

#### 6. Consumer Group (Группа консьюмеров)
Группа консьюмеров, совместно обрабатывающих сообщения из топика. Каждая партиция назначается ровно одному консьюмеру в группе.

```java
props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-consumer-group");
```

## Основные концепции

### 1. Партиционирование
- Сообщения распределяются по партициям на основе ключа
- Сообщения с одинаковым ключом попадают в одну партицию
- Порядок сообщений гарантируется только внутри одной партиции

### 2. Репликация
- Каждая партиция реплицируется на несколько брокеров
- Один брокер является лидером (leader), остальные — репликами (follower)
- При отказе лидера автоматически выбирается новый

### 3. Offset (Смещение)
- Уникальный идентификатор сообщения внутри партиции
- Consumer отслеживает прогресс через offset
- Commit offset можно делать вручную или автоматически

### 4. Retention (Хранение)
```properties
log.retention.hours=168          # По времени
log.retention.bytes=1073741824   # По размеру
log.segment.bytes=1073741824     # Размер сегмента
```

> [!WARNING] Частые ловушки:
> - **Порядок сообщений** гарантируется только внутри одной партиции. Если нужен глобальный порядок — используйте один топик с одной партицией (но тогда нет масштабируемости).
> - **at-least-once vs exactly-once**: по умолчанию Kafka гарантирует at-least-once. Для exactly-once нужны идемпотентные producer'ы и транзакции — это усложняет систему и снижает пропускную способность.
> - **Kafka — не RPC**: не используйте Kafka для синхронного запрос-ответ между сервисами. Для этого есть REST/gRPC.

## Сравнение с другими системами

### Kafka vs RabbitMQ
| Характеристика | Kafka | RabbitMQ |
|----------------|-------|----------|
| Модель | Публикация/Подписка | Очереди сообщений |
| Производительность | Очень высокая | Высокая |
| Сохраняемость | На диске | В памяти/на диске |
| Масштабируемость | Горизонтальная | Ограниченная |
| Сложность | Средняя | Низкая |

### Kafka vs Apache Pulsar
| Характеристика | Kafka | Pulsar |
|----------------|-------|--------|
| Архитектура | Монолитная | Многослойная |
| Geo-replication | Ручная настройка | Встроенная |
| Многоарендность | Ограниченная | Полная |
| Производительность | Высокая | Очень высокая |

## Области применения

- **Логирование и мониторинг**: централизованный сбор логов из множества источников
- **Потоковая обработка**: обработка событий в реальном времени, агрегация
- **Интеграция микросервисов**: event-driven архитектура, связывание систем
- **Аналитика**: трекинг пользовательского поведения, бизнес-метрики
- **IoT и телеметрия**: сбор данных с устройств

## Вопросы для собеседования

### Базовые вопросы
1. **Что такое Kafka и для чего она используется?**
   - Распределённая платформа потоковой передачи данных
   - Высокая пропускная способность, отказоустойчивость, хранение на диске

2. **Какие основные компоненты Kafka?**
   - Broker, Topic, Partition, Producer, Consumer, Consumer Group

3. **Что такое партиционирование?**
   - Разделение топика на несколько частей для параллельной обработки
   - Сообщения с одинаковым ключом всегда в одной партиции

### Продвинутые вопросы
4. **Как работает репликация?**
   - Leader-follower модель, in-sync replicas (ISR)
   - Автоматическое переключение на новый лидер при отказе

5. **Что такое exactly-once семантика?**
   - Идемпотентные producer'ы (`enable.idempotence=true`)
   - Транзакции (`transactional.id`)
   - Consumer читает только подтверждённые транзакции (`isolation.level=read_committed`)

6. **Как обеспечить порядок сообщений?**
   - Использование ключей для партиционирования
   - Порядок гарантирован только внутри одной партиции

### Практические вопросы
7. **Как настроить высокую доступность?**
   - Минимум 3 брокера, replication factor = 3
   - `min.insync.replicas=2`

8. **Как мониторить производительность?**
   - JMX метрики, Prometheus + Grafana, Kafka Manager / Conduktor

9. **Какие проблемы возникают при использовании Kafka?**
   - Проблемы с порядком сообщений при нескольких партициях
   - Consumer lag при медленной обработке
   - Сложность настройки exactly-once
