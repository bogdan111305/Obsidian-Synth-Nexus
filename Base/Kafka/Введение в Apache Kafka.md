## Что такое Apache Kafka

**Apache Kafka** — это распределённая система обмена сообщениями (distributed streaming platform), разработанная в LinkedIn и переданная в Apache Software Foundation. Kafka предназначена для обработки потоков данных в реальном времени и обеспечивает высокую пропускную способность, отказоустойчивость и горизонтальную масштабируемость.

### Ключевые характеристики:
- **Высокая пропускная способность**: способна обрабатывать миллионы сообщений в секунду
- **Отказоустойчивость**: репликация данных обеспечивает надёжность
- **Горизонтальная масштабируемость**: легко добавлять новые узлы
- **Сохраняемость**: данные хранятся на диске и не теряются при перезапуске
- **Распределённость**: работает в кластере из множества серверов

## История создания

### Предпосылки создания
Kafka была создана в LinkedIn в 2011 году для решения проблем с обработкой больших объёмов данных в реальном времени. Основные проблемы, которые решала Kafka:

1. **Медленная обработка данных**: существующие системы не справлялись с объёмами
2. **Сложность интеграции**: множество систем требовали разных интерфейсов
3. **Потеря данных**: отсутствие надёжного механизма доставки сообщений
4. **Масштабируемость**: сложность горизонтального масштабирования

### Развитие проекта
- **2011**: Создание в LinkedIn
- **2012**: Передача в Apache Incubator
- **2014**: Статус Apache Top-Level Project
- **2017**: Версия 1.0.0
- **2021**: Версия 3.0.0 с удалением зависимости от ZooKeeper
- **2023**: Версия 3.5.0 с улучшенной производительностью

## Основные преимущества

### 1. Высокая производительность
```java
// Kafka может обрабатывать до 2 миллионов сообщений в секунду
// на одном узле с SSD дисками
Properties props = new Properties();
props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(ProducerConfig.BATCH_SIZE_CONFIG, "16384");
props.put(ProducerConfig.LINGER_MS_CONFIG, "1");
props.put(ProducerConfig.BUFFER_MEMORY_CONFIG, "33554432");
```

### 2. Отказоустойчивость
- **Репликация**: каждая партиция реплицируется на несколько брокеров
- **Автоматическое восстановление**: при отказе узла автоматически выбирается новый лидер
- **Гарантии доставки**: настраиваемые уровни надёжности (at-least-once, exactly-once)

### 3. Горизонтальная масштабируемость
- **Партиционирование**: топики разделяются на партиции для параллельной обработки
- **Consumer Groups**: позволяют масштабировать потребление сообщений
- **Добавление узлов**: можно добавлять брокеры без остановки системы

### 4. Сохраняемость данных
```java
// Данные хранятся на диске и не теряются при перезапуске
// Настройка времени хранения
log.retention.hours=168  // 7 дней по умолчанию
log.retention.bytes=1073741824  // 1GB
```

## Архитектура Kafka

### Основные компоненты

#### 1. Broker (Брокер)
```java
// Брокер - это сервер Kafka, который хранит данные
// и обрабатывает запросы от producer'ов и consumer'ов
server.properties:
  broker.id=0
  listeners=PLAINTEXT://localhost:9092
  log.dirs=/tmp/kafka-logs
  num.partitions=1
  default.replication.factor=1
```

#### 2. Topic (Топик)
```java
// Топик - это категория или канал для сообщений
// Создание топика с несколькими партициями
kafka-topics.sh --create --topic my-topic \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 2
```

#### 3. Partition (Партиция)
```java
// Партиция - это упорядоченная последовательность сообщений
// Каждая партиция имеет уникальный идентификатор (offset)
// Сообщения в партиции упорядочены по времени
```

#### 4. Producer (Продюсер)
```java
// Producer отправляет сообщения в топики
KafkaProducer<String, String> producer = new KafkaProducer<>(props);
ProducerRecord<String, String> record = 
    new ProducerRecord<>("my-topic", "key", "value");
producer.send(record);
```

#### 5. Consumer (Консьюмер)
```java
// Consumer читает сообщения из топиков
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("my-topic"));
ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
```

#### 6. Consumer Group (Группа консьюмеров)
```java
// Consumer Group - это группа консьюмеров, которые совместно
// обрабатывают сообщения из топика
props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-consumer-group");
```

## Основные концепции

### 1. Партиционирование
```java
// Сообщения распределяются по партициям на основе ключа
// Сообщения с одинаковым ключом попадают в одну партицию
ProducerRecord<String, String> record = 
    new ProducerRecord<>("my-topic", "user-123", "message");
```

### 2. Репликация
```java
// Каждая партиция реплицируется на несколько брокеров
// Один брокер является лидером, остальные - репликами
// Настройка фактора репликации
kafka-topics.sh --create --topic my-topic \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 3
```

### 3. Offset (Смещение)
```java
// Offset - это уникальный идентификатор сообщения в партиции
// Consumer отслеживает свой прогресс через offset
consumer.commitSync();  // Сохранение текущего offset
```

### 4. Retention (Хранение)
```java
// Настройка времени хранения сообщений
log.retention.hours=168  // 7 дней
log.retention.bytes=1073741824  // 1GB
log.segment.bytes=1073741824  // Размер сегмента
```

## Сравнение с другими системами

### Kafka vs RabbitMQ
| Характеристика | Kafka | RabbitMQ |
|----------------|-------|----------|
| Модель | Публикация/Подписка | Очереди сообщений |
| Производительность | Очень высокая | Высокая |
| Сохраняемость | На диске | В памяти/на диске |
| Масштабируемость | Горизонтальная | Ограниченная |
| Сложность | Средняя | Низкая |

### Kafka vs Apache Pulsar
| Характеристика | Kafka | Pulsar |
|----------------|-------|--------|
| Архитектура | Монолитная | Многослойная |
| Geo-replication | Ручная настройка | Встроенная |
| Многоарендность | Ограниченная | Полная |
| Производительность | Высокая | Очень высокая |

## Вопросы для собеседования

### Базовые вопросы
1. **Что такое Kafka и для чего она используется?**
   - Распределённая система обмена сообщениями
   - Обработка потоков данных в реальном времени
   - Высокая пропускная способность и отказоустойчивость

2. **Какие основные компоненты Kafka вы знаете?**
   - Broker, Topic, Partition, Producer, Consumer, Consumer Group
   - ZooKeeper (в старых версиях), Schema Registry

3. **Что такое партиционирование в Kafka?**
   - Разделение топика на несколько частей
   - Параллельная обработка сообщений
   - Распределение нагрузки между consumer'ами

### Продвинутые вопросы
4. **Как работает репликация в Kafka?**
   - Leader-follower модель
   - In-sync replicas
   - Автоматическое восстановление при отказе

5. **Что такое exactly-once семантика?**
   - Идемпотентные producer'ы
   - Транзакции
   - Consumer group координация

6. **Как обеспечить порядок сообщений в Kafka?**
   - Использование ключей для партиционирования
   - Одна партиция = гарантированный порядок
   - Consumer group с одним consumer'ом

### Практические вопросы
7. **Как настроить высокую доступность в Kafka?**
   - Минимум 3 брокера
   - Replication factor = 3
   - Настройка min.insync.replicas

8. **Как мониторить производительность Kafka?**
   - JMX метрики
   - Prometheus + Grafana
   - Kafka Manager / Conduktor

9. **Какие проблемы могут возникнуть при использовании Kafka?**
   - N+1 проблема при чтении
   - Проблемы с порядком сообщений
   - Сложность настройки и эксплуатации
