> [!QUOTE] Git
> Распределённая система контроля версий. Каждый разработчик хранит **полную** историю репозитория локально — операции выполняются без сети.

## Конфигурация

```bash
# Глобальный пользователь (для всех проектов)
git config --global user.name "Pragmatic Programmer"
git config --global user.email "prag_prog@gmail.com"

# Цветной вывод
git config --global color.ui true

# Редактор по умолчанию
git config core.editor nano

# Просмотр настроек
git config --list
git config --list --global
git config --list --local

# Значение конкретного параметра
git config user.name
git config user.email
```

## Расположение конфиг-файлов

| Уровень | Путь |
|---|---|
| Системный (все пользователи) | `C:\Program Files\Git\etc\gitconfig` |
| Глобальный (текущий пользователь) | `%USERPROFILE%\.gitconfig` |
| Локальный (репозиторий) | `<репозиторий>\.git\config` |

> [!INFO] Windows
> Команды работы с файлами (`mkdir`, `ls`, `pwd`, `cat`) — это команды Linux-терминала. В Windows используйте **Git Bash**, а не стандартный `cmd`/`PowerShell`.

## Инициализация репозитория

```bash
mkdir first_project     # создать папку проекта
cd first_project/
git init                # инициализировать Git-репозиторий
ls -la                  # убедиться, что создалась папка .git/
```

## Основные команды

```bash
# Состояние рабочей директории и индекса
git status

# История коммитов
git log
git log --oneline       # компактный вид

# Добавить файл в индекс (staging area)
git add page1.html
git add .               # все изменённые файлы

# Сделать коммит
git commit              # откроет редактор для сообщения
git commit -m "add some new features"
git commit -a           # добавить все изменённые + коммит (без git add)
git commit -am "add some new features"

# Детали коммита
git show                # последний коммит
git show <commit_id>
git show --name-only <commit_id>   # только имена файлов
```

## Отмена изменений: git reset

> [!WARNING] git reset --hard удаляет данные безвозвратно
> Все несохранённые изменения в рабочей директории будут уничтожены. Проверьте `git status` перед выполнением.

```bash
# Сбросить индекс (staging area), сохраняя изменения в файлах
git reset
git reset <file_name>   # убрать конкретный файл из индекса

# Вернуть рабочую директорию к состоянию последнего коммита
git reset --hard

# Отменить последний коммит (изменения остаются в файлах)
git reset HEAD~1
git reset HEAD^         # то же самое

# Отменить несколько коммитов
git reset HEAD~2
git reset HEAD~3

# Отменить коммиты с удалением изменений
git reset HEAD~2 --hard

# Откат к конкретному коммиту
git reset --hard <commit_id>

# ORIG_HEAD — предыдущее значение HEAD (сохраняется после reset/merge)
cat .git/ORIG_HEAD
git reset ORIG_HEAD --hard   # вернуться к состоянию до операции
```

## Отмена изменений: git revert

> [!INFO] reset vs revert
> `git reset` переписывает историю — опасно для общих веток. `git revert` создаёт **новый коммит**, отменяющий изменения — безопасен для `main`/`master`.

```bash
# Справка
git revert --help

# Откатить последний коммит (создаст новый "обратный" коммит)
git revert HEAD
git revert HEAD --no-edit   # без ввода сообщения

# Откатить по хэшу или позиции
git revert 8ab14c
git revert HEAD~3

# Откатить диапазон коммитов
git revert 9e59768..4b2413e

# Откатить без создания коммита (изменения попадают в индекс)
git revert 9e59768 --no-commit

# Отменить незавершённый revert
git revert --abort
```

## Рекомендации по сообщениям коммита

- Тема — не длиннее 50 символов, без точки в конце
- Тема отвечает на вопрос: **"Что делает этот коммит?"**
- Пустая строка между темой и телом
- Тело — не длиннее 80 символов
- Тело объясняет **что** изменено (не "почему")
