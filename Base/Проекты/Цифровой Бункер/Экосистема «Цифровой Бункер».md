## 1. Архитектурные инварианты (нарушение = блокер)

| # | Принцип | Суть |
|---|---------|------|
| P1 | **Zero-Trust** | Каждый запрос несёт JWT (пользователь) или mTLS-сертификат (сервис). Смежность в Docker-сети не означает доверия. |
| P2 | **Envelope Encryption везде** | Сервер никогда не видит plaintext. Ключи — только в RAM, только `byte[]`, уничтожаются сразу после использования. |
| P3 | **Аппаратная привязка** | Master Key привязан к Titan M2 / Secure Enclave. NFC-кольцо — обязательный второй фактор. |
| P4 | **Код не содержит адресов** | Все хосты/порты/ключи — в `.env`. local→prod = смена `.env` без правки кода. |
| P5 | **mTLS с первого дня** | mTLS между сервисами включён с Phase 0. Добавить «потом» невозможно. |

## 4. Криптографические протоколы

### 4.1 Envelope Encryption (иерархия ключей)
```
Password + Salt(32B, HMAC-верифицированный)
  → Argon2id(m=65536KB, t=3, p=4) → Master Key (Secure Enclave / Titan M2)
  → AES-256-GCM decrypt(ECK) → Category Key (RAM only, Redis TTL=30s, bind=127.0.0.1)
  → AES-256-GCM decrypt(EDK) → Data Key (RAM only, stream-decrypt)
  → Tink StreamingAead (AES-256-GCM-HKDF-4KB) → Plaintext
```
После операции: `Arrays.fill(ck/dk, 0)` + `VarHandle.fullFence()` + `redis.del("ck:"+id)`

### 4.2 Верификация соли (MITM-защита)
- **Регистрация:** `salt_hmac = HMAC-SHA256(salt, SERVER_KEY)`, клиент хранит `salt_fp = SHA256(salt_hmac)` в Enclave
- **Каждый вход:** клиент проверяет `SHA256(salt_hmac) == salt_fp`; несовпадение → ABORT + предупреждение

### 4.3 WebAuthn / NFC-кольцо (FIDO2 CTAP2)
- Требования: CTAP2 (не U2F), user presence, signCount, ES256
- Clone detection: `if (response.signCount <= stored_signCount) → REJECT + audit WEBAUTHN_CLONE_DETECTED`
- JWT claims: `ES256, sub, iss="bunker-auth", hardware_verified=true, device_fp, exp=15min`

### 4.4 Category Key → Vault Redis
Клиент генерирует ephemeral X25519, вычисляет ECDH с `VAULT_STATIC_PUB_KEY`, шифрует CK через AES-GCM, отправляет на Vault. Vault кладёт CK в локальный Redis (bind=127.0.0.1, TTL=30s). CK через Sentinel **никогда** не проходит.

### 4.5 X3DH (Signal Protocol)
Публичные ключи Bob'а в PostgreSQL: `IK_B` (постоянный), `SPK_B` (ротация 7 дней, Ed25519 подпись), `OPK_B[]` (50 one-time, после использования DELETE).

```
Alice: verify Ed25519(IK_B, SPK_B_sig) → EK_A = X25519.generateKeyPair()
DH1..4 → MS = HKDF-SHA256(0xFF*32||DH1||DH2||DH3||DH4, zeros, "bunker-x3dh-v1", 32)
→ Double Ratchet (каждое сообщение = новый ключ)
```
Kafka — слепой курьер: `enc_payload` не декодируется, `sender_id = HMAC(userId, SERVER_KEY)`, логирование payload **запрещено**.

### 4.6 BIP39 Recovery (36 слов)
```
24 слова + 12 passphrase → PBKDF2-HMAC-SHA512 → seed
→ Argon2id(seed[0:32], static_salt, m=131072, t=4, p=4) → recovery_mk
→ расшифровать ECK категории "Recovery" → CK
recovery_mk → fill(0) немедленно
```

## 5. Правила написания кода (Java/JVM)

```java
// ❌ НИКОГДА для секретов
String s = new String(secretBytes);
String k = Base64.encode(secretBytes);

// ✅ ВСЕГДА
byte[] secret = getSecret();
// ... использование ...
Arrays.fill(secret, (byte) 0);
VarHandle.fullFence();

// ✅ Только Platform Threads для крипто (Virtual Thread сериализует стек на heap → ключ в dump)
private static final ExecutorService CRYPTO_EXEC =
    Executors.newFixedThreadPool(
        Runtime.getRuntime().availableProcessors(),
        Thread.ofPlatform().name("crypto-", 0).factory()
    );

// ✅ SecureRandom
SecureRandom.getInstanceStrong();  // не new SecureRandom()

// ✅ Streaming (файлы никогда целиком в heap)
StreamingAead sa = handle.getPrimitive(StreamingAead.class);
try (OutputStream enc = sa.newEncryptingStream(Files.newOutputStream(dest), aad)) {
    raw.transferTo(enc);  // 4KB chunks
} finally { Arrays.fill(dk, (byte) 0); VarHandle.fullFence(); }
```

**Secure Delete (3 прохода):**
```java
void secureDelete(Path path) throws IOException {
    long size = Files.size(path);
    try (FileChannel ch = FileChannel.open(path, StandardOpenOption.WRITE)) {
        ch.write(ByteBuffer.wrap(new byte[(int) size])); ch.force(true);
        ch.position(0);
        byte[] ones = new byte[(int) size]; Arrays.fill(ones, (byte) 0xFF);
        ch.write(ByteBuffer.wrap(ones)); ch.force(true);
        ch.position(0);
        byte[] rnd = SecureRandom.getInstanceStrong().generateSeed((int) size);
        ch.write(ByteBuffer.wrap(rnd)); ch.force(true);
        Arrays.fill(rnd, (byte) 0); VarHandle.fullFence();
    }
    Files.delete(path);
}
```

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username TEXT NOT NULL UNIQUE,
    argon2_salt BYTEA NOT NULL CHECK (length(argon2_salt) = 32),
    salt_hmac   BYTEA NOT NULL CHECK (length(salt_hmac) = 32),  -- HMAC-SHA256(salt, SERVER_KEY)
    is_locked BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at TIMESTAMPTZ,
    CONSTRAINT username_length CHECK (length(username) BETWEEN 3 AND 64),
    CONSTRAINT username_chars  CHECK (username ~ '^[a-zA-Z0-9_\-\.]+$')
);

CREATE TABLE webauthn_credentials (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    credential_id BYTEA NOT NULL UNIQUE,
    public_key_cose BYTEA NOT NULL,
    sign_count BIGINT NOT NULL DEFAULT 0,  -- монотонный, clone detection
    aaguid BYTEA,
    device_label TEXT,
    transports TEXT[],
    backed_up BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_used_at TIMESTAMPTZ,
    CONSTRAINT one_credential_per_user_device UNIQUE (user_id, aaguid)
);

CREATE TABLE refresh_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash BYTEA NOT NULL UNIQUE CHECK (length(token_hash) = 32),
    device_fp TEXT NOT NULL,
    expires_at TIMESTAMPTZ NOT NULL,
    revoked BOOLEAN NOT NULL DEFAULT FALSE,
    revoked_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

CREATE INDEX idx_refresh_tokens_user_active ON refresh_tokens(user_id, expires_at) WHERE revoked = FALSE;

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    eck BYTEA NOT NULL, eck_iv BYTEA NOT NULL CHECK (length(eck_iv) = 12),
    eck_tag BYTEA NOT NULL CHECK (length(eck_tag) = 16),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

-- ФАЙЛЫ

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    edk_tag BYTEA NOT NULL CHECK (length(edk_tag) = 16),

    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ, wiped_at TIMESTAMPTZ,

CREATE INDEX idx_file_metadata_user_category ON file_metadata(user_id, category_id, created_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_file_metadata_pending_wipe ON file_metadata(deleted_at) WHERE deleted_at IS NOT NULL AND wiped_at IS NULL;

-- ПАРОЛИ
CREATE TABLE vault_secrets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    category_id UUID NOT NULL REFERENCES categories(id),
    encrypted_blob BYTEA NOT NULL, blob_iv BYTEA NOT NULL CHECK (length(blob_iv) = 12),
    blob_tag BYTEA NOT NULL CHECK (length(blob_tag) = 16),
    edk BYTEA, edk_iv BYTEA, edk_tag BYTEA,

## 10. ClickHouse Schema

CREATE TABLE chat_messages (
    chat_id UUID, message_id UUID DEFAULT generateUUIDv4(),
    sender_hmac String,   -- HMAC(user_id, server_key), не raw UUID

) ENGINE = MergeTree()

  ORDER BY (chat_id, ts)
  TTL ts + INTERVAL 1 YEAR DELETE;

## 11. Kill Switch

```lua
-- Атомарный Lua script (один EVAL)
local locked = redis.call('SET', 'SYSTEM_LOCKED', '1', 'NX')
if locked then
    local keys = redis.call('KEYS', 'ck:*')
    for _, k in ipairs(keys) do redis.call('DEL', k) end
    redis.call('SET', 'SYSTEM_LOCKED', '1')
end
return locked
```

**Триггер:** `POST /auth/kill-switch {emergency_password}`
1. `bcrypt.verify(emergency_password, EMERGENCY_PASSWORD_HASH)` — fail → reject
2. `redis.eval(kill_switch.lua)` — SYSTEM_LOCKED=1 + DEL все `ck:*`
3. Gateway: 503 на все `/api/*` (Caffeine cache 1s)
4. Параллельно: revoke все refresh_tokens + Kafka produce `system-events/SYSTEM_LOCKED` + PG read-only
5. `audit_log: KILL_SWITCH`

**Разблокировка (только BIP39):** `POST /auth/recovery {words_24, passphrase_12, new_password}` → recovery_mk → verify ECK "Recovery" → reset PG + Redis + audit.

## 12. Phase 0 — Развёртывание

### WSL2 особенности
- Bind mounts: только Windows-пути (`C:/tmp/bunker`), не UNC
- Ключи `.key`: chmod **644** (иначе non-owner процессы не читают)
- Docker CLI: `/mnt/c/Program Files/Docker/.../docker.exe`

**WSL2 память** (`%USERPROFILE%\.wslconfig`):
```ini
[wsl2]
memory=24GB
processors=8
swap=4GB
```

### Подготовка файлов (WSL2, однократно и после gen-certs.sh --force)

## 13. Матрица угроз

| Угроза | Защита |
|---|---|
| Brute force пароля | Argon2id m=65536 KB, GPU-неэффективен |
| Кража телефона | MK в Secure Enclave: без биометрии не извлечь |
| Клонирование NFC-кольца | signCount ≤ stored → REJECT |
| MITM подмена соли | HMAC-SHA256 + TOFU fingerprint |
| Компрометация Sentinel | Redis только на Vault; данные зашифрованы на app-уровне |
| Redis forensics | TTL=30s, no persistence, bind=127.0.0.1, DEL после операции |
| JVM heap dump | Platform Thread + Arrays.fill + VarHandle.fullFence |
| SQL dump БД | Все данные зашифрованы; ключей в БД нет |
| Принуждение | Kill Switch: аварийный пароль → полная блокировка |
| Потеря кольца / устройства | BIP39 recovery (36 слов) |
