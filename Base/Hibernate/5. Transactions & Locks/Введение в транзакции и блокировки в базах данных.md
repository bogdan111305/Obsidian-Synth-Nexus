## 1. Что такое транзакция <a name="транзакция"></a>

**Транзакция** — это логическая единица работы с базой данных, которая должна быть выполнена полностью или не выполнена вовсе. Транзакции обеспечивают целостность данных при одновременной работе нескольких пользователей и сбоев.

## 2. ACID: основные свойства транзакций <a name="acid"></a>

- **Atomicity (Атомарность)**: все операции внутри транзакции выполняются как единое целое. Если одна операция не удалась — откатываются все.
- **Consistency (Согласованность)**: после завершения транзакции данные должны оставаться в согласованном состоянии.
- **Isolation (Изолированность)**: параллельные транзакции не должны мешать друг другу.
- **Durability (Долговечность)**: после фиксации транзакции изменения не теряются даже при сбоях.

## 3. Типовые проблемы изоляции транзакций <a name="проблемы-изоляции"></a>

### 1. Lost Update (Потерянное обновление)
- Две транзакции читают одну и ту же строку, изменяют и сохраняют — изменения одной теряются.

### 2. Dirty Read (Грязное чтение)
- Одна транзакция читает данные, изменённые, но не зафиксированные другой транзакцией.

### 3. Non-Repeatable Read (Неповторяемое чтение)
- Одна транзакция дважды читает одну и ту же строку, а между чтениями другая транзакция изменяет или удаляет эту строку.

### 4. Phantom Read (Фантомное чтение)
- Одна транзакция дважды выполняет запрос по условию, а между запросами другая транзакция добавляет/удаляет строки, подходящие под условие.

## 4. Уровни изоляции транзакций <a name="уровни-изоляции"></a>

| Уровень изоляции         | Lost Update | Dirty Read | Non-Repeatable Read | Phantom Read |
|-------------------------|-------------|------------|---------------------|--------------|
| Read Uncommitted        | +           | +          | +                   | +            |
| Read Committed          | +           | -          | +                   | +            |
| Repeatable Read         | +           | -          | -                   | +            |
| Serializable            | -           | -          | -                   | -            |

- **+** — проблема возможна
- **-** — проблема невозможна

### Описание уровней:
- **Read Uncommitted**: минимальная изоляция, возможны все проблемы.
- **Read Committed**: нельзя читать незакоммиченные данные (нет dirty read).
- **Repeatable Read**: нельзя изменять строки, прочитанные в транзакции (нет non-repeatable read).
- **Serializable**: максимальная изоляция, транзакции полностью изолированы.

## 5. Уровни изоляции в PostgreSQL <a name="pg-isolation"></a>

PostgreSQL поддерживает:
- **Read Committed** (по умолчанию)
- **Repeatable Read**
- **Serializable**

**Особенности:**
- В PostgreSQL Repeatable Read защищает от phantom read (реализовано через MVCC).
- Serializable реализован через Serializable Snapshot Isolation (SSI).

### Пример установки уровня изоляции:
```sql
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

## 6. Best practices <a name="best-practices"></a>
- Используйте минимально необходимый уровень изоляции для производительности
- Для критичных операций — Repeatable Read или Serializable
- Следите за длительностью транзакций (долгие транзакции блокируют ресурсы)
- Используйте явное управление транзакциями в коде
- Логируйте ошибки и откаты транзакций
