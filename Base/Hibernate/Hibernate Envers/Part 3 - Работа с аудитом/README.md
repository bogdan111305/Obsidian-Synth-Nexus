# Part 3: Работа с аудитом Hibernate Envers

Этот раздел содержит материалы по чтению, анализу и работе с аудитом в Hibernate Envers. Здесь рассматриваются практические аспекты получения и обработки данных аудита.

## Содержание раздела

### Основные темы

1. **[Класс AuditReaderFactory](./Класс AuditReaderFactory.md)** - Создание и настройка AuditReader
   - Получение экземпляра AuditReader
   - Конфигурация AuditReader
   - Основные методы фабрики
   - Практические примеры использования

2. **[Перемещение по ревизиям](./Перемещение по ревизиям.md)** - Навигация по истории изменений
   - Получение списка ревизий
   - Переход между ревизиями
   - Фильтрация ревизий
   - Поиск конкретных версий

3. **[Методы класса AuditReader](./Методы класса AuditReader.md)** - API для чтения аудита
   - Основные методы чтения
   - Поиск сущностей по ревизиям
   - Получение изменений
   - Работа с коллекциями и ассоциациями

4. **[Audit Queries](./Audit Queries.md)** - Запросы к аудиту
   - Создание запросов к аудиту
   - Фильтрация и сортировка
   - Агрегация данных
   - Сложные запросы

5. **[Revision rollback](./Revision rollback.md)** - Откат к предыдущим версиям
   - Механизм отката ревизий
   - Восстановление данных
   - Создание новых ревизий при откате
   - Практические сценарии

6. **[Практические примеры запросов](./Практические примеры запросов.md)** - Реальные сценарии использования
   - Анализ изменений пользователей
   - Отслеживание финансовых операций
   - Аудит безопасности
   - Генерация отчетов

### Ключевые концепции

- **AuditReaderFactory** - фабрика для создания AuditReader
- **AuditReader** - основной API для чтения аудита
- **Ревизии (Revisions)** - версии данных с метаинформацией
- **Audit Queries** - запросы к аудиту
- **Revision rollback** - откат к предыдущим версиям
- **Анализ изменений** - изучение истории изменений

### Практические навыки

После изучения этого раздела вы сможете:

- Создавать и настраивать AuditReader
- Навигировать по истории ревизий
- Выполнять сложные запросы к аудиту
- Откатывать изменения к предыдущим версиям
- Анализировать изменения в данных
- Генерировать отчеты по аудиту
