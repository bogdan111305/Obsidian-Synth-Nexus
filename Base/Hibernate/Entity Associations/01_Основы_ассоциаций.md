## <a name="введение"></a>Введение в ассоциации

Ассоциации сущностей в Hibernate позволяют моделировать отношения между объектами в Java-приложении и соответствующими таблицами в базе данных. Они охватывают связи типа "многие к одному" (`@ManyToOne`), "один ко многим" (`@OneToMany`), "один к одному" (`@OneToOne`) и "многие ко многим" (`@ManyToMany`), а также управление коллекциями, их сортировкой и производительностью.

### Ключевые концепции

- **Маппинг объектно-реляционных связей**: Преобразование связей между объектами в SQL-связи между таблицами
- **Управление загрузкой данных**: Контроль времени и способа загрузки связанных данных
- **Каскадирование операций**: Автоматическое применение операций к связанным сущностям
- **Производительность**: Оптимизация запросов и управления памятью

## <a name="концепция"></a>Концепция ассоциаций

Ассоциации в Hibernate представляют собой связи между сущностями, которые отражают отношения между таблицами в базе данных. Каждая ассоциация имеет:

### Кардинальность

Определяет количество связанных объектов:
- **Один-к-одному** (`@OneToOne`): Каждая сущность связана ровно с одной другой сущностью
- **Один-ко-многим** (`@OneToMany`): Одна сущность связана с множеством других сущностей
- **Многие-к-одному** (`@ManyToOne`): Множество сущностей связаны с одной сущностью
- **Многие-ко-многим** (`@ManyToMany`): Множество сущностей связаны с множеством других сущностей

### Направленность

- **Однонаправленная связь**: Связь доступна только с одной стороны
- **Двунаправленная связь**: Связь доступна с обеих сторон

### Стратегия загрузки

- **Ленивая загрузка** (`LAZY`): Данные загружаются только при обращении
- **Немедленная загрузка** (`EAGER`): Данные загружаются сразу при загрузке основной сущности

### Каскадирование

Автоматическое применение операций к связанным сущностям:
- Сохранение (`PERSIST`)
- Обновление (`MERGE`)
- Удаление (`REMOVE`)
- Отсоединение (`DETACH`)
- Обновление из БД (`REFRESH`)

## <a name="аннотации"></a>Аннотации для ассоциаций

### Основные аннотации

#### @ManyToOne
Связь "многие к одному" - несколько сущностей ссылаются на одну общую сущность.

```java
@ManyToOne
@JoinColumn(name = "company_id")
private Company company;
```

#### @OneToMany
Связь "один ко многим" - одна сущность связана с множеством других сущностей.

```java
@OneToMany(mappedBy = "department")
private List<Employee> employees;
```

#### @OneToOne
Связь "один к одному" - каждая сущность связана ровно с одной другой сущностью.

```java
@OneToOne
@JoinColumn(name = "profile_id")
private UserProfile profile;
```

#### @ManyToMany
Связь "многие ко многим" - множество сущностей связаны с множеством других сущностей.

```java
@ManyToMany
@JoinTable(name = "user_roles")
private Set<Role> roles;
```

### Вспомогательные аннотации

#### @JoinColumn
Настройка внешнего ключа для связи:

```java
@JoinColumn(
    name = "company_id",           // Имя столбца
    referencedColumnName = "id",   // Имя столбца в целевой таблице
    nullable = false,              // Может ли быть null
    unique = false                 // Должен ли быть уникальным
)
```

#### @JoinTable
Настройка промежуточной таблицы для связи "многие ко многим":

```java
@JoinTable(
    name = "user_roles",                    // Имя промежуточной таблицы
    joinColumns = @JoinColumn(name = "user_id"),      // Столбец для текущей сущности
    inverseJoinColumns = @JoinColumn(name = "role_id") // Столбец для связанной сущности
)
```

#### @OrderBy
Указание порядка сортировки элементов коллекции:

```java
@OneToMany(mappedBy = "user")
@OrderBy("createdAt DESC, id ASC")
private List<Post> posts;
```

#### @OrderColumn
Создание дополнительного столбца для хранения порядка элементов:

```java
@OneToMany(mappedBy = "user")
@OrderColumn(name = "post_order")
private List<Post> posts;
```

### Параметры аннотаций

#### fetch
Определяет стратегию загрузки:
- `FetchType.LAZY` (по умолчанию для коллекций)
- `FetchType.EAGER` (по умолчанию для @ManyToOne, @OneToOne)

#### cascade
Определяет типы каскадирования:
- `CascadeType.PERSIST`
- `CascadeType.MERGE`
- `CascadeType.REMOVE`
- `CascadeType.DETACH`
- `CascadeType.REFRESH`
- `CascadeType.ALL`

#### optional
Указывает, может ли связь быть null (только для @ManyToOne, @OneToOne):
- `true` (по умолчанию) - связь может быть null
- `false` - связь обязательна

#### mappedBy
Указывает поле в связанной сущности, которое управляет связью (для @OneToMany, @OneToOne):
```java
@OneToMany(mappedBy = "department")
private List<Employee> employees;
```
