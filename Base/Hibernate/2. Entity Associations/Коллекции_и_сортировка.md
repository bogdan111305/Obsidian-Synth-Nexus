## <a name="введение"></a>Введение в коллекции

Hibernate поддерживает различные типы коллекций для маппинга связей. Каждый тип коллекции имеет свои особенности и подходит для определенных сценариев использования.

### Ключевые концепции

- **Типы коллекций**: List, Set, Map, Bag
- **Сортировка**: @OrderBy и @OrderColumn
- **Индексация**: Управление порядком элементов
- **Производительность**: Выбор оптимального типа коллекции

---

## <a name="типы-коллекций"></a>Типы коллекций

### List - Упорядоченная коллекция

Упорядоченная коллекция с возможностью дублирования элементов.

#### Характеристики List:

- **Порядок**: Элементы сохраняют порядок вставки
- **Дублирование**: Разрешены дублирующиеся элементы
- **Индексация**: Доступ по индексу
- **Производительность**: Быстрые операции вставки и удаления

#### Пример с List:

```java
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    @OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
    @OrderBy("createdAt DESC")
    private List<Post> posts = new ArrayList<>();
    
    // Геттеры и сеттеры
}

@Entity
public class Post {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String title;
    private String content;
    
    @Column(name = "created_at")
    private LocalDateTime createdAt;
    
    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
    
    // Геттеры и сеттеры
}
```

### Set - Неупорядоченная коллекция

Неупорядоченная коллекция без дублирования элементов.

#### Характеристики Set:

- **Порядок**: Элементы не гарантируют порядок
- **Дублирование**: Дублирующиеся элементы запрещены
- **Производительность**: Быстрый поиск элементов
- **Уникальность**: Автоматическое удаление дубликатов

#### Пример с Set:

```java
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    @ManyToMany
    @JoinTable(
        name = "user_roles",
        joinColumns = @JoinColumn(name = "user_id"),
        inverseJoinColumns = @JoinColumn(name = "role_id")
    )
    private Set<Role> roles = new HashSet<>();
    
    // Геттеры и сеттеры
}

@Entity
public class Role {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @ManyToMany(mappedBy = "roles")
    private Set<User> users = new HashSet<>();
    
    // Геттеры и сеттеры
}
```

### Map - Коллекция пар ключ-значение

Коллекция пар ключ-значение с уникальными ключами.

#### Характеристики Map:

- **Ключи**: Уникальные ключи
- **Значения**: Могут дублироваться
- **Доступ**: Быстрый доступ по ключу
- **Порядок**: Может быть упорядоченным или неупорядоченным

#### Пример с Map:

```java
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    @OneToMany(mappedBy = "user")
    @MapKey(name = "id")
    private Map<Long, Post> posts = new HashMap<>();
    
    // Геттеры и сеттеры
}

@Entity
public class Post {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String title;
    
    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
    
    // Геттеры и сеттеры
}
```

### Bag - Неупорядоченная коллекция с дублированием

Неупорядоченная коллекция с возможностью дублирования элементов.

#### Характеристики Bag:

- **Порядок**: Элементы не гарантируют порядок
- **Дублирование**: Разрешены дублирующиеся элементы
- **Производительность**: Оптимизирована для больших коллекций
- **Использование**: Редко используется, предпочтительнее List

#### Пример с Bag:

```java
@Entity
public class Department {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @OneToMany(mappedBy = "department")
    private Collection<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}
```

## <a name="сортировка"></a>Сортировка и индексация

### @OrderBy - Сортировка по полям

Аннотация `@OrderBy` позволяет указать порядок сортировки элементов коллекции.

#### Базовая сортировка:

```java
@OneToMany(mappedBy = "user")
@OrderBy("createdAt DESC")
private List<Post> posts;
```

#### Сложная сортировка:

```java
@OneToMany(mappedBy = "user")
@OrderBy("createdAt DESC, id ASC")
private List<Post> posts;
```

#### Сортировка по связанным полям:

```java
@OneToMany(mappedBy = "department")
@OrderBy("name ASC")
private List<Employee> employees;
```

### @OrderColumn - Индексация порядка

Аннотация `@OrderColumn` создаёт дополнительный столбец для хранения порядка элементов.

#### Базовая индексация:

```java
@OneToMany(mappedBy = "user")
@OrderColumn(name = "post_order")
private List<Post> posts;
```

#### Настройка индексации:

```java
@OneToMany(mappedBy = "user")
@OrderColumn(
    name = "post_order",
    nullable = false
)
private List<Post> posts;
```

### Сравнение @OrderBy и @OrderColumn

| Характеристика | @OrderBy | @OrderColumn |
|----------------|----------|--------------|
| **Хранение** | Сортировка в запросе | Дополнительный столбец |
| **Производительность** | Медленнее для больших коллекций | Быстрее для больших коллекций |
| **Гибкость** | Легко изменить порядок | Сложнее изменить порядок |
| **Использование** | Для небольших коллекций | Для больших коллекций |

## <a name="примеры"></a>Практические примеры

### Пример 1: Сортировка постов пользователя

```java
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, unique = true)
    private String username;
    
    @OneToMany(mappedBy = "author", fetch = FetchType.LAZY)
    @OrderBy("createdAt DESC, title ASC")
    private List<Post> posts = new ArrayList<>();
    
    // Методы управления связью
    public void addPost(Post post) {
        posts.add(post);
        post.setAuthor(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
@Table(name = "posts")
public class Post {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "title", nullable = false)
    private String title;
    
    @Column(name = "content", columnDefinition = "TEXT")
    private String content;
    
    @Column(name = "created_at")
    private LocalDateTime createdAt;
    
    @ManyToOne
    @JoinColumn(name = "author_id", nullable = false)
    private User author;
    
    // Геттеры и сеттеры
}

// Использование
public class PostSortingExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        User user = em.find(User.class, 1L);
        
        // Посты автоматически отсортированы по дате создания (новые сначала)
        // и по названию (алфавитный порядок)
        for (Post post : user.getPosts()) {
            System.out.println(post.getTitle() + " - " + post.getCreatedAt());
        }
    }
}
```

### Пример 2: Индексация элементов списка

```java
@Entity
@Table(name = "playlists")
public class Playlist {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "name", nullable = false)
    private String name;
    
    @OneToMany(mappedBy = "playlist")
    @OrderColumn(name = "track_order")
    private List<Track> tracks = new ArrayList<>();
    
    // Методы управления связью
    public void addTrack(Track track) {
        tracks.add(track);
        track.setPlaylist(this);
    }
    
    public void moveTrack(int fromIndex, int toIndex) {
        Track track = tracks.remove(fromIndex);
        tracks.add(toIndex, track);
    }
    
    // Геттеры и сеттеры
}

@Entity
@Table(name = "tracks")
public class Track {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "title", nullable = false)
    private String title;
    
    @Column(name = "artist")
    private String artist;
    
    @ManyToOne
    @JoinColumn(name = "playlist_id")
    private Playlist playlist;
    
    // Геттеры и сеттеры
}

// Использование
public class PlaylistOrderExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        Playlist playlist = em.find(Playlist.class, 1L);
        
        // Треки сохраняют порядок в плейлисте
        for (int i = 0; i < playlist.getTracks().size(); i++) {
            Track track = playlist.getTracks().get(i);
            System.out.println((i + 1) + ". " + track.getTitle() + " - " + track.getArtist());
        }
        
        // Изменение порядка треков
        em.getTransaction().begin();
        playlist.moveTrack(0, 2); // Перемещаем первый трек на третье место
        em.getTransaction().commit();
    }
}
```

### Пример 3: Работа с Set и уникальностью

```java
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, unique = true)
    private String username;
    
    @ManyToMany
    @JoinTable(
        name = "user_roles",
        joinColumns = @JoinColumn(name = "user_id"),
        inverseJoinColumns = @JoinColumn(name = "role_id")
    )
    private Set<Role> roles = new HashSet<>();
    
    // Методы управления связью
    public void addRole(Role role) {
        roles.add(role);
        role.getUsers().add(this);
    }
    
    public void removeRole(Role role) {
        roles.remove(role);
        role.getUsers().remove(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
@Table(name = "roles")
public class Role {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "name", nullable = false, unique = true)
    private String name;
    
    @ManyToMany(mappedBy = "roles")
    private Set<User> users = new HashSet<>();
    
    // Геттеры и сеттеры
}

// Использование
public class RoleSetExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        User user = em.find(User.class, 1L);
        Role adminRole = em.find(Role.class, 1L);
        
        // Добавление роли (дубликаты автоматически игнорируются)
        user.addRole(adminRole);
        user.addRole(adminRole); // Второй вызов не добавит дубликат
        
        em.getTransaction().begin();
        em.merge(user);
        em.getTransaction().commit();
        
        System.out.println("User roles count: " + user.getRoles().size()); // 1
    }
}
```

### Пример 4: Работа с Map

```java
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, unique = true)
    private String username;
    
    @OneToMany(mappedBy = "user")
    @MapKey(name = "id")
    private Map<Long, Post> posts = new HashMap<>();
    
    // Методы управления связью
    public void addPost(Post post) {
        posts.put(post.getId(), post);
        post.setUser(this);
    }
    
    public Post getPost(Long postId) {
        return posts.get(postId);
    }
    
    // Геттеры и сеттеры
}

@Entity
@Table(name = "posts")
public class Post {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "title", nullable = false)
    private String title;
    
    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
    
    // Геттеры и сеттеры
}

// Использование
public class PostMapExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        User user = em.find(User.class, 1L);
        
        // Быстрый доступ к посту по ID
        Post post = user.getPost(5L);
        if (post != null) {
            System.out.println("Found post: " + post.getTitle());
        }
        
        // Итерация по всем постам
        for (Map.Entry<Long, Post> entry : user.getPosts().entrySet()) {
            System.out.println("Post " + entry.getKey() + ": " + entry.getValue().getTitle());
        }
    }
}
```

## <a name="практики"></a>Лучшие практики

### 1. Выбор правильного типа коллекции

```java
// Для упорядоченных данных с дублированием
@OneToMany(mappedBy = "user")
@OrderBy("createdAt DESC")
private List<Post> posts = new ArrayList<>();

// Для уникальных данных без порядка
@ManyToMany
@JoinTable(name = "user_roles")
private Set<Role> roles = new HashSet<>();

// Для быстрого доступа по ключу
@OneToMany(mappedBy = "user")
@MapKey(name = "id")
private Map<Long, Post> posts = new HashMap<>();
```

### 2. Использование @OrderBy для небольших коллекций

```java
@OneToMany(mappedBy = "user")
@OrderBy("createdAt DESC, title ASC")
private List<Post> posts;
```

### 3. Использование @OrderColumn для больших коллекций

```java
@OneToMany(mappedBy = "playlist")
@OrderColumn(name = "track_order")
private List<Track> tracks;
```

### 4. Правильное управление связями

```java
@Entity
public class User {
    @OneToMany(mappedBy = "user")
    private List<Post> posts = new ArrayList<>();
    
    public void addPost(Post post) {
        posts.add(post);
        post.setUser(this);
    }
    
    public void removePost(Post post) {
        posts.remove(post);
        post.setUser(null);
    }
}
```

### 5. Избегание N+1 проблем

```java
// Проблемный код
List<User> users = em.createQuery("SELECT u FROM User u", User.class).getResultList();
for (User user : users) {
    for (Post post : user.getPosts()) { // N+1 проблема
        System.out.println(post.getTitle());
    }
}

// Оптимизированный код
List<User> users = em.createQuery(
    "SELECT u FROM User u JOIN FETCH u.posts", User.class).getResultList();
for (User user : users) {
    for (Post post : user.getPosts()) { // Нет дополнительных запросов
        System.out.println(post.getTitle());
    }
}
```

### 6. Использование индексов для производительности

```sql
-- Создание индексов для улучшения производительности сортировки
CREATE INDEX idx_posts_user_created_at ON posts(user_id, created_at);
CREATE INDEX idx_posts_user_title ON posts(user_id, title);
```

### 7. Обработка исключений при работе с коллекциями

```java
public void processUserPosts(User user) {
    try {
        List<Post> posts = user.getPosts();
        if (posts != null && !posts.isEmpty()) {
            for (Post post : posts) {
                System.out.println(post.getTitle());
            }
        }
    } catch (LazyInitializationException e) {
        // Обработка случая, когда коллекция не загружена
        System.err.println("Posts collection not loaded");
    }
}
```
