## <a name="введение"></a>Введение в каскадирование

Каскадирование определяет, какие операции (например, сохранение, удаление) применяются к связанным сущностям автоматически. Это позволяет упростить код и обеспечить целостность данных.

### Принцип работы каскадирования

- **Операция над родительской сущностью**: Операция, выполненная над родительской сущностью, автоматически применяется к связанным сущностям
- **Упрощение кода**: Позволяет упростить код и обеспечить целостность данных
- **Осторожность**: Требует осторожности, чтобы избежать неожиданных операций

## <a name="cascade-types"></a>Cascade Types - Типы каскадирования

Перечисление `CascadeType` включает следующие значения:

### CascadeType.PERSIST

Сохранение родительской сущности сохраняет связанные сущности.

```java
@Entity
public class Department {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @OneToMany(mappedBy = "department", cascade = CascadeType.PERSIST)
    private List<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}

@Entity
public class Employee {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @ManyToOne
    @JoinColumn(name = "department_id")
    private Department department;
    
    // Геттеры и сеттеры
}
```

#### Использование:

```java
Department dept = new Department();
dept.setName("IT");

Employee emp1 = new Employee();
emp1.setName("John");
emp1.setDepartment(dept);

Employee emp2 = new Employee();
emp2.setName("Jane");
emp2.setDepartment(dept);

dept.getEmployees().add(emp1);
dept.getEmployees().add(emp2);

// Сохраняется только department, employees сохраняются автоматически
em.persist(dept);
```

### CascadeType.MERGE

Обновление родительской сущности обновляет связанные сущности.

```java
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String orderNumber;
    
    @OneToMany(mappedBy = "order", cascade = CascadeType.MERGE)
    private List<OrderItem> items = new ArrayList<>();
    
    // Геттеры и сеттеры
}

@Entity
public class OrderItem {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String product;
    private Integer quantity;
    
    @ManyToOne
    @JoinColumn(name = "order_id")
    private Order order;
    
    // Геттеры и сеттеры
}
```

#### Использование:

```java
Order order = em.find(Order.class, 1L);
order.setOrderNumber("ORD-001");

OrderItem item = order.getItems().get(0);
item.setQuantity(5);

// Обновляется только order, items обновляются автоматически
em.merge(order);
```

### CascadeType.REMOVE

Удаление родительской сущности удаляет связанные сущности.

```java
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String orderNumber;
    
    @OneToMany(mappedBy = "order", cascade = CascadeType.REMOVE)
    private List<OrderItem> items = new ArrayList<>();
    
    // Геттеры и сеттеры
}

@Entity
public class OrderItem {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String product;
    
    @ManyToOne
    @JoinColumn(name = "order_id")
    private Order order;
    
    // Геттеры и сеттеры
}
```

#### Использование:

```java
Order order = em.find(Order.class, 1L);

// При удалении заказа все его элементы удаляются автоматически
em.remove(order);
```

### CascadeType.DETACH

Отсоединение родительской сущности от сессии отсоединяет связанные сущности.

```java
@Entity
public class Department {
    @OneToMany(mappedBy = "department", cascade = CascadeType.DETACH)
    private List<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}
```

#### Использование:

```java
Department dept = em.find(Department.class, 1L);

// При отсоединении department все employees также отсоединяются
em.detach(dept);
```

### CascadeType.REFRESH

Обновление родительской сущности из базы обновляет связанные сущности.

```java
@Entity
public class Department {
    @OneToMany(mappedBy = "department", cascade = CascadeType.REFRESH)
    private List<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}
```

#### Использование:

```java
Department dept = em.find(Department.class, 1L);

// При обновлении department все employees также обновляются из БД
em.refresh(dept);
```

### CascadeType.ALL

Включает все вышеперечисленные операции.

```java
@Entity
public class Department {
    @OneToMany(mappedBy = "department", cascade = CascadeType.ALL)
    private List<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}
```

## <a name="примеры"></a>Практические примеры

### Пример 1: Каскадирование сохранения

```java
@Entity
public class Customer {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    private String email;
    
    @OneToMany(mappedBy = "customer", cascade = CascadeType.PERSIST)
    private List<Order> orders = new ArrayList<>();
    
    // Методы управления связью
    public void addOrder(Order order) {
        orders.add(order);
        order.setCustomer(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String orderNumber;
    private LocalDateTime orderDate;
    
    @ManyToOne
    @JoinColumn(name = "customer_id")
    private Customer customer;
    
    // Геттеры и сеттеры
}

// Использование
public class CascadePersistExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        em.getTransaction().begin();
        
        Customer customer = new Customer();
        customer.setName("John Doe");
        customer.setEmail("john@example.com");
        
        Order order1 = new Order();
        order1.setOrderNumber("ORD-001");
        order1.setOrderDate(LocalDateTime.now());
        
        Order order2 = new Order();
        order2.setOrderNumber("ORD-002");
        order2.setOrderDate(LocalDateTime.now());
        
        customer.addOrder(order1);
        customer.addOrder(order2);
        
        // Сохраняется только customer, orders сохраняются автоматически
        em.persist(customer);
        
        em.getTransaction().commit();
        
        System.out.println("Customer ID: " + customer.getId());
        System.out.println("Order 1 ID: " + order1.getId());
        System.out.println("Order 2 ID: " + order2.getId());
    }
}
```

### Пример 2: Каскадирование удаления

```java
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String orderNumber;
    
    @OneToMany(mappedBy = "order", cascade = CascadeType.REMOVE)
    private List<OrderItem> items = new ArrayList<>();
    
    // Методы управления связью
    public void addItem(OrderItem item) {
        items.add(item);
        item.setOrder(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
public class OrderItem {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String product;
    private Integer quantity;
    private BigDecimal price;
    
    @ManyToOne
    @JoinColumn(name = "order_id")
    private Order order;
    
    // Геттеры и сеттеры
}

// Использование
public class CascadeRemoveExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        em.getTransaction().begin();
        
        Order order = new Order();
        order.setOrderNumber("ORD-001");
        
        OrderItem item1 = new OrderItem();
        item1.setProduct("Laptop");
        item1.setQuantity(1);
        item1.setPrice(new BigDecimal("999.99"));
        
        OrderItem item2 = new OrderItem();
        item2.setProduct("Mouse");
        item2.setQuantity(2);
        item2.setPrice(new BigDecimal("29.99"));
        
        order.addItem(item1);
        order.addItem(item2);
        
        em.persist(order);
        em.getTransaction().commit();
        
        // Проверка создания
        System.out.println("Order created with ID: " + order.getId());
        System.out.println("Items created: " + order.getItems().size());
        
        // Удаление заказа
        em.getTransaction().begin();
        em.remove(order);
        em.getTransaction().commit();
        
        // Проверка удаления
        Order foundOrder = em.find(Order.class, order.getId());
        System.out.println("Order after deletion: " + foundOrder); // null
        
        OrderItem foundItem = em.find(OrderItem.class, item1.getId());
        System.out.println("Item after deletion: " + foundItem); // null
    }
}
```

### Пример 3: Каскадирование обновления

```java
@Entity
public class Department {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @OneToMany(mappedBy = "department", cascade = CascadeType.MERGE)
    private List<Employee> employees = new ArrayList<>();
    
    // Методы управления связью
    public void addEmployee(Employee employee) {
        employees.add(employee);
        employee.setDepartment(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
public class Employee {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    private String email;
    
    @ManyToOne
    @JoinColumn(name = "department_id")
    private Department department;
    
    // Геттеры и сеттеры
}

// Использование
public class CascadeMergeExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        // Создание данных
        em.getTransaction().begin();
        
        Department dept = new Department();
        dept.setName("IT");
        
        Employee emp1 = new Employee();
        emp1.setName("John Doe");
        emp1.setEmail("john@company.com");
        
        Employee emp2 = new Employee();
        emp2.setName("Jane Smith");
        emp2.setEmail("jane@company.com");
        
        dept.addEmployee(emp1);
        dept.addEmployee(emp2);
        
        em.persist(dept);
        em.getTransaction().commit();
        
        // Обновление с каскадированием
        em.getTransaction().begin();
        
        Department foundDept = em.find(Department.class, dept.getId());
        foundDept.setName("Information Technology");
        
        Employee foundEmp = foundDept.getEmployees().get(0);
        foundEmp.setEmail("john.doe@company.com");
        
        // Обновляется только department, employees обновляются автоматически
        em.merge(foundDept);
        
        em.getTransaction().commit();
        
        // Проверка обновления
        Department updatedDept = em.find(Department.class, dept.getId());
        System.out.println("Updated department name: " + updatedDept.getName());
        
        Employee updatedEmp = updatedDept.getEmployees().get(0);
        System.out.println("Updated employee email: " + updatedEmp.getEmail());
    }
}
```

### Пример 4: Комбинированное каскадирование

```java
@Entity
public class Company {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @OneToMany(mappedBy = "company", cascade = {CascadeType.PERSIST, CascadeType.MERGE, CascadeType.REMOVE})
    private List<Department> departments = new ArrayList<>();
    
    // Методы управления связью
    public void addDepartment(Department department) {
        departments.add(department);
        department.setCompany(this);
    }
    
    // Геттеры и сеттеры
}

@Entity
public class Department {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @ManyToOne
    @JoinColumn(name = "company_id")
    private Company company;
    
    @OneToMany(mappedBy = "department", cascade = CascadeType.ALL)
    private List<Employee> employees = new ArrayList<>();
    
    // Геттеры и сеттеры
}

@Entity
public class Employee {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    @ManyToOne
    @JoinColumn(name = "department_id")
    private Department department;
    
    // Геттеры и сеттеры
}

// Использование
public class CombinedCascadeExample {
    public static void main(String[] args) {
        EntityManager em = // получение EntityManager
        
        em.getTransaction().begin();
        
        Company company = new Company();
        company.setName("TechCorp");
        
        Department dept1 = new Department();
        dept1.setName("IT");
        
        Department dept2 = new Department();
        dept2.setName("HR");
        
        Employee emp1 = new Employee();
        emp1.setName("John Doe");
        
        Employee emp2 = new Employee();
        emp2.setName("Jane Smith");
        
        dept1.addEmployee(emp1);
        dept2.addEmployee(emp2);
        
        company.addDepartment(dept1);
        company.addDepartment(dept2);
        
        // Сохраняется только company, все остальное сохраняется автоматически
        em.persist(company);
        
        em.getTransaction().commit();
        
        System.out.println("Company ID: " + company.getId());
        System.out.println("Departments: " + company.getDepartments().size());
        System.out.println("Total employees: " + 
            company.getDepartments().stream()
                .mapToInt(d -> d.getEmployees().size())
                .sum());
    }
}
```

## <a name="практики"></a>Лучшие практики

### 1. Будьте осторожны с CascadeType.REMOVE

```java
// Осторожно используйте REMOVE каскадирование
@OneToMany(mappedBy = "order", cascade = CascadeType.REMOVE)
private List<OrderItem> items = new ArrayList<>();

// Альтернатива - использовать orphanRemoval
@OneToMany(mappedBy = "order", orphanRemoval = true)
private List<OrderItem> items = new ArrayList<>();
```

### 2. Используйте каскадирование только когда это логично

```java
// Логично - заказ и его элементы
@OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
private List<OrderItem> items = new ArrayList<>();

// Нелогично - пользователь и его роли
@ManyToMany(cascade = CascadeType.ALL)
private Set<Role> roles = new HashSet<>();
```

### 3. Тестируйте каскадные операции

```java
@Test
public void testCascadePersist() {
    EntityManager em = // получение EntityManager
    
    Department dept = new Department("IT");
    Employee emp = new Employee("John", "john@company.com");
    dept.addEmployee(emp);
    
    em.persist(dept);
    
    // Проверяем, что employee был сохранен
    Employee foundEmp = em.find(Employee.class, emp.getId());
    assertNotNull(foundEmp);
    assertEquals("John", foundEmp.getName());
}

@Test
public void testCascadeRemove() {
    EntityManager em = // получение EntityManager
    
    Order order = new Order("ORD-001");
    OrderItem item = new OrderItem("Product", 1, new BigDecimal("100"));
    order.addItem(item);
    
    em.persist(order);
    
    // Удаляем заказ
    em.remove(order);
    
    // Проверяем, что item был удален
    OrderItem foundItem = em.find(OrderItem.class, item.getId());
    assertNull(foundItem);
}
```

### 4. Используйте orphanRemoval для управления "сиротами"

```java
@Entity
public class Department {
    @OneToMany(mappedBy = "department", orphanRemoval = true)
    private List<Employee> employees = new ArrayList<>();
    
    public void removeEmployee(Employee employee) {
        employees.remove(employee);
        // Employee будет удален из БД благодаря orphanRemoval
    }
}
```

### 5. Комбинируйте каскадирование с правильным управлением связями

```java
@Entity
public class Department {
    @OneToMany(mappedBy = "department", cascade = CascadeType.ALL)
    private List<Employee> employees = new ArrayList<>();
    
    public void addEmployee(Employee employee) {
        employees.add(employee);
        employee.setDepartment(this);
    }
    
    public void removeEmployee(Employee employee) {
        employees.remove(employee);
        employee.setDepartment(null);
    }
}
```

### 6. Избегайте циклических каскадных операций

```java
// Проблемный код - может привести к бесконечному циклу
@Entity
public class User {
    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List<Post> posts = new ArrayList<>();
}

@Entity
public class Post {
    @ManyToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "user_id")
    private User user;
}

// Правильный подход - каскадирование только в одном направлении
@Entity
public class User {
    @OneToMany(mappedBy = "user", cascade = CascadeType.PERSIST)
    private List<Post> posts = new ArrayList<>();
}

@Entity
public class Post {
    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}
```
