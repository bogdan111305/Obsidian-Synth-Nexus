## Производительность и оптимизация

### 1. Оптимизация запросов аудита

```java
@Service
public class OptimizedAuditService {
    
    @Autowired
    private AuditReaderFactory auditReaderFactory;
    
    @Autowired
    private EntityManager entityManager;
    
    public List<Object> getOptimizedAuditHistory(String entityName, Serializable id) {
        AuditReader auditReader = auditReaderFactory.get(entityManager);
        
        // Использование пагинации
        List<Number> revisions = auditReader.getRevisions(entityName, id);
        
        // Ограничение количества ревизий
        int maxRevisions = 100;
        if (revisions.size() > maxRevisions) {
            revisions = revisions.subList(revisions.size() - maxRevisions, revisions.size());
        }
        
        return revisions.stream()
            .map(rev -> auditReader.find(entityName, id, rev))
            .collect(Collectors.toList());
    }
    
    public List<Object> getAuditHistoryWithFilter(String entityName, Serializable id, 
                                                LocalDateTime from, LocalDateTime to) {
        AuditReader auditReader = auditReaderFactory.get(entityManager);
        
        // Фильтрация по дате
        List<Number> allRevisions = auditReader.getRevisions(entityName, id);
        
        return allRevisions.stream()
            .map(rev -> {
                Object revision = auditReader.find(entityName, id, rev);
                // Дополнительная фильтрация
                return revision;
            })
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }
}
```

### 2. Батчинг аудита

```java
@Component
public class BatchedAuditService {
    
    private final List<AuditWorkUnit> auditBatch = new ArrayList<>();
    private final int batchSize = 50;
    
    @Transactional
    public void addToAuditBatch(AuditWorkUnit workUnit) {
        auditBatch.add(workUnit);
        
        if (auditBatch.size() >= batchSize) {
            processBatch();
        }
    }
    
    @Transactional
    public void processBatch() {
        if (auditBatch.isEmpty()) {
            return;
        }
        
        // Группировка по типам операций
        Map<String, List<AuditWorkUnit>> groupedByEntity = auditBatch.stream()
            .collect(Collectors.groupingBy(AuditWorkUnit::getEntityName));
        
        for (Map.Entry<String, List<AuditWorkUnit>> entry : groupedByEntity.entrySet()) {
            processEntityBatch(entry.getKey(), entry.getValue());
        }
        
        auditBatch.clear();
    }
    
    private void processEntityBatch(String entityName, List<AuditWorkUnit> workUnits) {
        // Оптимизированная обработка батча
        String auditTableName = getAuditTableName(entityName);
        
        // Подготовка batch insert
        StringBuilder sql = new StringBuilder();
        sql.append("INSERT INTO ").append(auditTableName);
        sql.append(" (id, rev, revtype, ...) VALUES ");
        
        List<Object[]> parameters = new ArrayList<>();
        
        for (AuditWorkUnit workUnit : workUnits) {
            parameters.add(new Object[]{workUnit.getId(), workUnit.getRevision(), 
                                      workUnit.getRevisionType()});
        }
        
        // Выполнение batch insert
        executeBatchInsert(sql.toString(), parameters);
    }
}
```

### 3. Кэширование аудита

```java
@Service
public class CachedAuditService {
    
    @Autowired
    private AuditReaderFactory auditReaderFactory;
    
    @Autowired
    private CacheManager cacheManager;
    
    public List<Object> getCachedAuditHistory(String entityName, Serializable id) {
        String cacheKey = "audit_" + entityName + "_" + id;
        
        Cache cache = cacheManager.getCache("auditCache");
        List<Object> cachedResult = cache.get(cacheKey, List.class);
        
        if (cachedResult != null) {
            return cachedResult;
        }
        
        // Получение данных из базы
        List<Object> auditHistory = getAuditHistoryFromDatabase(entityName, id);
        
        // Кэширование результата
        cache.put(cacheKey, auditHistory);
        
        return auditHistory;
    }
    
    private List<Object> getAuditHistoryFromDatabase(String entityName, Serializable id) {
        AuditReader auditReader = auditReaderFactory.get(entityManager);
        List<Number> revisions = auditReader.getRevisions(entityName, id);
        
        return revisions.stream()
            .map(rev -> auditReader.find(entityName, id, rev))
            .collect(Collectors.toList());
    }
}
```

### 4. Асинхронный аудит

```java
@Service
public class AsyncAuditService {
    
    @Autowired
    private ExecutorService auditExecutor;
    
    @Autowired
    private AuditRepository auditRepository;
    
    @Async("auditExecutor")
    public CompletableFuture<Void> auditAsync(Object entity, String operation) {
        return CompletableFuture.runAsync(() -> {
            try {
                performAudit(entity, operation);
            } catch (Exception e) {
                log.error("Ошибка асинхронного аудита", e);
            }
        }, auditExecutor);
    }
    
    private void performAudit(Object entity, String operation) {
        // Логика аудита
        AuditRecord record = new AuditRecord();
        record.setEntityType(entity.getClass().getSimpleName());
        record.setOperation(operation);
        record.setTimestamp(LocalDateTime.now());
        record.setData(JsonUtils.toJson(entity));
        
        auditRepository.save(record);
    }
}

@Configuration
public class AuditAsyncConfig {
    
    @Bean("auditExecutor")
    public ExecutorService auditExecutor() {
        return Executors.newFixedThreadPool(5);
    }
}
```

### 2. Аудит с валидацией

```java
@Service
public class ValidatedAuditService {
    
    @Autowired
    private Validator validator;
    
    @Autowired
    private AuditService auditService;
    
    @Transactional
    public void auditWithValidation(Object entity, String operation) {
        // Валидация перед аудитом
        Set<ConstraintViolation<Object>> violations = validator.validate(entity);
        
        if (violations.isEmpty()) {
            // Стандартный аудит
            auditService.audit(entity, operation);
        } else {
            // Аудит с ошибками валидации
            auditService.auditWithValidationErrors(entity, operation, violations);
            
            // Логирование ошибок валидации
            logValidationErrors(violations);
        }
    }
    
    private void logValidationErrors(Set<ConstraintViolation<Object>> violations) {
        violations.forEach(violation -> 
            log.error("Validation error: {} - {}", violation.getPropertyPath(), violation.getMessage())
        );
    }
}
```

```java
@Service
public class EncryptedAuditService {
    
    @Autowired
    private EncryptionService encryptionService;

    private AuditService auditService;
    
    @Transactional
    public void auditWithEncryption(Object entity, String operation) {

        // Аудит зашифрованных данных
        auditService.audit(encryptedEntity, operation);

            User user = (User) entity;
            user.setPassword(encryptionService.encrypt(user.getPassword()));
            user.setEmail(encryptionService.encrypt(user.getEmail()));
        }
        
        return entity;
    }
}
```

## Мониторинг и метрики

### Сбор метрик производительности

```java
@Component
public class AuditMetricsCollector {
    
    private final MeterRegistry meterRegistry;
    private final Timer auditTimer;
    private final Counter auditCounter;
    private final Gauge auditQueueSize;
    
    public AuditMetricsCollector(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        this.auditTimer = Timer.builder("envers.audit.duration")
                              .description("Время выполнения аудита")
                              .register(meterRegistry);
        this.auditCounter = Counter.builder("envers.audit.operations")
                                  .description("Количество операций аудита")
                                  .register(meterRegistry);
        this.auditQueueSize = Gauge.builder("envers.audit.queue.size")
                                   .description("Размер очереди аудита")
                                   .register(meterRegistry, this, AuditMetricsCollector::getQueueSize);
    }
    
    public void recordAuditOperation(String entityType, String operation, long duration) {
        auditTimer.record(duration, TimeUnit.MILLISECONDS);
        auditCounter.increment();
        
        // Метрики по типам сущностей
        Counter.builder("envers.audit.by.entity")
               .tag("entity", entityType)
               .tag("operation", operation)
               .register(meterRegistry)
               .increment();
    }
    
    private double getQueueSize() {
        // Возврат размера очереди аудита
        return 0.0;
    }
}
```

## Заключение

Продвинутые сценарии аудита в Hibernate Envers предоставляют мощные возможности для решения сложных бизнес-требований:

- **Гибкость**: Поддержка сложных ассоциаций и вложенных объектов
- **Производительность**: Оптимизация через батчинг, кэширование и асинхронность
- **Безопасность**: Условный аудит и шифрование чувствительных данных
- **Соответствие**: Интеграция с бизнес-правилами и валидацией
- **Мониторинг**: Сбор метрик и статистики производительности

При реализации продвинутых сценариев аудита важно помнить о производительности, безопасности и соответствии бизнес-требованиям.
