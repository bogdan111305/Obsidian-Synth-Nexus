# Интерфейс Repository

## Обзор

Интерфейс `Repository` является базовым интерфейсом в Spring Data JPA, который предоставляет базовую функциональность для работы с сущностями.

## Основные возможности

### Базовые CRUD операции
- `save(S entity)` - сохранение сущности
- `findById(ID id)` - поиск по ID
- `findAll()` - получение всех сущностей
- `delete(T entity)` - удаление сущности
- `count()` - подсчет количества записей

### Пример использования

```java
@Repository
public interface UserRepository extends Repository<User, Long> {
    // Базовые методы наследуются автоматически
}
```

## Иерархия интерфейсов

```
Repository (базовый)
    ↓
CrudRepository (добавляет CRUD операции)
    ↓
PagingAndSortingRepository (добавляет пагинацию и сортировку)
    ↓
JpaRepository (специфичный для JPA)
```

## Преимущества использования Repository

1. **Абстракция доступа к данным** - скрывает детали реализации
2. **Типобезопасность** - работает с конкретными типами сущностей
3. **Автоматическая генерация реализации** - Spring создает реализацию автоматически
4. **Единообразие API** - стандартизированный подход к работе с данными

## Настройка

```java
@Configuration
@EnableJpaRepositories(basePackages = "com.example.repository")
public class JpaConfig {
    // конфигурация
}
```

## Лучшие практики

1. **Именование** - используйте суффикс `Repository` для интерфейсов
2. **Пакеты** - размещайте в отдельном пакете `repository`
3. **Аннотации** - используйте `@Repository` для явного указания
4. **Типизация** - всегда указывайте типы сущности и ID
