## @Lazy

Представим ситуацию: у нас есть супермен, который когда устанет может захотеть использовать бластер.

``` Java
@Component
public class Superman{
	@Autowired
	public Bluster bluster;
}

```

Но мы хотим давать покупать для него бластер (т.е. создавать) только когда он ему реально пригодился, потому что такой ситуации может не произойти.

В этом нам может помочь аннотация <i style="color:#f3ff00">@Lazy</i>

#### Неправильное использование @Lazy: 

``` Java
@Component
@Lazy
public class Bluster{
	...
}
```

Проблема в том что, когда spring увидит @Autowired над bluster в каком-нибудь другом компоненте, он решит, что теперь Bluster нужен и создаст его. То есть фактически ничего не изменится и это логично.

#### Как решали эту проблему раньше? 

Раньше injection целый контекст в компонент, и вручную через getBean() забирали его, когда он оказывался нужен - это извращение, но правильный способ под капотом, по сути делает что-то похожее.

#### Правильное использование

``` Java
@Component
public class Superman{
	@Autowired 
	@Lazy
	public Bluster bluster;
}
```

Теперь в место Bluster будет создана proxy и injection вместо него. При обращении bluster она обратится к контексту и выполнит необходимые действия.
