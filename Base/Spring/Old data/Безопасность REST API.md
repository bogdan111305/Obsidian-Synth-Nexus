# Безопасность REST API

## OAuth 2

OAuth 2 - богатая спецификация безопасности, поддерживающая много способов её использования.

Процесс представленный на изображении ниже называется *предоставление кода авторизации*

В число других процессов авторизаций, поддерживаемыз OAuth 2 также входит:
 - *неявное предоставление разрешений* - браузер также переадресовывается на сервер авторизации, но при обратной переодресации вместо кода авторизации неявно поставляется токен. (not recommended)
 - *предоставление учётных данных(пароля) пользователя* - не предусматривает переадресации, получает данные и обмениваает их на токен.
 - *предоставление учётных данных клиента* - похоже на предыдущий, только сервер сам посылает запос к серверу авторизации.
