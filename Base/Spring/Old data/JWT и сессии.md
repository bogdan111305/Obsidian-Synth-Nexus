## Сессии 

1. Клиент отправляет логин и пароль 
2. Сервер создаёт сессию, и посылает её id в куках обратно клиенту
	Как сервер хранит сессию: 
	   - в памяти
	   - в БД
	   - в кэше  (например Redis)
3. Клиент подкладывает id сессии при каждом запросе также в куках, за счёт чего сервер его идентифицирует 
4. Когда клиент делает /logout сервер удаляет данные о сессии (чистит её)

Сессии это *stateful* подход - то есть они хранятся на стороне сервера

## Токены  

- Хранятся на стороне клиента, сервер у себя их как правило не хранит.
Он иожет их хранить только для более корректного обновления refresh токена.  Однако в процессе аутентификации, он не пользуется сохранёнными токенами. 

- Подписываются секретом (SHA256 и проч.)

- Отправляются клиентом в заголовке Authorization 

- Часто хранятся в local/session storage - что является не безопасным подходом, так как Js имеет к ним прямой доступ (XSS -атаки). Лучше хранить в httpOnly cookie
