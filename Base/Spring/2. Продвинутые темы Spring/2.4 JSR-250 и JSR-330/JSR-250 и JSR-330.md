# JSR-250 и JSR-330

JSR (Java Specification Request) - это спецификации Java, которые определяют стандартные интерфейсы и аннотации. Spring Framework предоставляет реализацию этих спецификаций, что обеспечивает совместимость с Java EE стандартами.

## JSR-250 Common Annotations

### Что такое JSR-250?

JSR-250 (Common Annotations for the Java Platform) определяет набор общих аннотаций, которые могут использоваться в различных Java технологиях, включая Spring Framework.

### Основные аннотации JSR-250

#### @PostConstruct

Выполняется после создания бина и внедрения всех зависимостей:

```java
import javax.annotation.PostConstruct;

@Component
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    @PostConstruct
    public void init() {
        System.out.println("UserService initialized with @PostConstruct");
        // Инициализация ресурсов
        validateConfiguration();
    }
    
    private void validateConfiguration() {
        // Проверка конфигурации
    }
}
```

#### @PreDestroy

Выполняется перед уничтожением бина:

```java
import javax.annotation.PreDestroy;

@Component
public class DatabaseConnection {
    
    private Connection connection;
    
    @PostConstruct
    public void connect() {
        // Установка соединения с БД
        connection = DriverManager.getConnection("jdbc:mysql://localhost:3306/test");
    }
    
    @PreDestroy
    public void disconnect() {
        if (connection != null) {
            try {
                connection.close();
                System.out.println("Database connection closed");
            } catch (SQLException e) {
                // Логирование ошибки
            }
        }
    }
}
```

#### @Resource

Внедрение зависимостей по имени:

```java
import javax.annotation.Resource;

@Component
public class EmailService {
    
    @Resource(name = "smtpMailSender")
    private MailSender mailSender;
    
    @Resource(name = "notificationTemplate")
    private Template notificationTemplate;
    
    public void sendEmail(String to, String subject, String content) {
        // Отправка email
    }
}
```

#### @ManagedBean

Аналог `@Component` из Spring:

```java
import javax.annotation.ManagedBean;

@ManagedBean
public class OrderService {
    
    public void processOrder(Order order) {
        // Обработка заказа
    }
}
```

#### @Priority

Указание приоритета бина:

```java
import javax.annotation.Priority;

@Component
@Priority(1)
public class HighPriorityService implements BusinessService {
    
    @Override
    public void process() {
        System.out.println("High priority processing");
    }
}

@Component
@Priority(2)
public class LowPriorityService implements BusinessService {
    
    @Override
    public void process() {
        System.out.println("Low priority processing");
    }
}
```

#### @Generated

Пометка сгенерированных классов:

```java
import javax.annotation.Generated;

@Generated("com.example.Generator")
public class GeneratedUserDto {
    
    private String name;
    private String email;
    
    // Геттеры и сеттеры
}
```

## JSR-330 Dependency Injection

### Что такое JSR-330?

JSR-330 (Dependency Injection for Java) определяет стандартную спецификацию для dependency injection в Java. Spring Framework полностью поддерживает эту спецификацию.

### Основные аннотации JSR-330

#### @Inject

Аналог `@Autowired`:

```java
import javax.inject.Inject;

@Component
public class UserService {
    
    @Inject
    private UserRepository userRepository;
    
    @Inject
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    public void setUserRepository(@Inject UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

#### @Named

Аналог `@Component` с именем:

```java
import javax.inject.Named;

@Named("userService")
public class UserService {
    
    public User findById(Long id) {
        // Поиск пользователя
        return null;
    }
}
```

#### @Qualifier

Указание конкретной реализации:

```java
import javax.inject.Qualifier;

@Qualifier
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.FIELD, ElementType.METHOD, ElementType.TYPE, ElementType.PARAMETER})
public @interface Premium {
}

@Component
@Premium
public class PremiumUserService implements UserService {
    
    @Override
    public User findById(Long id) {
        // Премиум логика
        return null;
    }
}

@Component
public class UserController {
    
    @Inject
    @Premium
    private UserService userService;
}
```

#### @Singleton

Аналог Spring singleton scope:

```java
import javax.inject.Singleton;

@Singleton
public class ConfigurationService {
    
    private final Map<String, String> config = new HashMap<>();
    
    public String getConfig(String key) {
        return config.get(key);
    }
}
```

#### @Provider

Создание провайдеров для сложных объектов:

```java
import javax.inject.Provider;

@Component
public class DatabaseConnectionProvider implements Provider<Connection> {
    
    @Override
    public Connection get() {
        // Создание нового соединения
        return createConnection();
    }
    
    private Connection createConnection() {
        // Логика создания соединения
        return null;
    }
}

@Component
public class UserRepository {
    
    @Inject
    private Provider<Connection> connectionProvider;
    
    public User findById(Long id) {
        Connection connection = connectionProvider.get();
        try {
            // Использование соединения
            return null;
        } finally {
            // Закрытие соединения
            connection.close();
        }
    }
}
```

## Сравнение с Spring аннотациями

### Таблица соответствий

| JSR-250/330 | Spring | Описание |
|-------------|--------|----------|
| `@Inject` | `@Autowired` | Внедрение зависимостей |
| `@Named` | `@Component` | Именованный компонент |
| `@Singleton` | `@Scope("singleton")` | Singleton scope |
| `@PostConstruct` | `@PostConstruct` | Инициализация |
| `@PreDestroy` | `@PreDestroy` | Уничтожение |
| `@Resource` | `@Autowired` + `@Qualifier` | Внедрение по имени |
| `@Qualifier` | `@Qualifier` | Указание реализации |

### Примеры использования

#### Смешанное использование

```java
import javax.inject.Inject;
import javax.inject.Named;
import org.springframework.stereotype.Component;

@Component
public class MixedService {
    
    // JSR-330
    @Inject
    @Named("userRepository")
    private UserRepository userRepository;
    
    // Spring
    @Autowired
    private EmailService emailService;
    
    // JSR-250
    @PostConstruct
    public void init() {
        System.out.println("Service initialized");
    }
}
```

#### Конфигурация

```java
@Configuration
public class JsrConfig {
    
    @Bean
    @Named("userRepository")
    public UserRepository userRepository() {
        return new JpaUserRepository();
    }
    
    @Bean
    @Named("emailService")
    public EmailService emailService() {
        return new SmtpEmailService();
    }
}
```

## Практические примеры

### 1. Сервис с жизненным циклом

```java
import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;
import javax.inject.Inject;
import javax.inject.Named;

@Named("orderService")
public class OrderService {
    
    @Inject
    private OrderRepository orderRepository;
    
    @Inject
    private NotificationService notificationService;
    
    private ExecutorService executorService;
    
    @PostConstruct
    public void initialize() {
        // Инициализация пула потоков
        executorService = Executors.newFixedThreadPool(10);
        System.out.println("OrderService initialized");
    }
    
    @PreDestroy
    public void cleanup() {
        // Очистка ресурсов
        if (executorService != null) {
            executorService.shutdown();
            try {
                if (!executorService.awaitTermination(60, TimeUnit.SECONDS)) {
                    executorService.shutdownNow();
                }
            } catch (InterruptedException e) {
                executorService.shutdownNow();
            }
        }
        System.out.println("OrderService destroyed");
    }
    
    public void processOrder(Order order) {
        // Обработка заказа
        orderRepository.save(order);
        notificationService.notifyUser(order.getUserId());
    }
}
```

### 2. Конфигурация с профилями

```java
import javax.inject.Inject;
import javax.inject.Named;

@Configuration
public class DatabaseConfig {
    
    @Bean
    @Named("dataSource")
    @Profile("dev")
    public DataSource devDataSource() {
        return new SimpleDriverDataSource();
    }
    
    @Bean
    @Named("dataSource")
    @Profile("prod")
    public DataSource prodDataSource() {
        return new ComboPooledDataSource();
    }
    
    @Bean
    @Inject
    public JdbcTemplate jdbcTemplate(@Named("dataSource") DataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
}
```

### 3. Фабрика объектов

```java
import javax.inject.Provider;

@Component
public class ConnectionFactory {
    
    @Inject
    private Provider<Connection> connectionProvider;
    
    public void executeInTransaction(Runnable task) {
        Connection connection = connectionProvider.get();
        try {
            connection.setAutoCommit(false);
            task.run();
            connection.commit();
        } catch (Exception e) {
            connection.rollback();
            throw new RuntimeException("Transaction failed", e);
        } finally {
            connection.close();
        }
    }
}
```

### 4. Условная конфигурация

```java
import javax.inject.Inject;
import javax.inject.Named;

@Component
public class NotificationService {
    
    @Inject
    @Named("emailSender")
    private MessageSender emailSender;
    
    @Inject
    @Named("smsSender")
    private MessageSender smsSender;
    
    public void sendNotification(User user, String message, NotificationType type) {
        MessageSender sender = getSender(type);
        sender.send(user.getContact(), message);
    }
    
    private MessageSender getSender(NotificationType type) {
        switch (type) {
            case EMAIL:
                return emailSender;
            case SMS:
                return smsSender;
            default:
                throw new IllegalArgumentException("Unknown notification type");
        }
    }
}
```

## Лучшие практики

### 1. Выбор между Spring и JSR аннотациями

```java
// Используйте JSR аннотации для:
// - Совместимости с Java EE
// - Переносимости кода
// - Следования стандартам

// Используйте Spring аннотации для:
// - Специфичных возможностей Spring
// - Более богатого функционала
// - Интеграции с Spring экосистемой
```

### 2. Смешанное использование

```java
@Component
public class BestPracticeService {
    
    // JSR-330 для базового DI
    @Inject
    private UserRepository userRepository;
    
    // Spring для специфичных возможностей
    @Autowired
    @Qualifier("cachedUserRepository")
    private UserRepository cachedRepository;
    
    // JSR-250 для жизненного цикла
    @PostConstruct
    public void init() {
        // Инициализация
    }
    
    @PreDestroy
    public void cleanup() {
        // Очистка
    }
}
```

### 3. Тестирование

```java
@SpringBootTest
class JsrServiceTest {
    
    @Inject
    private UserService userService;
    
    @MockBean
    @Named("userRepository")
    private UserRepository userRepository;
    
    @Test
    void shouldInjectDependencies() {
        // Тест внедрения зависимостей
        assertThat(userService).isNotNull();
    }
}
```

### 4. Конфигурация

```java
@Configuration
@EnableJsr250Support
public class JsrConfiguration {
    
    @Bean
    @Named("defaultUserService")
    public UserService defaultUserService() {
        return new DefaultUserService();
    }
    
    @Bean
    @Named("premiumUserService")
    public UserService premiumUserService() {
        return new PremiumUserService();
    }
}
```
