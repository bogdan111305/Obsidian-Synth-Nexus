## 1. Что такое JDBC и ORM? <a name="что-это"></a>
- **JDBC** — низкоуровневый API для работы с реляционными БД через SQL
- **ORM** (Hibernate, JPA, MyBatis) — абстракция для работы с БД через объекты, автоматическое сопоставление классов и таблиц

## 2. Таблица сравнения <a name="таблица"></a>
| Критерий         | JDBC                | ORM (Hibernate, JPA)      |
|------------------|---------------------|---------------------------|
| Производительность | Высокая (ручной контроль) | Высокая (кэш, batch) |
| Простота         | Требует много кода  | Меньше кода, декларативно |
| Гибкость         | Максимальная        | Ограничена абстракцией    |
| Транзакции       | Ручное управление   | Автоматизация             |
| Миграции         | Нет                 | Есть (Liquibase, Flyway)  |
| Поддержка БД     | Любая               | Зависит от реализации     |
| Кэширование      | Нет                 | Есть                      |
| Работа с JOIN    | Явно через SQL      | Через связи/аннотации     |
| Batch            | Ручная настройка    | Автоматизация             |
| Ленивая загрузка | Нет                 | Есть                      |

## 3. Плюсы и минусы каждого подхода <a name="плюсы-минусы"></a>
### JDBC
+ Максимальный контроль
+ Прозрачность SQL
+ Минимум магии
- Много шаблонного кода
- Ручное управление транзакциями, ресурсами
- Нет кэширования, миграций

### ORM
+ Быстрая разработка, меньше кода
+ Кэширование, миграции, ленивые коллекции
+ Автоматизация транзакций
- Сложность отладки
- Неочевидные SQL-запросы
- Overhead при сложных запросах

## 4. Когда использовать JDBC, когда ORM? <a name="когда"></a>
- JDBC: простые задачи, максимальный контроль, интеграция с legacy, критичные к производительности участки
- ORM: сложные доменные модели, кэширование, миграции, быстрое прототипирование
- Можно комбинировать: ORM для бизнес-логики, JDBC для сложных/оптимизированных запросов

## 5. Best practices <a name="best-practices"></a>
- Не бойтесь использовать оба подхода в одном проекте
- Для сложных отчётов — используйте JDBC/named queries
- Для CRUD и бизнес-логики — ORM
- Логируйте SQL и профилируйте
