## 1. Что такое JDBC? <a name="что-такое-jdbc"></a>

JDBC — стандартный API Java для работы с реляционными базами данных. Позволяет выполнять SQL-запросы, управлять транзакциями, работать с метаданными и поддерживает портативность между СУБД.

- **Портативность**: работает с любой СУБД при наличии драйвера
- **Интеграция**: поддержка транзакций, connection pool, batch

## 2. Архитектура и компоненты <a name="архитектура"></a>

- **DriverManager**: загрузка драйверов, создание соединений
- **Connection**: сессия с БД
- **Statement/PreparedStatement/CallableStatement**: выполнение SQL
- **ResultSet**: результаты запросов
- **SQLException**: обработка ошибок

## 3. Типы драйверов JDBC <a name="типы-драйверов"></a>

| Тип | Описание | Пример |
|-----|----------|--------|
| 1   | JDBC-ODBC Bridge (устарел) | sun.jdbc.odbc.JdbcOdbcDriver |
| 2   | Native-API/partly Java     | Oracle OCI |
| 3   | Network Protocol           | DataDirect |
| 4   | Pure Java                  | MySQL Connector/J, PostgreSQL |

- **Рекомендуется Type 4** (чистый Java)

## 4. Установка и настройка <a name="установка"></a>

- Добавьте зависимость драйвера (Maven/Gradle)
- Настройте JDBC URL (см. примеры для MySQL, PostgreSQL, Oracle)
- Проверьте совместимость версий драйвера и JDK

## 5. Примеры использования <a name="примеры"></a>

### Базовое подключение
```java
Connection conn = DriverManager.getConnection(url, user, password);
```

### Statement/PreparedStatement
```java
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
pstmt.setInt(1, 1);
ResultSet rs = pstmt.executeQuery();
```

### Batch
```java
PreparedStatement pstmt = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
for (String name : names) {
    pstmt.setString(1, name);
    pstmt.addBatch();
}
pstmt.executeBatch();
```

### Метаданные
```java
DatabaseMetaData meta = conn.getMetaData();
System.out.println(meta.getDatabaseProductName());
```

## 6. Расширенные возможности <a name="расширенные"></a>

- **Connection Pool**: используйте HikariCP, DBCP, c3p0
- **Batch-операции**: ускоряют массовые вставки/обновления
- **Scrollable/Updatable ResultSet**: для сложной навигации по данным
- **Savepoints**: частичный откат транзакций

## 7. Работа с большими данными (BLOB/CLOB, streaming) <a name="большие-данные"></a>

- Используйте методы getBinaryStream/setBinaryStream для BLOB
- Для больших выборок используйте setFetchSize и ResultSet.TYPE_FORWARD_ONLY
- Пример:
```java
PreparedStatement ps = conn.prepareStatement("SELECT data FROM files WHERE id=?");
ps.setInt(1, fileId);
ResultSet rs = ps.executeQuery();
if (rs.next()) {
    InputStream in = rs.getBinaryStream(1);
    // Чтение потока
}
```

## 8. Ошибки и диагностика <a name="ошибки"></a>

- **Типовые ошибки**: SQLException, SQLState, deadlock, timeout, connection refused
- **Диагностика**: анализируйте SQLState, используйте логирование драйвера
- **Пример обработки**:
```java
try {
    // ...
} catch (SQLException e) {
    System.err.println("SQLState: " + e.getSQLState());
    System.err.println("Error Code: " + e.getErrorCode());
    e.printStackTrace();
}
```

## 9. Безопасность <a name="безопасность"></a>

- Используйте PreparedStatement для защиты от SQL-инъекций
- Настройте SSL-соединение (например, ?useSSL=true)
- Не храните пароли в коде, используйте переменные окружения/секреты

## 10. Интеграция с Spring <a name="spring"></a>

- Используйте DataSource и JdbcTemplate
- Пример:
```java
@Configuration
public class JdbcConfig {
    @Bean
    public DataSource dataSource() {
        DriverManagerDataSource ds = new DriverManagerDataSource();
        ds.setUrl("jdbc:mysql://localhost:3306/mydb");
        ds.setUsername("root");
        ds.setPassword("password");
        return ds;
    }
    @Bean
    public JdbcTemplate jdbcTemplate(DataSource ds) {
        return new JdbcTemplate(ds);
    }
}
```

## 11. Сравнение JDBC и ORM <a name="jdbc-vs-orm"></a>

| Критерий         | JDBC                | ORM (Hibernate, JPA)      |
|------------------|---------------------|---------------------------|
| Производительность | Высокая (ручной контроль) | Высокая (кэш, batch) |
| Простота         | Требует много кода  | Меньше кода, декларативно |
| Гибкость         | Максимальная        | Ограничена абстракцией    |
| Транзакции       | Ручное управление   | Автоматизация             |
| Миграции         | Нет                 | Есть (Liquibase, Flyway)  |
| Поддержка БД     | Любая               | Зависит от реализации     |

- **Когда использовать JDBC**: простые задачи, максимальный контроль, интеграция с legacy
- **Когда использовать ORM**: сложные доменные модели, кэширование, миграции

## 12. Best practices <a name="best-practices"></a>

- Используйте try-with-resources для автоматического закрытия
- Предпочитайте PreparedStatement
- Используйте connection pool для многопоточных приложений
- Минимизируйте время жизни соединения
- Логируйте ошибки и SQLState
- Не храните пароли в коде
