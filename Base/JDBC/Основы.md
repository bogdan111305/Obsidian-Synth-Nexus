# JDBC: Основы

> [!QUOTE] JDBC (Java Database Connectivity) — стандартный Java API для работы с реляционными БД. Цепочка: Driver → Connection → Statement → ResultSet.

## Архитектура и компоненты

- **DriverManager** — загрузка драйверов, создание соединений
- **Connection** — сессия с БД
- **Statement / PreparedStatement / CallableStatement** — выполнение SQL
- **ResultSet** — результаты запроса
- **SQLException** — обработка ошибок

> [!INFO] `PreparedStatement` компилирует SQL один раз и позволяет переиспользовать его с разными параметрами. Это быстрее и безопаснее, чем `Statement`.

---

## Типы драйверов JDBC

| Тип | Описание | Статус |
|-----|----------|--------|
| 1 | JDBC-ODBC Bridge | Устарел |
| 2 | Native-API / partly Java | Редко |
| 3 | Network Protocol | Редко |
| 4 | Pure Java | **Рекомендуется** |

Используйте только Type 4 (MySQL Connector/J, PostgreSQL Driver и т.д.).

## Подключение

```java
// Базовое подключение
Connection conn = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/mydb", "user", "password"
);
```

> [!WARNING] Соединение — дорогостоящий ресурс. Не создавайте `Connection` на каждый запрос. Используйте пул соединений ([[Пула Соединений с JDBC]]).

> [!WARNING] Всегда закрывайте `Connection`, `Statement` и `ResultSet`. Незакрытые ресурсы — утечка памяти и исчерпание пула соединений.

```java
// Правильно — try-with-resources
try (Connection conn = dataSource.getConnection();
     PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?")) {
    ps.setInt(1, userId);
    try (ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            // обработка
        }
    }
}
```

## Statement и PreparedStatement

```java
// PreparedStatement — для параметризованных запросов
PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
ps.setInt(1, 1);
ResultSet rs = ps.executeQuery();
```

> [!WARNING] **SQL Injection**: никогда не конкатенируйте пользовательский ввод в строку SQL.
> `"SELECT * FROM users WHERE name = '" + userInput + "'"` — уязвимость!
> Всегда используйте `PreparedStatement` с `?` параметрами.

## Batch-операции

Batch сокращает количество round-trip к БД при массовых вставках/обновлениях.

```java
PreparedStatement ps = conn.prepareStatement("INSERT INTO users (name) VALUES (?)");
conn.setAutoCommit(false);
for (String name : names) {
    ps.setString(1, name);
    ps.addBatch();
}
ps.executeBatch();
conn.commit();
```

> [!INFO] Рекомендуется делать `commit()` после каждого батча (не только в конце), чтобы не держать транзакцию открытой слишком долго.

## Интеграция с Spring

```java
@Configuration
public class JdbcConfig {
    @Bean
    public DataSource dataSource() {
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");
        ds.setUsername("root");
        ds.setPassword("password");
        ds.setMaximumPoolSize(10);
        return ds;
    }

    @Bean
    public JdbcTemplate jdbcTemplate(DataSource ds) {
        return new JdbcTemplate(ds);
    }
}
```

> [!INFO] `JdbcTemplate` устраняет шаблонный код: открытие/закрытие соединений, обработку `SQLException`. Используйте его вместо прямого JDBC в Spring-приложениях.

## Сравнение JDBC и ORM

| Критерий | JDBC | ORM (Hibernate, JPA) |
|----------|------|-----------------------|
| Производительность | Высокая (ручной контроль) | Высокая (кэш, batch) |
| Объём кода | Много шаблонного кода | Меньше кода |
| Гибкость SQL | Максимальная | Ограничена абстракцией |
| Транзакции | Ручное управление | Автоматизация |
| Кэширование | Нет | Есть |
| Миграции | Нет | Liquibase, Flyway |

Подробнее: [[JDBC vs ORM]]

## Best practices

- `try-with-resources` для всех JDBC-ресурсов
- Всегда `PreparedStatement`, никогда `Statement` с конкатенацией
- Connection Pool для любого production-приложения
- Минимизируйте время жизни транзакции
- Логируйте `SQLState` и `ErrorCode` при ошибках
- Не храните пароли в коде — используйте переменные окружения
