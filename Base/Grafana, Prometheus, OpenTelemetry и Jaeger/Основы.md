## Логи: Основы и Важность

### Что такое Логи?

Логи — это записи событий, происходящих в приложении (например, ошибки, предупреждения, информационные сообщения). Они помогают диагностировать проблемы и отслеживать поведение системы.

### Настройка Логов в Spring Boot

Spring Boot использует библиотеку Logback по умолчанию. Для настройки:

1. Откройте или создайте файл `src/main/resources/application.properties`:
    
    ```
    logging.level.root=INFO
    logging.level.com.example=DEBUG
    logging.file.name=app.log
    ```
    
2. Добавьте зависимости в `build.gradle`, если нужно (уже включено в `spring-boot-starter`):
    
    ```
    implementation 'org.springframework.boot:spring-boot-starter'
    ```
    
3. Используйте в коде:
    
    ```java
    import org.slf4j.Logger;
    import org.slf4j.LoggerFactory;
    
    public class MyService {
        private static final Logger logger = LoggerFactory.getLogger(MyService.class);
    
        public void doSomething() {
            logger.info("Выполняется операция...");
            try {
                // Логика
            } catch (Exception e) {
                logger.error("Ошибка: ", e);
            }
        }
    }
    ```
    

### Как Работают Логи?

- **Механизм**: Logback обрабатывает вызовы `logger` и записывает сообщения в файл, консоль или внешнюю систему (например, ELK Stack) в зависимости от конфигурации (`logback.xml` или `application.properties`).
- **Уровни**: Логи фильтруются по уровням (TRACE, DEBUG, INFO, WARN, ERROR), что позволяет включать или отключать детализацию.
- **Асинхронность**: Для производительности можно настроить асинхронную запись логов, уменьшая задержки в приложении.
- **Интеграция**: Spring Boot автоматически интегрирует логи с Actuator, позволяя экспортировать их в централизованные системы.

### Лучшие Практики

- Используйте уровни логирования (TRACE, DEBUG, INFO, WARN, ERROR).
- Избегайте избыточных логов, чтобы не перегружать систему.

## Метрики: Измерение Производительности

### Что такое Метрики?

Метрики — это количественные данные, такие как время ответа, количество запросов или использование памяти, которые собираются для анализа производительности.

### Введение в Prometheus

Prometheus — это система мониторинга и временных рядов, которая собирает метрики через HTTP-endpoints.

#### Установка Prometheus

1. Скачайте Prometheus с официального сайта ([https://prometheus.io/download/](https://prometheus.io/download/)).
2. Настройте файл `prometheus.yml`:
    
    ```
    scrape_configs:
      - job_name: 'spring-app'
        static_configs:
          - targets: ['localhost:8080']
    ```
    
3. Запустите: `./prometheus --config.file=prometheus.yml`.

#### Интеграция с Spring Boot

Добавьте зависимость:

```
implementation 'io.micrometer:micrometer-registry-prometheus'
```

Включите Actuator и экспорт метрик:

```
management.endpoints.web.exposure.include=prometheus
management.endpoint.prometheus.enabled=true
```

Метрики будут доступны по `/actuator/prometheus`.

### Введение в Grafana

Grafana — это инструмент визуализации, который работает с Prometheus для создания дашбордов.

#### Установка Grafana

1. Скачайте Grafana ([https://grafana.com/get](https://grafana.com/get)).
2. Запустите: `./grafana-server`.
3. Доступ через браузер: `http://localhost:3000`.

#### Настройка

- Добавьте источник данных (Data Source): выберите Prometheus и укажите URL (например, `http://localhost:9090`).
- Создайте дашборд, добавив панели с запросами (например, `rate(http_server_requests_seconds_count[5m])`).

### Как Работают Метрики?

- **Сбор Данных**: Prometheus периодически (по умолчанию каждые 15 секунд) запрашивает метрики через HTTP-endpoint (например, `/actuator/prometheus`), используя pull-модель. Метрики представлены в формате временных рядов (time series) с метками (labels).
- **Хранение**: Данные хранятся локально в базе Prometheus с поддержкой сжатия и экспирации (обычно до 15 дней, зависит от настроек).
- **Обработка**: Grafana запрашивает данные из Prometheus через PromQL (язык запросов) и отображает их в виде графиков или таблиц.
- **Алертинг**: Prometheus поддерживает алерты, которые можно настроить для уведомлений (например, через email или Slack).
- **Интеграция**: Spring Boot Actuator предоставляет встроенные метрики (например, `http_server_requests`, `jvm_memory_used`), которые можно расширить кастомными метриками через Micrometer.

## Трейсинг: Отслеживание Запросов

### Что такое Трейсинг?

Трейсинг позволяет отслеживать путь запроса через систему, выявляя узкие места и ошибки.

### Введение в OpenTelemetry

OpenTelemetry — это набор инструментов для сбора и экспорта телеметрии (логов, метрик, трейсов).

#### Установка

1. Добавьте зависимость в `build.gradle`:
    
    ```
    implementation 'io.opentelemetry:opentelemetry-api:1.42.0'
    implementation 'io.opentelemetry:opentelemetry-sdk:1.42.0'
    ```
    
2. Настройте экспортер (например, для Jaeger):
    
    ```java
    import io.opentelemetry.api.OpenTelemetry;
    import io.opentelemetry.api.trace.Tracer;
    import io.opentelemetry.sdk.OpenTelemetrySdk;
    import io.opentelemetry.exporters.jaeger.JaegerGrpcSpanExporter;
    
    public class TelemetryConfig {
        public static OpenTelemetry init() {
            JaegerGrpcSpanExporter exporter = JaegerGrpcSpanExporter.builder()
                .setEndpoint("http://localhost:14250")
                .build();
            return OpenTelemetrySdk.builder()
                .setTracerProvider(SdkTracerProvider.builder()
                    .addSpanProcessor(SimpleSpanProcessor.create(exporter))
                    .build())
                .buildAndRegisterGlobal();
        }
    }
    ```
    
3. Инициализируйте в `main`:
    
    ```java
    public static void main(String[] args) {
        TelemetryConfig.init();
        SpringApplication.run(Application.class, args);
    }
    ```
    

### Введение в Jaeger

Jaeger — это система трейсинга, совместимая с OpenTelemetry.

#### Установка

1. Запустите Jaeger через Docker:
    
    ```bash
    docker run -d -p 16686:16686 -p 14268:14268 jaegertracing/all-in-one:latest
    ```
    
2. Доступ через браузер: `http://localhost:16686`.

#### Настройка

- Настройте OpenTelemetry для отправки трейсов на Jaeger (как в примере выше).
- Отслеживайте запросы в интерфейсе Jaeger.

### Как Работает Трейсинг?

- **Генерация Трейсов**: OpenTelemetry создает "спаны" (единицы работы, например, HTTP-запрос) и связывает их в "трейсы" (полный путь запроса). Каждый спан имеет идентификатор, временные метки и атрибуты.
- **Контекст**: Трейсинг использует контекст распространения (propagation context), передаваемый через заголовки (например, `traceparent`), чтобы связать спаны между сервисами.
- **Экспорт**: Спаны отправляются на бэкенд (например, Jaeger) через экспортеры (gRPC, HTTP). OpenTelemetry поддерживает асинхронную отправку для минимизации влияния на производительность.
- **Хранение и Визуализация**: Jaeger сохраняет трейсы в базе данных (обычно Cassandra или Elasticsearch) и предоставляет веб-интерфейс для анализа (например, задержки, ошибки).
- **Интеграция**: Spring Boot интегрирует OpenTelemetry через Actuator или кастомные интерцепторы, автоматически отслеживая запросы.

## Объединение Инструментов

### Архитектура

- **Prometheus**: Собирает метрики с использованием pull-модели.
- **Grafana**: Визуализирует метрики через запросы к Prometheus.
- **OpenTelemetry**: Генерирует и экспортирует трейсы.
- **Jaeger**: Хранит и отображает трейсы.

### Как Это Работает Вместе?

- **Сбор Данных**: Приложение экспортирует метрики (Prometheus) и трейсы (OpenTelemetry) через HTTP или gRPC.
- **Обработка**: Prometheus агрегирует метрики, а Jaeger собирает трейсы в реальном времени.
- **Визуализация и Анализ**: Grafana отображает метрики на дашбордах, а Jaeger предоставляет детализированные трейсы для отладки.
- **Алертинг**: Prometheus и Grafana могут отправлять уведомления при превышении порогов (например, если время ответа > 500 мс).
- **Интеграция с Приложением**: Spring Boot Actuator выступает мостом, предоставляя эндпоинты для метрик и поддерживая трейсинг через OpenTelemetry.

## Лучшие Практики

### Логи

- Храните логи централизованно (например, с ELK Stack).
- Используйте структурированные логи (JSON) для упрощения анализа.

### Метрики

- Собирайте ключевые метрики: latency, throughput, error rate.
- Настраивайте алерты в Grafana для своевременного реагирования.

### Трейсинг

- Добавляйте контекст трейсинга в каждый сервис (например, корреляционные ID).
- Используйте семантические атрибуты (например, `http.method`, `db.operation`) для детализации.

## Заключение

Логи, метрики и трейсинг работают как единая система мониторинга. Логи фиксируют события, метрики (с Prometheus и Grafana) измеряют производительность, а трейсинг (с OpenTelemetry и Jaeger) отслеживает запросы. Понимание их работы позволяет эффективно управлять приложением. Начните с базовой настройки логов в Spring Boot, подключите Prometheus и Grafana для метрик, а затем добавьте OpenTelemetry с Jaeger для трейсинга. Экспериментируйте с тестовым приложением, чтобы освоить эти инструменты!
